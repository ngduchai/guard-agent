# ExaMPM: Manual Build, Execute & Validation Guide

This guide explains how to build, run, and compare the **original** (baseline) and **resilient** (VeloC-enabled) ExaMPM implementations without using the automated validation scripts.

---

## 1. Overview

| Aspect | Original (Baseline) | Resilient (VeloC) |
|--------|---------------------|-------------------|
| **Location** | `build/ecp/ExaMPM` | `build/ecp_output/resilient_ExaMPM` |
| **Source origin** | Copied from `tests/ecp/vanillas/ExaMPM` by `setup.sh` | Generated by the VeloC LLM agent |
| **Dependencies** | Kokkos, Cabana (MPI, HDF5), CMake | Kokkos, Cabana (MPI, HDF5), CMake, **VeloC** |
| **Executables** | `DamBreak`, `FreeFall` | `DamBreak`, `FreeFall` (same names) |
| **Key difference** | No checkpoint/restart | VeloC checkpoint/restart integrated |

### What is ExaMPM?

ExaMPM is an ECP CoPA proxy application implementing the **Material Point Method** (MPM) for fluid/solid simulation. It uses [Cabana](https://github.com/ECP-copa/Cabana) (which uses [Kokkos](https://github.com/kokkos/kokkos)) for performance-portable particle and grid operations.

The two example simulations are:
- **DamBreak** — A static water column collapses under gravity in a unit cube domain with free-slip boundary conditions
- **FreeFall** — A sphere of material falls under gravity in a periodic domain

### What the resilient version adds

The LLM agent modified **5 files** and added **1 new file** to integrate VeloC checkpoint/restart:

| File | Changes |
|------|---------|
| `CMakeLists.txt` (root) | Added `VELOC_DIR` cache variable for VeloC installation path |
| `src/CMakeLists.txt` | Added `${VELOC_DIR}/include` to target include directories |
| `examples/CMakeLists.txt` | Added VeloC include dirs, link dirs, `veloc-client` library, and RPATH for both executables |
| `examples/dam_break.cpp` | Added `#include <veloc.h>`, `--veloc-cfg` CLI arg parsing, `VELOC_Init()` / `VELOC_Finalize()` calls |
| `examples/free_fall.cpp` | Same VeloC lifecycle additions as `dam_break.cpp` |
| `src/ExaMPM_Solver.hpp` | Added `#include <veloc.h>`, `writeCheckpoint()` and `tryRestart()` methods, modified `solve()` to call them |
| `src/ExaMPM_ProblemManager.hpp` | Added `resizeParticles()` method (needed for checkpoint restore) |
| `veloc.cfg` | **New file** — VeloC configuration (scratch/persistent paths, sync mode) |

### Correctness guarantee

The resilient version produces **identical physics output** (time progress lines) as the original for the same input, because:
- The simulation algorithm (MPM time integration, particle-to-grid, grid-to-particle) is unchanged
- VeloC only adds checkpoint writes (every `write_freq` steps) and optional restart reads
- On a fresh run (no prior checkpoint), `tryRestart()` finds no checkpoint and the loop runs identically
- The checkpoint saves/restores all particle state: affine matrix (3×3), velocity (3), position (3), mass, volume, and deformation gradient determinant (J)

## 5. Understanding the CLI Arguments

Both `DamBreak` and `FreeFall` take the same 7 positional arguments:

```
Usage: ./DamBreak cell_size parts_per_cell halo_cells dt t_end write_freq exec_space

Arguments:
  cell_size       Edge length of a computational cell (domain is unit cube)
  parts_per_cell  Particles per cell in each direction (e.g., 2)
  halo_cells      Number of halo cells (0 for minimal)
  dt              Time step size (e.g., 0.001)
  t_end           Simulation end time (e.g., 0.10)
  write_freq      Number of steps between output/progress lines (e.g., 10)
  exec_space      Execution backend: serial, openmp, cuda, hip
```

The resilient version adds one optional argument:
```
  --veloc-cfg <path>   Path to VeloC config file (default: veloc.cfg in CWD)
```

### Recommended test parameters

| Parameter | Quick Test | Medium Test | Long Test |
|-----------|-----------|-------------|-----------|
| cell_size | 0.1 | 0.1 | 0.1 |
| parts_per_cell | 2 | 2 | 2 |
| halo_cells | 0 | 0 | 0 |
| dt | 0.001 | 0.001 | 0.001 |
| t_end | 0.05 | 0.10 | 0.20 |
| write_freq | 10 | 10 | 10 |
| exec_space | serial | serial | serial |
| ~Timesteps | 50 | 100 | 200 |

## 6. Running the Original (Baseline)

```bash
# From the original build directory:
cd build/ecp/ExaMPM/build_dir

# Run DamBreak with medium test parameters (1 MPI rank)
mpirun -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial 2>&1 | tee /tmp/exampm_original_stdout.txt

# Or with 2 MPI ranks:
mpirun -np 2 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial 2>&1 | tee /tmp/exampm_original_stdout.txt
```

### Expected output format

The solver prints a progress line every `write_freq` steps (only on rank 0):

```
Time 0.000000 / 0.100000
Time 0.010000 / 0.100000
Time 0.020000 / 0.100000
Time 0.030000 / 0.100000
Time 0.040000 / 0.100000
Time 0.050000 / 0.100000
Time 0.060000 / 0.100000
Time 0.070000 / 0.100000
Time 0.080000 / 0.100000
Time 0.090000 / 0.100000
```

If Cabana was built with HDF5 support, particle data files (`particles_*.h5`) are also written every `write_freq` steps. Without HDF5, a warning is printed but the simulation still runs correctly.

## 7. Running the Resilient (VeloC) Version

### 7.1 Prepare VeloC directories and config

The resilient version needs a VeloC configuration file. One is provided at `build/ecp_output/resilient_ExaMPM/veloc.cfg`:

```ini
scratch=/tmp/exampm_scratch
persistent=/tmp/exampm_persistent
mode=sync
max_versions=3
scratch_versions=3
```

Create the scratch/persistent directories:

```bash
mkdir -p /tmp/exampm_scratch /tmp/exampm_persistent
```

### 7.2 Fresh run (no prior checkpoint)

```bash
# Clean any previous checkpoints
rm -rf /tmp/exampm_scratch/* /tmp/exampm_persistent/*

# From the resilient build directory:
cd build/ecp_output/resilient_ExaMPM/build_dir

# Run DamBreak with the same parameters as the original
mpirun -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial \
  --veloc-cfg ../veloc.cfg 2>&1 | tee /tmp/exampm_resilient_stdout.txt

# Or with 2 MPI ranks:
mpirun -np 2 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial \
  --veloc-cfg ../veloc.cfg 2>&1 | tee /tmp/exampm_resilient_stdout.txt
```

### 7.3 Restart run (from checkpoint)

To test the checkpoint/restart functionality:

```bash
# Step 1: Run a partial simulation (e.g., t_end=0.05)
rm -rf /tmp/exampm_scratch/* /tmp/exampm_persistent/*
cd build/ecp_output/resilient_ExaMPM/build_dir

mpirun -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.05 10 serial \
  --veloc-cfg ../veloc.cfg 2>&1 | tee /tmp/exampm_partial_stdout.txt

# Verify checkpoint files were created
ls -la /tmp/exampm_scratch/

# Step 2: Run the full simulation (t_end=0.10) — it will restart from checkpoint
mpirun -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial \
  --veloc-cfg ../veloc.cfg 2>&1 | tee /tmp/exampm_restart_stdout.txt

# You should see a restart message like:
#   VeloC: Restarting from checkpoint version 50
#   VeloC: Restored step=50 time=0.050000 dt=1.000000e-03 num_particles=...
```

## 8. Comparing Output (Manual Validation)

### 8.1 What to compare

The primary output to compare is the **time progress lines** printed to stdout. These lines have the format:

```
Time <current_time> / <t_final>
```

Both versions should produce identical time progress values for the same input parameters when the resilient version runs fresh (no prior checkpoint).

### 8.2 Quick visual comparison

```bash
# Extract only the "Time X / Y" lines from both outputs
grep "^Time " /tmp/exampm_original_stdout.txt > /tmp/original_times.txt
grep "^Time " /tmp/exampm_resilient_stdout.txt > /tmp/resilient_times.txt

# Compare them
diff /tmp/original_times.txt /tmp/resilient_times.txt
```

If `diff` produces no output, the simulations are **identical** — validation passes.

### 8.3 Detailed numerical comparison

For a more rigorous comparison with tolerance:

```bash
# Parse and compare time values with tolerance
python3 -c "
import re, sys

def parse_times(filename):
    times = []
    with open(filename) as f:
        for line in f:
            m = re.match(r'^\s*Time\s+([0-9eE.+-]+)\s*/\s*([0-9eE.+-]+)\s*$', line)
            if m:
                times.append((float(m.group(1)), float(m.group(2))))
    return times

orig = parse_times('/tmp/original_times.txt')
resil = parse_times('/tmp/resilient_times.txt')

print(f'Original:  {len(orig)} time steps')
print(f'Resilient: {len(resil)} time steps')

if len(orig) != len(resil):
    print(f'WARNING: Different number of time steps ({len(orig)} vs {len(resil)})')
    print('  (This is expected if the resilient version restarted from a checkpoint)')

# Compare common steps
rtol = 1e-6
mismatches = 0
n_compared = min(len(orig), len(resil))

for i in range(n_compared):
    t_orig, tf_orig = orig[i]
    t_resil, tf_resil = resil[i]
    denom = max(abs(t_orig), abs(t_resil), 1e-300)
    rel_err = abs(t_orig - t_resil) / denom
    if rel_err > rtol:
        mismatches += 1
        print(f'  MISMATCH at step {i}: orig={t_orig:.10f} resil={t_resil:.10f} rel_err={rel_err:.2e}')

if mismatches == 0:
    print(f'PASS: All {n_compared} common time steps match within rtol={rtol:.1e}')
else:
    print(f'FAIL: {mismatches}/{n_compared} time steps have mismatches')
"
```

### 8.4 Comparing HDF5 particle output (if available)

If Cabana was built with HDF5 support, both versions produce `particles_*.h5` files. You can compare them:

```bash
# Install h5py if not already available
pip install h5py numpy

# Compare particle output files
python3 -c "
import h5py
import numpy as np
import glob
import sys

orig_files = sorted(glob.glob('build/ecp/ExaMPM/build_dir/particles_*.h5'))
resil_files = sorted(glob.glob('build/ecp_output/resilient_ExaMPM/build_dir/particles_*.h5'))

if not orig_files:
    print('No HDF5 output files found for original (Cabana may not have HDF5 support)')
    sys.exit(0)
if not resil_files:
    print('No HDF5 output files found for resilient')
    sys.exit(0)

print(f'Original files:  {len(orig_files)}')
print(f'Resilient files: {len(resil_files)}')

# Compare the last output file (final state)
for of, rf in zip(orig_files, resil_files):
    with h5py.File(of, 'r') as ho, h5py.File(rf, 'r') as hr:
        for key in ho.keys():
            if key in hr:
                orig_data = np.array(ho[key])
                resil_data = np.array(hr[key])
                if orig_data.shape == resil_data.shape:
                    max_diff = np.max(np.abs(orig_data - resil_data))
                    print(f'  {of} vs {rf}: key={key} max_diff={max_diff:.2e}')
                else:
                    print(f'  {of} vs {rf}: key={key} SHAPE MISMATCH {orig_data.shape} vs {resil_data.shape}')
"
```

## 9. Validation Checklist

Use this checklist to validate the resilient implementation:

### ✅ Build Validation

- [ ] Original ExaMPM builds without errors
- [ ] Resilient ExaMPM builds without errors
- [ ] Resilient binary links against `libveloc-client.so` (check with `ldd`)

### ✅ Fresh Run Correctness (no checkpoint)

- [ ] Clean checkpoint directories: `rm -rf /tmp/exampm_scratch/* /tmp/exampm_persistent/*`
- [ ] Run original with: `mpirun -np 1 ./DamBreak 0.1 2 0 0.001 0.10 10 serial`
- [ ] Run resilient with same args + `--veloc-cfg ../veloc.cfg`
- [ ] Compare time progress lines: `diff` shows no differences
- [ ] Both produce the same number of output steps

### ✅ Checkpoint Creation

- [ ] After resilient run, checkpoint files exist in `/tmp/exampm_scratch/`
- [ ] Checkpoint files are non-empty binary files

### ✅ Restart Correctness

- [ ] Run resilient with `t_end=0.05` (partial run, creates checkpoint)
- [ ] Run resilient again with `t_end=0.10` (should restart from checkpoint)
- [ ] Restart message appears: `VeloC: Restarting from checkpoint version ...`
- [ ] Restored state message appears: `VeloC: Restored step=... time=... dt=...`
- [ ] Final output matches a fresh full run with `t_end=0.10`

### ✅ Code Diff Validation

- [ ] Only expected files were modified (see Section 1 table)
- [ ] Physics code (`ExaMPM_TimeIntegrator.hpp`, `ExaMPM_TimeStepControl.hpp`, etc.) is unchanged
- [ ] No simulation parameters were altered

## 11. Complete End-to-End Validation Script

Here is a self-contained bash script you can copy-paste to perform the full validation:

```bash
#!/usr/bin/env bash
set -euo pipefail

REPO_ROOT="/home/ndhai/diaspora/guard-agent"
ORIG_DIR="${REPO_ROOT}/build/ecp/ExaMPM"
RESIL_DIR="${REPO_ROOT}/build/ecp_output/resilient_ExaMPM"
VELOC_DIR="/home/ndhai/usr"
OUTPUT_DIR="/tmp/exampm_validation"

echo "=== ExaMPM Manual Validation ==="
echo ""

# --- Step 1: Build original ---
echo "[1/6] Building original ExaMPM..."
cd "${ORIG_DIR}"
rm -rf build_dir && mkdir build_dir && cd build_dir
cmake .. -DCMAKE_PREFIX_PATH="$HOME/.local" 2>&1 | tail -5
cmake --build . --parallel $(nproc) 2>&1 | tail -3
echo "  ✓ Original built: $(ls examples/DamBreak)"

# --- Step 2: Build resilient ---
echo "[2/6] Building resilient ExaMPM..."
cd "${RESIL_DIR}"
rm -rf build_dir && mkdir build_dir && cd build_dir
cmake .. -DCMAKE_PREFIX_PATH="$HOME/.local" -DVELOC_DIR="${VELOC_DIR}" 2>&1 | tail -5
cmake --build . --parallel $(nproc) 2>&1 | tail -3
echo "  ✓ Resilient built: $(ls examples/DamBreak)"
echo "  ✓ VeloC linked: $(ldd examples/DamBreak | grep veloc || echo 'NOT FOUND')"

# --- Step 3: Run original ---
echo "[3/6] Running original DamBreak..."
mkdir -p "${OUTPUT_DIR}"
cd "${ORIG_DIR}/build_dir"
mpirun --oversubscribe -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial \
  2>&1 | tee "${OUTPUT_DIR}/original_stdout.txt"

# --- Step 4: Run resilient (fresh, no checkpoint) ---
echo "[4/6] Running resilient DamBreak (fresh)..."
rm -rf /tmp/exampm_scratch /tmp/exampm_persistent
mkdir -p /tmp/exampm_scratch /tmp/exampm_persistent
cd "${RESIL_DIR}/build_dir"
mpirun --oversubscribe -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial \
  --veloc-cfg "${RESIL_DIR}/veloc.cfg" \
  2>&1 | tee "${OUTPUT_DIR}/resilient_stdout.txt"

# --- Step 5: Compare outputs ---
echo "[5/6] Comparing outputs..."
grep "^Time " "${OUTPUT_DIR}/original_stdout.txt" > "${OUTPUT_DIR}/original_times.txt"
grep "^Time " "${OUTPUT_DIR}/resilient_stdout.txt" > "${OUTPUT_DIR}/resilient_times.txt"

if diff -q "${OUTPUT_DIR}/original_times.txt" "${OUTPUT_DIR}/resilient_times.txt" > /dev/null 2>&1; then
  echo "  ✓ PASS: Time progress lines are IDENTICAL"
else
  echo "  ✗ FAIL: Time progress lines DIFFER"
  diff "${OUTPUT_DIR}/original_times.txt" "${OUTPUT_DIR}/resilient_times.txt" || true
fi

# --- Step 6: Test restart ---
echo "[6/6] Testing checkpoint/restart..."
rm -rf /tmp/exampm_scratch /tmp/exampm_persistent
mkdir -p /tmp/exampm_scratch /tmp/exampm_persistent
cd "${RESIL_DIR}/build_dir"

# Partial run (t_end=0.05)
echo "  Running partial simulation (t_end=0.05)..."
mpirun --oversubscribe -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.05 10 serial \
  --veloc-cfg "${RESIL_DIR}/veloc.cfg" \
  2>&1 | tee "${OUTPUT_DIR}/partial_stdout.txt"

CKPT_COUNT=$(find /tmp/exampm_scratch -name "*.dat" 2>/dev/null | wc -l)
echo "  Checkpoint files created: ${CKPT_COUNT}"

# Full run with restart (t_end=0.10)
echo "  Running full simulation with restart (t_end=0.10)..."
mpirun --oversubscribe -np 1 ./examples/DamBreak 0.1 2 0 0.001 0.10 10 serial \
  --veloc-cfg "${RESIL_DIR}/veloc.cfg" \
  2>&1 | tee "${OUTPUT_DIR}/restart_stdout.txt"

if grep -q "VeloC: Restarting from checkpoint" "${OUTPUT_DIR}/restart_stdout.txt"; then
  echo "  ✓ Restart detected in output"
else
  echo "  ✗ No restart message found"
fi

# Compare restart output with fresh full run
grep "^Time " "${OUTPUT_DIR}/restart_stdout.txt" | sort -u > "${OUTPUT_DIR}/restart_times.txt"
if diff -q "${OUTPUT_DIR}/original_times.txt" "${OUTPUT_DIR}/restart_times.txt" > /dev/null 2>&1; then
  echo "  ✓ PASS: Restarted run produces same time steps as original"
else
  echo "  Note: Restarted run may have fewer early time steps (expected behavior)"
  # Check that all restart times are a subset of original times
  EXTRA=$(comm -23 "${OUTPUT_DIR}/restart_times.txt" "${OUTPUT_DIR}/original_times.txt" | wc -l)
  if [ "${EXTRA}" -eq 0 ]; then
    echo "  ✓ PASS: All restart time steps match original (subset)"
  else
    echo "  ✗ FAIL: Restart produced unexpected time values"
  fi
fi

echo ""
echo "=== Validation Complete ==="
echo "Output files saved to: ${OUTPUT_DIR}/"
```

## 12. Execution Flow Comparison

### Original ExaMPM flow

```
main()
  ├── MPI_Init()
  ├── Kokkos::initialize()
  ├── Parse CLI args (7 positional)
  ├── damBreak() / freeFall()
  │     ├── Create mesh, partitioner, boundary conditions
  │     ├── createSolver() → Solver constructor
  │     │     ├── Create Mesh
  │     │     ├── Create ProblemManager (init particles)
  │     │     └── Get MPI rank
  │     └── solver->solve(t_final, write_freq)
  │           ├── outputParticles()  [initial state]
  │           └── while (time < t_final)
  │                 ├── Print "Time X / Y" (every write_freq)
  │                 ├── timeStepControl() → compute dt
  │                 ├── TimeIntegrator::step() → P2G, grid update, G2P
  │                 ├── communicateParticles()
  │                 ├── time += dt; step++
  │                 └── outputParticles() (every write_freq)
  ├── Kokkos::finalize()
  └── MPI_Finalize()
```

### Resilient ExaMPM flow (additions in **bold**)

```
main()
  ├── MPI_Init()
  ├── **Parse --veloc-cfg argument**
  ├── **VELOC_Init(MPI_COMM_WORLD, veloc_cfg)**
  ├── Kokkos::initialize()
  ├── Parse CLI args (7 positional)
  ├── damBreak() / freeFall()
  │     ├── Create mesh, partitioner, boundary conditions
  │     ├── createSolver() → Solver constructor
  │     │     ├── Create Mesh
  │     │     ├── Create ProblemManager (init particles)
  │     │     └── Get MPI rank
  │     └── solver->solve(t_final, write_freq)
  │           ├── **tryRestart()** → check for VeloC checkpoint
  │           │     ├── VELOC_Restart_test("exampm", 0)
  │           │     ├── If found: VELOC_Restart_begin → read file → restore state
  │           │     └── If not found: return false
  │           ├── outputParticles()  [only if NOT restarted]
  │           └── while (time < t_final)
  │                 ├── Print "Time X / Y" (every write_freq)
  │                 ├── timeStepControl() → compute dt
  │                 ├── TimeIntegrator::step() → P2G, grid update, G2P
  │                 ├── communicateParticles()
  │                 ├── time += dt; step++
  │                 ├── outputParticles() (every write_freq)
  │                 └── **writeCheckpoint(step)** (every write_freq)
  │                       ├── VELOC_Checkpoint_begin("exampm", version)
  │                       ├── VELOC_Route_file("exampm_ckpt.dat")
  │                       ├── Write: step, time, dt, num_particles
  │                       ├── Write: per-particle [affine(9), vel(3), pos(3), mass, vol, J]
  │                       └── VELOC_Checkpoint_end(valid)
  ├── Kokkos::finalize()
  ├── **VELOC_Finalize(1)**
  └── MPI_Finalize()
```

## 13. Troubleshooting

### Build errors

| Error | Solution |
|-------|----------|
| `Could not find Cabana` | Run `./install_deps.sh` in the ExaMPM directory, or set `-DCMAKE_PREFIX_PATH` |
| `Cabana must be compiled with MPI` | Reinstall Cabana with MPI support |
| `veloc.h: No such file` | Set `-DVELOC_DIR=/home/ndhai/usr` in cmake |
| `cannot find -lveloc-client` | Check VeloC is installed at the VELOC_DIR path |

### Runtime errors

| Error | Solution |
|-------|----------|
| `Serial Backend Not Enabled` | Rebuild Kokkos with `-DKokkos_ENABLE_SERIAL=ON` |
| `OpenMP Backend Not Enabled` | Rebuild Kokkos with `-DKokkos_ENABLE_OPENMP=ON` |
| `VeloC: Checkpoint_begin failed` | Check veloc.cfg paths exist and are writable |
| `No particle output enabled` | Rebuild Cabana with `-DCabana_REQUIRE_HDF5=ON` (optional) |
| MPI errors with `-np 2` | Try `--oversubscribe` flag or use `-np 1` |

### VeloC config issues

The `veloc.cfg` file must have valid, writable paths:

```ini
scratch=/tmp/exampm_scratch       # Fast local storage for checkpoints
persistent=/tmp/exampm_persistent # Persistent storage (can be same as scratch)
mode=sync                         # sync or async
max_versions=3                    # Keep last N checkpoint versions
scratch_versions=3                # Keep last N scratch versions
```

Create the directories before running:
```bash
mkdir -p /tmp/exampm_scratch /tmp/exampm_persistent
```
