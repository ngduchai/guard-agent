# Cross-app state-coverage matrix (fast tier)

**Scope**: synthesises the ┬π1.4 *State coverage* tables of the six fast-tier per-app reports
([CoMD](CoMD.md), [HPCG](HPCG.md), [SPARTA](SPARTA.md), [Athena++](Athena++.md),
[CLAMR](CLAMR.md), [PRK_Stencil](PRK_Stencil.md))
into one normalised view. The intent is to surface systematic patterns έΑΦ
where the LLM and the upstream-reference checkpointers agree, and where they
systematically diverge.

**Companion figure**: [`docs/figs/fast_tier_state_coverage.{pdf,png}`](../figs/fast_tier_state_coverage.png)
(generated by [`docs/figs/_plot_fast_tier_state_coverage.py`](../figs/_plot_fast_tier_state_coverage.py)).

## 2. Patterns

### 2.1 Universal agreement (LLM έΚκ Reference on 5 out of 15 rows)

Rows **A**, **B**, **C**, **M** are the structural skeleton of every checkpoint
in the dataset. Both implementations agree on what to save and what to skip:

- **Save**: the loop progress counter, the primary mutating per-element
  arrays, and the per-rank dynamic counts that size those arrays.
- **Skip**: halo / ghost / persistent MPI buffers έΑΦ repopulated by the next
  halo exchange on the first post-restart timestep.

The structural skeleton is so consistent that no LLM iteration in the dataset
ever attempted to checkpoint halo data, and no LLM iteration ever forgot to
checkpoint the primary mutating arrays. This is the *low-decision* part of the
problem and the LLM gets it right on iteration 1 every time.

### 2.2 The systematic LLMέΗΦReference divergence: trust deterministic re-init

Rows **I**, **J**, and parts of **K** are the load-bearing divergence:

- **Geometry / decomposition** (Domain, LinkCell, RCB partition, full
  `MallocPlus` dictionary): all five LLM implementations that have a
  meaningful row-I entry **skip** it. Three out of four references that have a
  meaningful row-I entry **save** it.
- **Species / material tables**: the LLM skips both available rows; the
  reference saves both.

The LLM consistently applies one principle: **"if it is deterministic from CLI
arguments + input files + the existing init code, do not checkpoint it."**
The reference implementations were typically written before the application
acquired robust deterministic-init guarantees, and they adopted the safer
"persist everything reachable from the data structure" stance.

This is responsible for most of the per-frame size delta between LLM and
reference: see the bytes column of the
[`_summary.md`](_summary.md) table έΑΦ LLM payloads are 1.3 ├Ω (CoMD) to 12 ├Ω
(HPCG) smaller per frame than the reference's full-state dumps.

### 2.3 The single LLM-only invention (row N)

SPARTA's stats-history mirror buffer (row N, έαΣ/έαΡ) is the only state class in
the entire dataset that the **LLM saves and the reference does not**. It is
not scientific state at all έΑΦ it is a rank-0 string that mirrors the lines
SPARTA prints to stdout, persisted into the checkpoint and replayed on
restart so the harness's stdout-keep-pattern (`   20000`) finds a match.
See [SPARTA.md ┬π1.1 iter 2](SPARTA.md#iter-2--1908s---pass) for the
discovery chain.

This is harness-induced state, not application state, and it is the single
cleanest example in the dataset of the LLM detecting and patching a
*test-contract* failure rather than a *correctness* failure. No reference
implementation has any analogue because the reference was not designed
against a black-box stdout comparator.

### 2.4 Defensive over-protection (row H)

HPCG's `optMaxIters / optTolerance / optNiters / opt_worst_time / err_count
/ tolerance_failures / global_failure` (row H, έαΣ/έαΡ) is the inverse pattern:
the LLM saves these values *defensively* in case `OptimizeProblem` produces
slightly different numbers on restart due to wall-time-driven sizing of
`numberOfCgSets`. The reference does not save them έΑΦ it re-derives and
**rejects** the load on mismatch (`hpcg_ckpt_load` returns false έΗΤ cold
start). The LLM's overwrite-locals strategy is more robust to
non-deterministic re-init paths but commits more bytes per frame.

### 2.5 Where the table reveals app diversity (rows G, K, L)

- **RNG state (G)**: only SPARTA needed it (DSMC is stochastic). Detecting
  this required the LLM to grep `collide_vss.cpp` callers and design a
  serialization API for the *private* `RanKnuth::ma[56]` vector.
- **AMR mesh topology (K)**: only Athena++ needed it explicitly (CLAMR
  encodes it as flat per-cell tuples).
- **Sparse-matrix / multigrid hierarchy (L)**: only HPCG had it; both
  implementations skip it because `GenerateProblem` /
  `GenerateCoarseProblem` is fully deterministic from `--nx/--ny/--nz`.

These are the "long-tail" rows where the choice is dictated by a single
domain-specific structural property.

## 3. What the matrix says about LLM resilience engineering

1. **The LLM is structurally aligned with the reference on 5 out of 15
   rows** έΑΦ the load-bearing core of any checkpoint frame (counter,
   primary arrays, sizes, halo-skip).

2. **The LLM's distinctive pattern is "save-only-the-mutable-delta"** έΑΦ it
   trusts the application's existing deterministic-init paths to rebuild
   geometry, decomposition, species, and matrix structure. This produces
   1.3 ├Ω έΑΦ 12 ├Ω smaller per-frame payloads but pushes more reconstruction
   work onto the first post-restart timestep (typically `set_bounds()` +
   neighbour computation, or `OptimizeProblem` + `TestCG`).

3. **The LLM occasionally invents non-application state** (SPARTA stats
   buffer) when the test contract demands it. References never do this
   because they were not built against test contracts.

4. **The LLM occasionally over-protects** (HPCG solver tunables) when the
   restart-vs-original determinism is hard to verify from source alone.

5. **No state class is ever saved by the reference and skipped by the LLM
   for a *correctness* reason in the fast tier** έΑΦ every row-I/J/K
   divergence is an LLM optimisation, not an LLM error. All six apps
   produce bit-identical (or numerically-equivalent within the validator's
   tolerance) output on the failure-prone path after the LLM's checkpoint
   succeeds.

## 4. Caveats

- **Mid- and slow-tier apps are not yet included.** Several mid-tier apps
  (HyPar, LAMMPS, SAMRAI, ROSS) have richer state classes (variable-order
  finite-volume, neighbour lists with cutoffs, hierarchical AMR with patch
  levels, optimistic time warp) that may add 3 έΑΦ 5 new rows. This page
  will be extended once those reports land in
  [`docs/llm_resilience_report/`](.).
- **Row granularity is coarse.** "Per-rank dynamic counts" merges
  `nLocal`, `nlocal`, `ncells`, `nbtotal` into one row even though they
  vary slightly in semantics. The companion per-app reports retain the
  original granular field-level breakdown.
- **The reference column is the *upstream-reference resilient* code in
  `tests/apps/checkpointed/`**, not the *vanilla* code in
  `tests/apps/vanillas/` (which has no checkpoint logic at all).
