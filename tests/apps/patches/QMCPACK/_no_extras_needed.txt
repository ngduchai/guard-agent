# QMCPACK — checkpoint enable is via XML attribute overlay, not a CLI flag.
#
# QMCPACK has no command-line opt-in for checkpointing. The driver input
# parser (src/QMCDrivers/QMCDriverInput.cpp:29 docstring + line 83
# `aAttrib.add(Period4CheckPoint, "checkpoint")`) accepts a `checkpoint`
# attribute on each `<qmc method="...">` block:
#   checkpoint="-1"  -> disabled (default)
#   checkpoint="0"   -> dump at end of every block
#   checkpoint="N"   -> dump every N blocks
# Without the attribute, no `.config.h5` / `.cont.xml` files are written.
#
# The existing overlay at
#   tests/apps/patches/QMCPACK/examples/molecules/He/he_simple_opt.xml
# adds `checkpoint="0"` to the `<qmc method="linear">` block inside the
# 60-iteration `<loop>`, producing roughly one checkpoint per optimization
# block. That overlay is what enables checkpoint_size_bytes accounting for
# the reference flavor.
#
# No additional `_extra_args.txt` is needed; bench `app_args` from
# tests/apps/configs/QMCPACK.yaml (`he_simple_opt.xml`) is correct as-is.
