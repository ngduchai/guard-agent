# SW4lite — checkpoint enable is via input-file overlay, not a CLI flag.
#
# SW4lite has no command-line opt-in for checkpointing. The input-file
# parser (src/EW.C:2258, `else if( startswith("checkpoint", buffer) )`)
# accepts a top-level `checkpoint cycleInterval=N file=PREFIX` directive.
# Without that line, no native restart files are written.
#
# The existing overlay at
#   tests/apps/patches/SW4lite/tests/validation_test.in
# appends:
#   checkpoint cycleInterval=50 file=restart
# which fires every 50 cycles -- well before the framework's 6 s injection
# delay (small grid runs >= 50 cyc/s on 4 ranks). That overlay is what
# enables checkpoint_size_bytes accounting for the reference flavor.
#
# No additional `_extra_args.txt` is needed; bench `app_args` from
# tests/apps/configs/SW4lite.yaml (`tests/validation_test.in`) is correct
# as-is.
