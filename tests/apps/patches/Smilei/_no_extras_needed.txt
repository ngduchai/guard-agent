# Smilei — checkpoint enable is via Python-namelist overlay, not a CLI flag.
#
# Smilei has no command-line opt-in for checkpointing. The Python namelist
# parser (src/Checkpoint/Checkpoint.cpp:58
# `if( PyTools::nComponents("Checkpoints") > 0 )` and line 60
# `PyTools::extract("dump_step", dump_step, "Checkpoints")`) reads a
# `Checkpoints(...)` block from the namelist; the constructor default is
# `dump_step(0)` (disabled — Checkpoint.cpp:48). Without the block, no
# `dump-*.h5` files are written.
#
# The existing overlay at
#   tests/apps/patches/Smilei/namelist_minimal.py
# appends:
#   Checkpoints(
#       dump_step       = 100,
#       dump_minutes    = 0.,
#       exit_after_dump = False,
#       keep_n_dumps    = 2,
#   )
# `exit_after_dump=False` is critical: otherwise the simulation terminates
# after the first dump and the harness loses real elapsed-time signal.
# That overlay is what enables checkpoint_size_bytes accounting for the
# reference flavor.
#
# No additional `_extra_args.txt` is needed; bench `app_args` from
# tests/apps/configs/Smilei.yaml (`namelist_minimal.py`) is correct as-is.
