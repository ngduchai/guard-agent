# ROSS RIO (Restart IO) opt-in flags for the reference checkpointer.
#
# WARNING — BUILD/BINARY MISMATCH:
# These flags are only recognised by the `pholdio` binary, which has RIO
# wired in (USE_RIO compile-time guard). The current ROSS.yaml builds and
# runs `models/phold/phold` (no RIO support — `--io-store` is unknown to
# its tw_opt set, and the program will exit on an unrecognized arg).
# Switching to RIO additionally requires:
#   1. cmake build flag `-DUSE_RIO=ON` (core/CMakeLists.txt:113)
#   2. build target `pholdio` (models/pholdio/CMakeLists.txt)
#   3. executable `./_build/models/pholdio/pholdio`
# Without those changes the args below will cause `phold` to abort. Coordinate
# with the ROSS.yaml build/executable update before relying on this patch.
#
# --io-store=1
#   Registered in models/pholdio/pholdio.c:153 via TWOPT_UINT in `app_opt[]`.
#   Default value (pholdio.h:27) is 2 = "skip RIO" — that's why current runs
#   produce zero checkpoint files. Values:
#     0 = load checkpoint (restart from pholdio_checkpoint*)
#     1 = store checkpoint (write pholdio_checkpoint* during run + at end)
#     2 = skip RIO entirely (default)
#   Setting 1 wires up `g_tw_gvt_hook = pholdio_gvt_hook` (pholdio.c:197),
#   which fires every 500 GVT computations (hardcoded
#   `tw_trigger_gvt_hook_every(500)` at pholdio.c:198). Each fire calls
#   `io_store_checkpoint("pholdio_checkpoint", data_file)` (pholdio.c:41).
#   With the default `--gvt-interval=16` (core/gvt/mpi_allreduce.c:49) this
#   gives roughly one checkpoint per 8000 scheduling-loop iterations — for a
#   30000-end run that lands in the recommended 5-10 checkpoint window
#   (plus a final checkpoint written unconditionally after `tw_run`,
#   pholdio.c:220-226). No CLI knob exists for the 500 cadence.
#
# --io-files=1
#   Registered in core/rio/io-mpi.c:14 via TWOPT_UINT in `io_opts[]`
#   (added to the global tw_opt set at core/tw-setup.c:90 under USE_RIO).
#   Sets `g_io_number_of_files` (core/rio/io-mpi.c:11, default 1) — number
#   of data files the ranks are partitioned across. Pholdio computes
#   `ranks_per_file = tw_nnodes() / g_io_number_of_files` (pholdio.c:39),
#   so this MUST evenly divide the rank count. ROSS.yaml uses 2 MPI ranks,
#   so 1 (both ranks → one file) or 2 (one file per rank) are the only
#   valid choices. Keeping 1 matches the default and keeps the on-disk
#   layout simple.
--io-store=1
--io-files=1
