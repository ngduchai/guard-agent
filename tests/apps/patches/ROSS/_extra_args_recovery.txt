# Recovery-attempt args for ROSS reference bench (attempt_2+).
#
# pholdio (RIO-enabled binary) supports 3 io_store modes:
#   0 = LOAD checkpoint (restart from pholdio_checkpoint.rio-{md,lp,data-N})
#   1 = STORE checkpoint (write during run + at end)
#   2 = SKIP RIO entirely (default)
#
# Primary args (_extra_args.txt) use --io-store=1 to write checkpoints
# during attempt_1.  This file (_extra_args_recovery.txt) flips to
# --io-store=0 so attempt_2 LOADS the checkpoint left by attempt_1
# instead of starting fresh.
#
# Note: --io-files=1 must match attempt_1's setting (the checkpoint
# file partitioning); otherwise pholdio aborts with a partition
# mismatch.  Source: pholdio.c:39 (ranks_per_file = tw_nnodes() /
# g_io_number_of_files).
#
# Recovery args FULLY REPLACE resilient_app_args on attempt_2 (per
# generate_benchmark_configs.py).  Must include workload params + the
# io_store flip.  Workload params match ROSS.yaml app_args exactly so
# attempt_2 sees the same simulation configuration as attempt_1.
--synch=3
--end=30000
--nlp=4000
--extramem=100000
--io-store=0
--io-files=1
