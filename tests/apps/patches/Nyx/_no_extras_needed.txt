# Nyx — checkpoint enable is via AMReX input-file overlay AND already in app_args.
#
# Nyx is AMReX-based. AMReX's checkpoint cadence is controlled by
# `amr.check_int = N` (steps between checkpoints). The default is
# `check_int = -1` i.e. disabled (subprojects/amrex/Src/Amr/AMReX_Amr.cpp:3287);
# `pp.queryAdd("check_int", check_int)` at line 3288 reads it from the
# input file. The output prefix `amr.check_file` defaults to "chk"
# (line 3284) and `amr.checkpoint_files_output` defaults to true
# (line 126), so once `check_int > 0` AMReX writes `chk*` directories.
#
# Two layers of opt-in are already in place for this app:
#   1. Input overlay at
#      tests/apps/patches/Nyx/Exec/HydroTests/inputs.validation
#      appends:
#        amr.checkpoint_files_output = 1
#        amr.check_file              = chk
#        amr.check_int               = 50
#   2. Bench `app_args` from tests/apps/configs/Nyx.yaml `sizes.small`
#      already passes `amr.check_int=50` on the CLI as a final override
#      (AMReX ParmParse honours late `key=val` CLI overrides).
#
# No additional `_extra_args.txt` is needed.
