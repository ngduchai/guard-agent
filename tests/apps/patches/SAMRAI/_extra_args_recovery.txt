# Recovery-attempt args for SAMRAI reference bench (attempt_2+).
# SAMRAI's main.cpp (tests/apps/checkpointed/SAMRAI/source/test/applications/
# LinAdv/main.cpp:198-214) accepts 4-argv invocation:
#   <input_file> <restart_dir> <restore_num>
# (the 4th argv element is added by mpirun as ./linadv).  When invoked
# with the 3-arg form, the application enters restart mode and reads
# state from <restart_dir>/restart.<restore_num>.
#
# attempt_1 writes checkpoints via patches/SAMRAI/validation_inputs/
# linadv.2d.input (`restart_interval = N`, `restart_write_dirname =
# "restart_linadv"`).  The mpirun cwd is shared across attempts (see
# runner.py:_run_one_attempt cwd=output_dir), so restart_linadv/restore.NNN
# from attempt_1 is visible to attempt_2.
#
# {LATEST_RESTORE:restart_linadv} is resolved by the runner at recovery
# time (validation/veloc/runner.py:_resolve_dynamic_recovery_args) to the
# highest restore_NNNNNN integer found on disk.  This works for ANY
# restart_interval value — no hardcoding.  If no restore.NNN exists
# (e.g., restart_interval too high relative to kill time), resolves to 0
# and recovery cold-starts.
validation_inputs/linadv.2d.input
restart_linadv
{LATEST_RESTORE:restart_linadv}
