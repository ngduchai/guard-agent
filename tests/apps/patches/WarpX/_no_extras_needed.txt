# WarpX — checkpoint enable is via AMReX input-file overlay, not a CLI flag.
#
# WarpX is AMReX-based. Its checkpoint mechanism uses the standard AMReX
# diagnostics framework: a diagnostic with `format = checkpoint` and
# `diag_type = Full` is parsed by Source/Diagnostics/MultiDiagnostics.cpp
# (line 67 `pp_diag_name.get("diag_type", diag_type_str)` and line 71
# `diag_type_str == "Full"`). Without a checkpoint-format diagnostic in
# `diagnostics.diags_names`, no checkpoint directories are written.
#
# The existing overlay at
#   tests/apps/patches/WarpX/test_input/inputs_validation
# appends:
#   diagnostics.diags_names = chkpoint
#   chkpoint.intervals      = 500
#   chkpoint.diag_type      = Full
#   chkpoint.format         = checkpoint
# Cadence 500 steps fires well before the 4 s injection delay (vanilla
# 384x384 PIC runs >= 125 steps/s on 4 ranks). That overlay is what
# enables checkpoint_size_bytes accounting for the reference flavor.
#
# No additional `_extra_args.txt` is needed; bench `app_args` from
# tests/apps/configs/WarpX.yaml (`test_input/inputs_validation`) is
# correct as-is.
