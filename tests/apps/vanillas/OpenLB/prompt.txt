Add VeloC checkpoint/restart support to this OpenLB lattice Boltzmann simulation (bstep2d example). The code simulates fluid flow over a backward-facing step using the Lattice Boltzmann method with MPI parallelism.

The main simulation is in examples/laminar/bstep2d/bstep2d.cpp. The time-stepping loop iterates over lattice Boltzmann collision and streaming steps.

Key state to checkpoint:
- The SuperLattice object (sLattice) which contains all distribution functions
- Current iteration number (iT)
- Any converter parameters needed for restart

Note: OpenLB already has a built-in save/load API (sLattice.save/sLattice.load) but we want VeloC-based checkpointing instead.

Requirements:
1. Initialize VeloC after MPI initialization (handled by olbInit)
2. Register the lattice data with VELOC_Mem_protect
3. Add restart logic before the main time-stepping loop
4. Add periodic VeloC checkpointing inside the loop
5. Finalize VeloC before program exit
6. Create veloc.cfg with appropriate scratch/persistent paths
7. Update the Makefile to link against the VeloC client library
