I have an MPI C program that computes accumulated multiplication among various matrix pairs.
Make it resilient against MPI process failures with VeloC checkpoints.
The source code is in the current directory.
We use OpenMPI and VeloC is already installed.

Requirements:
- Identify critical state that must survive process failures.
- Add VeloC checkpoint/restart calls to protect that state.
- Checkpoint once every 10 seconds.
- Save all checkpoints under /tmp (scratch and persistent).
- Each MPI rank checkpoints independently; on restart, each process loads the latest checkpoint.
- Create a veloc.cfg configuration file.
- Update CMakeLists.txt to link against VeloC (find_package(veloc REQUIRED), link veloc-client).
- Do NOT change the application's existing command-line arguments or behavior.
- Modify the source files in place in the current directory.

Check the README.md for build and execution instructions.
