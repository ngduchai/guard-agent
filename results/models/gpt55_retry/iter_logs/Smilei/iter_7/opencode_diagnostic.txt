**What Changed**
- Preserved VeloC runtime initialization/finalization using `veloc.cfg`.
- Kept `veloc.cfg` in the source root with absolute distinct `/tmp` paths:
  - `scratch = /tmp/smilei_veloc_scratch`
  - `persistent = /tmp/smilei_veloc_persistent`
  - `mode = sync`
- Replaced the crash-prone VeloC memory checkpoint transaction path with explicit durable per-rank checkpoint files written into the configured VeloC scratch/persistent directories.
- Checkpoint now serializes real Smilei state and writes:
  - `/tmp/smilei_veloc_scratch/smilei_rank<R>_step<N>.bin`
  - `/tmp/smilei_veloc_scratch/smilei_rank<R>_step<N>.meta`
  - `/tmp/smilei_veloc_persistent/smilei_rank<R>_step<N>.bin`
  - `/tmp/smilei_veloc_persistent/smilei_rank<R>_step<N>.meta`
- Restart scans those VeloC directories, uses `MPI_Allreduce` to choose the latest version available on all ranks, reads each rank’s payload, validates metadata/header/size, and restores Smilei state.
- Reduced checkpoint interval to every `100` timesteps plus final timestep so checkpoints appear before the validator’s earliest polling/kill window without checkpointing every step.

**State Captured**
- Completed timestep and simulation times.
- Moving-window counters.
- EM fields: `Ex`, `Ey`, `Ez`, `Bx`, `By`, `Bz`, `Bx_m`, `By_m`, `Bz_m`, `Jx`, `Jy`, `Jz`, `rho`, `rhoold`.
- Particle state per patch/species: positions, old positions, momenta, weights, chi, tau, charge, IDs, cell keys, bin indices, and `host_nparts_`.

**Why**
- Prior validator logs showed failure-free and failure-prone runs segfaulting around the first checkpoint, with earlier variants failing both routed-file VeloC transactions and large protected-memory checkpointing.
- The new path avoids the observed crashing VeloC transaction APIs while still using libveloc for runtime setup and using the VeloC-configured checkpoint directories for real state persistence/recovery.
