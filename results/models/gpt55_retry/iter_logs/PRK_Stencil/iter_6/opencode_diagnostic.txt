What changed:

- `veloc.cfg` remains at the source root with valid absolute `/tmp` directories:
  - `scratch = /tmp/prkstencil_veloc_scratch`
  - `persistent = /tmp/prkstencil_veloc_persistent`
  - `mode = sync`

- Removed runtime config-copy generation from the binary.
  - The program now locates the validator-provided static config via `veloc.cfg`, `../veloc.cfg`, etc.
  - This avoids stale per-attempt config files and complies with the validator requirement that `veloc.cfg` exist before launch.

- Hardened restart recovery.
  - Added `findLoadableCheckpoint()` to probe VeloC checkpoint versions.
  - All ranks agree on the same candidate version with `MPI_Allreduce`.
  - The code verifies every rank can actually load the routed checkpoint payload before committing to restart.
  - If the newest VeloC metadata points at an incomplete killed checkpoint, it falls back to older versions instead of aborting immediately.

- Prevented partial corrupt restart state.
  - Checkpoint payloads are first read into temporary buffers.
  - `in`, `out`, and `iter` are updated only after all ranks confirm the version loaded successfully.

- Preserved real VeloC persistence.
  - Checkpointing still uses `VELOC_Checkpoint_begin`, `VELOC_Route_file`, and `VELOC_Checkpoint_end`.
  - Restart still uses `VELOC_Restart_test`, `VELOC_Restart_begin`, `VELOC_Route_file`, and `VELOC_Restart_end`.
  - Each rank saves real state: next iteration, `in` array, and `out` array.

I consulted the required previous validator output first. The rejection showed checkpoints were observed, but recovery and failure-free runs exited almost immediately with code `255` and no `validation_output.bin`; these changes target that likely restart-abort path while keeping real checkpoint/load behavior.

I could not run a local build or MPI validation because this environment does not expose a shell execution tool.
