- Kept required static `./veloc.cfg` valid for the validator:
  - `scratch = /tmp/ross_veloc_scratch`
  - `persistent = /tmp/ross_veloc_persistent`
  - `mode = sync`

- Preserved `veloc-runtime.cfg` as the app-preferred VeloC config for runtime compatibility.

- Switched VeloC integration from manual `VELOC_Route_file()` file writes to real VeloC memory-region checkpointing:
  - Registers checkpoint regions with `VELOC_Mem_protect()`.
  - Checkpoints via `VELOC_Checkpoint()`.
  - Restores via `VELOC_Restart()`.

- Switched initialization to `VELOC_Init_single((unsigned int)g_tw_mynode, ...)`.
  - The previous attempt died within ~1 second with empty stderr and no checkpoint files; this avoids the likely collective VeloC client startup failure while still using libveloc for persistence.

- Moved `VELOC_Finalize(0)` before `tw_end()`.
  - `tw_end()` eventually stops/finalizes MPI, so finalizing VeloC afterward can abort or behave incorrectly.

- Checkpoint state still captures and restores actual simulation state:
  - PHOLD LP state.
  - LP RNG/core RNG streams.
  - KP state.
  - GVT/LVT/sequence state.
  - Pending events from both `pe->pq` and `pe->event_q`.

- Removed stale manual checkpoint file read/write helpers and route-file calls.
