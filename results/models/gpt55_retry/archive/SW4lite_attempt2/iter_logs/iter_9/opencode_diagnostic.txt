What changed:

- Restart version selection is now collective:
  - Each rank calls `VELOC_Restart_test`.
  - The app uses `MPI_Allreduce(..., MPI_MIN, ...)` so all ranks restart from the newest checkpoint version that every rank has completed.
  - This addresses the validator failure where output was wrong only after recovery, consistent with different ranks restoring different checkpoint versions after a kill during/near a checkpoint.

- Checkpoint creation is now bracketed with MPI barriers:
  - A barrier before `VELOC_Checkpoint_begin`.
  - A barrier after successful `VELOC_Checkpoint_end`.
  - This keeps completed checkpoint generations aligned across ranks and reduces the chance of advertising a partially rank-complete version.
