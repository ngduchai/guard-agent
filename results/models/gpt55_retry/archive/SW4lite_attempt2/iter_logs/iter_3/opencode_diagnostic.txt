    - synchronizes the device
    - copies current `U` and `Um` from device to host before VeloC writes host memory

Why this fixes the rejection:

- The previous validator failure was disk exhaustion: about `17 GB` of checkpoint files in roughly `12s`.
- The app was checkpointing every timestep and retaining all versions.
- The new config keeps only the latest version in scratch and persistent storage.
- The new cadence still provides restart points before failure injection, but avoids unbounded checkpoint growth.
