  - Reduced checkpoint cadence from every timestep to approximately 20 checkpoints over the full run:
    - Always checkpoints at step `1`.
    - Checkpoints periodically via `mNumberOfTimeSteps / 20`.
    - Always checkpoints at the final timestep.
  - Preserved real VeloC state capture of:
    - `checkpointCycle`
    - current solution array `U`
    - previous solution array `Um`
  - Preserved real restart path using:
    - `VELOC_Restart_test`
    - `VELOC_Restart_begin`
    - `VELOC_Recover_mem`
    - `VELOC_Restart_end`
  - Added CUDA/device correctness handling before checkpoint:
    - synchronizes the device
    - copies current `U` and `Um` from device to host before VeloC writes host memory

- The previous validator failure was disk exhaustion: about `17 GB` of checkpoint files in roughly `12s`.
- The app was checkpointing every timestep and retaining all versions.
- The new config keeps only the latest version in scratch and persistent storage.
