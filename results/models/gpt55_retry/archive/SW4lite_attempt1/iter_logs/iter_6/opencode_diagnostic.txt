What changed:

- Fixed the root memory registration bug:
  - `Sarray::m_npts` already includes all components.
  - The code was registering `m_nc * m_npts`, causing VeloC to read/write 3x beyond each array allocation.
  - This likely caused the prior numeric mismatch and the later `VELOC_Checkpoint` / `VELOC_Restart` failures.
  - All `U`, `Um`, and `Up` registrations now use `m_npts`.

- Kept full timestep state capture:
  - `beginCycle`
  - `t`
  - `U`
  - `Um`
  - `Up`
  - receiver `TimeSeries` buffers via the existing `protectForCheckpoint()` hook

- Avoided the problematic terminal checkpoint:
  - Removed checkpointing at `mNumberOfTimeSteps`.
  - This prevents a final or shutdown-adjacent checkpoint from becoming the latest restart target.

- Adjusted checkpoint cadence:
  - Real checkpoints now happen around 40%, 60%, and 75% of the run.
  - This gives the validator a completed checkpoint before early kill retries and a later checkpoint for faster recovery, without checkpointing at final timestep.

- Confirmed `veloc.cfg` is present and valid:
  - `scratch = /tmp/sw4lite_veloc_scratch`
  - `persistent = /tmp/sw4lite_veloc_persistent`
  - `mode = sync`

I could not run `make`, `mpirun`, or the validator because this environment exposes file/search/edit tools only, not shell execution. Static verification shows the VeloC protected memory sizes now match the actual allocations and checkpoint/restart registrations are consistent.
