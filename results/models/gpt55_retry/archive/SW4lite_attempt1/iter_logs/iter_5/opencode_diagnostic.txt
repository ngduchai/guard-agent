What changed:

- `src/EW.C`
  - Restores a fuller timestep state by checkpointing `Up` in addition to `U`, `Um`, `beginCycle`, and `t`.
  - Registers the same memory IDs before `VELOC_Restart()` and before each `VELOC_Checkpoint()`, so restart layout matches checkpoint layout.
  - Allocates receiver `TimeSeries` buffers before restart registration, so any receiver output state can be restored rather than being recreated empty.
  - Changed checkpoint cadence to one pre-failure checkpoint at about 40% of timesteps plus the final checkpoint. This gives the validator a completed checkpoint before its earliest kill window while avoiding the larger storage cost of multiple 3-array snapshots.

- `src/TimeSeries.h` and `src/TimeSeries.C`
  - Added `TimeSeries::protectForCheckpoint(int baseId)`.
  - Registers `mLastTimeStep`, `mRecordedSol`, and SAC float buffers with VeloC when a receiver is owned by the rank.

- `veloc.cfg`
  - Already existed and satisfies the validator requirement:
    - `scratch = /tmp/sw4lite_veloc_scratch`
    - `persistent = /tmp/sw4lite_veloc_persistent`
    - `mode = sync`

Why:

- The previous validator rejection was not a crash or missing checkpoint issue. It produced checkpoints and recovered, but the recovered numeric output differed from baseline by `1.15e-05`.
- The existing checkpoint saved only `U` and `Um`. The solver also carries `Up`, and after array cycling/checkpointing that buffer contains real timestep/scratch state. Restarting with a zero-initialized `Up` can perturb subsequent computation even if `U` and `Um` are restored.
- Receiver buffers were also not checkpointed; they are now covered for completeness and real state capture.
