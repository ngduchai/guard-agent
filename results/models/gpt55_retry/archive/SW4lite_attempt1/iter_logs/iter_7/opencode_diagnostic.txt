The restart version is now collective: each rank first probes VeloC locally, then all ranks agree on the minimum available checkpoint version before calling `VELOC_Restart`. This prevents mixed-version wavefield restoration across MPI ranks.
Implemented the restart-correctness fix in `src/EW.C`.

What changed:

- Made checkpoint version selection collective across MPI ranks:
  - Each rank calls `VELOC_Restart_test(...)`.
  - Ranks now agree on the minimum available checkpoint version with `MPI_Allreduce(..., MPI_MIN, ...)`.
  - This prevents different MPI ranks from restoring different VeloC checkpoint versions after a kill near an in-flight checkpoint.

- Reduced checkpoint cadence to stable completed snapshots:
  - Kept real wavefield checkpoints at about 40% and 60%.
  - Removed the 75% checkpoint, which could become the latest partially written or rank-skewed checkpoint when the validator kills in the 65-80% range.

- Preserved real state capture:
  - `beginCycle`
  - `t`
  - `U`
  - `Um`
  - `Up`
  - `TimeSeries` buffers via `protectForCheckpoint(...)`

- Confirmed `veloc.cfg` is present and validator-compatible:
  - `scratch = /tmp/sw4lite_veloc_scratch`
  - `persistent = /tmp/sw4lite_veloc_persistent`
  - `mode = sync`

The previous validator failure was failure-prone output mismatch while failure-free output passed. The most likely cause was mixed checkpoint-version recovery across ranks after a checkpoint-near-kill window; the new collective version selection addresses that directly.
