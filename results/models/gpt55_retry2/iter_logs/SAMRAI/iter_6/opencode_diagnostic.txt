The serialized payload still used a two-step selective recovery to read its size. Since the previous failure pattern could be caused by repeated selective recovery inside one restart, I’m simplifying it further to use `VELOC_Recover_size()` and perform exactly one payload recovery.
The checkpoint format is now atomic and recovery does one VeloC payload read per candidate version, with fallback to older versions if the newest is incomplete. I’m making two final safety tweaks: reject oversized payloads that do not fit VeloC’s `int` size API, and avoid zero-length payload registration.
I noticed include ordering is non-standard after the last edit. I’m fixing that while applying the payload size guard.
I found no remaining references to the removed multi-region IDs. The final source pass is checking recovery validation and restore behavior, especially that every restored patch field is matched by checkpointed ID.
Implemented the recovery fix in `source/test/applications/LinAdv/main.cpp`.

What changed:
- Kept required static `veloc.cfg` in the repo root:
  - `scratch = /tmp/samrai_veloc_scratch`
  - `persistent = /tmp/samrai_veloc_persistent`
  - `mode = sync`
- Replaced the brittle multi-region VeloC checkpoint format with one serialized VeloC memory payload per rank.
- The serialized payload captures real application state:
  - integrator time/step/dt state
  - AMR hierarchy shape
  - rank-local patch field IDs and sizes
  - actual `uval` patch field data
- Recovery now:
  - uses `VELOC_Recover_size()` to size the payload
  - performs one selective VeloC payload recovery per candidate version
  - deserializes and validates the checkpoint content
  - rebuilds the hierarchy and restores patch data by checkpointed field ID
- Added fallback across older VeloC versions if the newest advertised version is incomplete due to the validator killing shortly after observing checkpoint files.

Why this addresses the rejection:
- The previous validator failure showed checkpoint files existed, but recovery exited immediately with `VeloC restart test succeeded but LinAdv checkpoint recovery failed`.
- That pointed to restart payload loading, not missing checkpoint creation.
- The new format avoids partially recovering many dynamic memory regions and makes each rank’s checkpoint self-contained and atomic from the application’s perspective.
