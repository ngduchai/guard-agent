Implemented VeloC checkpoint/restart support for the MPI SPPARKS build.

What changed:
- Added static root config file `veloc.cfg` with absolute, distinct `/tmp` scratch/persistent paths:
  - `/tmp/spparks_veloc_scratch`
  - `/tmp/spparks_veloc_persistent`
  - `mode = sync`
- Initialized and finalized VeloC in `src/main.cpp`.
- Linked against `libveloc` and added VeloC include paths in:
  - `src/MAKE/Makefile.mpi`
  - `src/Obj_mpi/Makefile`
- Added generic checkpoint/restart support in `src/app.cpp` / `src/app.h`:
  - Registers simulation time, output scheduling state, checkpoint version, and all local per-site integer/double arrays.
  - Restarts from latest VeloC checkpoint after input/setup has rebuilt topology and allocated arrays.
  - Re-runs setup after recovery so communication, solver, and propensity state are rebuilt from restored site state.
  - Uses a persisted monotonically increasing checkpoint version counter.
- Added lattice-specific state capture in `src/app_lattice.cpp` / `src/app_lattice.h`:
  - Registers `naccept`, `nattempt`, `nsweeps`, `ranapp` seed, `ranstrict` seed, and strict per-site seeds.
  - Takes real mid-execution checkpoints at safe lattice timestep/event boundaries in KMC, sector KMC, rejection KMC, and app-update-only loops.
