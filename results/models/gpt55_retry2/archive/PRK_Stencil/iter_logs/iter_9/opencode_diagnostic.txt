What changed:

- `Stencil/stencil.c`
  - Delays VeloC initialization until after normal MPI startup, CLI validation, and broadcasting `n`/`iterations`.
  - Switches VeloC startup to `VELOC_Init_single((unsigned int)rank, cfg_path)` to avoid the collective VeloC init path that was causing immediate MPI process exits before any app output/checkpoint.
  - Keeps real libveloc persistence through:
    - `VELOC_Checkpoint_begin`
    - `VELOC_Route_file`
    - `VELOC_Checkpoint_end`
    - `VELOC_Restart_test`
    - `VELOC_Restart_begin`
    - `VELOC_Restart_end`
    - `VELOC_Finalize`
  - Persists actual per-rank state:
    - `completed_iter`
    - full `in` grid, including ghost cells
    - full accumulated `out` grid
  - Restores that state on restart and resumes from `completed_iter + 1`.
  - Fixes restart probing to use `VELOC_Restart_test("stencil", iterations)` so a completed final checkpoint from the same workload is not selected as a restart point.
  - Ensures routed checkpoint files are closed even when partial read/write failures occur.

- `veloc.cfg`
  - Confirmed present as a static source-root file with validator-compatible absolute `/tmp` paths:
    ```text
    scratch = /tmp/prkstencil_veloc_scratch
    persistent = /tmp/prkstencil_veloc_persistent
    mode = sync
    ```

The previous validator failure was an immediate resilient binary exit with no checkpoints and no `validation_output.bin`. These changes target that startup path while preserving real VeloC-backed checkpoint and restart state capture. I could not run a local build or MPI validation because this environment does not provide a shell execution tool.
