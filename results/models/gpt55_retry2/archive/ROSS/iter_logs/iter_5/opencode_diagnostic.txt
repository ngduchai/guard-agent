- `core/tw-veloc.c`
  - Uses `VELOC_Init_single((unsigned int)g_tw_mynode, cfg)` instead of collective `VELOC_Init(...)` to avoid the prior immediate MPI/VeloC early-collapse path.
  - Keeps real VeloC persistence via `VELOC_Mem_protect`, `VELOC_Checkpoint`, `VELOC_Restart_begin`, `VELOC_Recover_mem`, and `VELOC_Restart_end`.
  - Makes `scratch`/`persistent` directory parsing robust to normal whitespace and creates the configured `/tmp` directories before VeloC init.
  - Captures/restores additional PE/GVT transient state:
    - `seq_num`
    - `s_nwhite_sent`
    - `s_nwhite_recv`
    - `gvt_status`
  - Continues capturing/restoring:
    - LP model state
    - LP RNG and core RNG streams
    - LP suspend/critical-path fields
    - KP GVT/rollback counters
    - PE GVT timestamps/signatures
    - pending event queue records and message payloads
  - Clears regenerated startup events on restart and restores checkpointed pending events, resetting event linkage before enqueueing.

- `models/phold/phold.main.c`
  - VeloC initialization now happens after LP definitions/type setup and before `tw_run()`.
  - Restart remains in the existing correct engine hook: `tw_sched_init()` calls `tw_veloc_restart_after_lps_init()` after LP state allocation and PHOLD initial event generation.
  - VeloC finalization remains before `tw_end()` so `VELOC_Finalize()` runs before MPI shutdown.
