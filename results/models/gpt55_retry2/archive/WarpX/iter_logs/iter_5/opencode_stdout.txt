**Root Cause**
The validator saw checkpoint files and correct output, but recovery was effectively a full rerun:

- `recovery/Z_P=1.011`
- `recovery_actually_resumed=False`
- `ratio=1.85x`

The existing restart code required every rank’s latest VeloC checkpoint version to be identical. After a SIGKILL, surviving ranks can have newer checkpoint metadata than the killed rank. That made recovery reject restart and fall back to a cold run.

**Change Made**
Updated `Source/Evolve/WarpXEvolve.cpp` in `WarpX::RestartFromVeloCCheckpoint()`:

- Replaced strict `min_version == max_version` latest-version check.
- Added collective negotiation for the newest checkpoint version available on every rank.
- Recovery now probes VeloC with `VELOC_Restart_test()` and chooses a common durable version before calling:
  - `VELOC_Restart_begin`
  - `VELOC_Recover_size`
  - `VELOC_Mem_protect`
  - `VELOC_Recover_mem`
  - `VELOC_Restart_end`
  - `ApplyVeloCState`

This preserves real VeloC persistence and real loaded state restoration, while avoiding the cold-rerun path caused by rank-version skew.
