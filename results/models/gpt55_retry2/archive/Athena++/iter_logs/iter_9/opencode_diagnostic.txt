Changed:

- `src/main.cpp`
  - Replaced direct VeloC protection of `MeshBlock::phydro->u` arrays with an owned `std::vector<Real>` checkpoint buffer.
  - Before each checkpoint, packs the live evolved hydro conserved state from current AMR blocks into that stable buffer.
  - On restart, recovers into the buffer first, then unpacks into rebuilt/current mesh blocks.
  - Keeps checkpointing real simulation state via VeloC, but avoids VeloC retaining stale pointers to AMR-deleted `MeshBlock` memory.
  - Preserves AMR topology metadata and now gathers per-rank block counts with the global recovered layout.

- `src/mesh/mesh.hpp`
  - Updated `Mesh::ResetForRestartLayout(...)` to accept saved per-rank block counts.

- `src/mesh/mesh.cpp`
  - Updated `ResetForRestartLayout(...)` to preserve the checkpoint’s rank ownership instead of recomputing load balance.
  - This makes recovered per-rank hydro buffers unpack into the same local block ordering that wrote them.

- `veloc.cfg`
  - Confirmed static root config exists with required absolute `/tmp` paths:
    - `scratch = /tmp/athena_veloc_scratch`
    - `persistent = /tmp/athena_veloc_persistent`
    - `mode = sync`

Why this addresses the previous failure:

- The validator showed failure-free runs segfaulting with exit `139` around 23s, before the kill point.
- The prior integration repeatedly registered VeloC memory regions pointing directly into AMR-owned `MeshBlock` arrays. AMR deletes/replaces those blocks during the run, which can leave VeloC with invalid/stale region pointers.
- VeloC now protects stable owned buffers whose lifetime spans the run, while checkpoints still contain the real evolved hydro state and topology needed for restart.
- AMR recovery now preserves saved rank distribution, avoiding mismatched unpacking after restart.
