## Summary of changes

### 1. `veloc.cfg` (new file, source tree root)

**What:** Created `veloc.cfg` with the required `scratch`, `persistent`, and `mode` keys using distinct absolute `/tmp` paths.

**Why:** The validator parses this file before launching `mpirun` to identify checkpoint directories. Without it, the validator fatals immediately.

**Result:** `scratch = /tmp/prkstencil_veloc_scratch`, `persistent = /tmp/prkstencil_veloc_persistent`, `mode = sync`.

### 2. `Stencil/Makefile`

**What:** Added `-I/home/ndhai/usr/include` to `INCLUDEPATHS`, and `-lveloc-client` / `-L/home/ndhai/usr/lib -Wl,-rpath,/home/ndhai/usr/lib` to `LIBS`/`LIBPATHS`.

**Why:** The compiler needs to find `veloc.h` and the linker needs to link against `libveloc-client.so`.

### 3. `Stencil/stencil.c`

**What:** Added full VeloC checkpoint/restart support:

- `#include <veloc.h>` and `#include <limits.h>` added.
- **`VELOC_Init(MPI_COMM_WORLD, "../veloc.cfg")`** called right after `MPI_Init`. The CWD when binary runs is `Stencil/`, so `../veloc.cfg` points to the source tree root.
- **Memory registration** with `VELOC_Mem_protect` for: `in[]` (id=1), `out[]` (id=2), `start_iter` int (id=3). Done after allocation but before potential restart.
- **Restart detection** via `VELOC_Restart_test(name, INT_MAX)` — `INT_MAX` means "find the latest available checkpoint" (the API returns the latest version strictly less than the given `needed_version`).
- **Restart logic**: if a checkpoint is found, call `VELOC_Restart` to restore all three memory regions, then set `start_iter = saved_iter + 1` to resume at the next iteration.
- **Conditional initialization**: arrays are initialized from scratch only when no checkpoint exists or restart fails.
- **Loop** starts at `start_iter` instead of 0.
- **Periodic checkpointing**: every `CKPT_FREQ=5` iterations (and only for `iter > 0` to skip the warmup), saves `in[]`, `out[]`, and `start_iter=iter` via `VELOC_Checkpoint(name, iter)`.
- **Timer fix**: if restarting at `start_iter > 1`, the timer starts immediately (not waiting for `iter==1` which would never come), to avoid undefined behavior with uninitialized `local_stencil_time`.
- **`VELOC_Finalize(0)`** called before `MPI_Finalize` (cleanup=0 preserves checkpoints for inspection).
