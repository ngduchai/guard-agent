## Changes Made

### Root cause analysis from iter_1 logs:

1. **Exit code 255 (VELOC_Init failure)**: The validator deletes the VeloC directories before the failure-free run. Since our binary never created them, VELOC_Init failed for the failure-free run and attempt 1 (first-ever run where dirs didn't exist yet).

2. **22 GB disk bloat**: We were checkpointing both `in[]` AND `out[]` without `max_versions`, creating 44 checkpoint versions × ~512 MB (in + out, scratch + persistent) = 22 GB.

3. **Wrong config path for failure-free run**: `"../veloc.cfg"` doesn't work when the validator runs with `cwd=resilient_clean/` (the veloc.cfg is at `resilient_clean/veloc.cfg`, not one level up).

### Fixes:

**`Stencil/stencil.c`**:
- Added `ensure_veloc_dirs()` helper that creates `/tmp/prkstencil_veloc_scratch` and `/tmp/prkstencil_veloc_persistent` on rank 0 before `VELOC_Init`, ensuring the directories always exist regardless of prior validator cleanup.
- Changed config path resolution to `stat("../veloc.cfg")` first; if that doesn't exist, fall back to `"veloc.cfg"`. This covers all validator-run scenarios (kill attempts use `../veloc.cfg`, failure-free uses `veloc.cfg`).
- Removed `in[]` from checkpointed memory (only checkpoint `out[]` + `start_iter`). Since `IN(i,j) = COEFX*i + COEFY*j + iter + 1` after `iter` iterations, it can be analytically reconstructed — saving ~50% checkpoint I/O.
- On restart, reconstruct `in[]` from the formula `COEFX*i + COEFY*j + (saved_iter + 1)` for all cells including ghost zones. Ghost zones are overwritten by MPI halo exchange at the start of the resumed iteration anyway.

**`veloc.cfg`**:
- Added `max_versions = 1` and `scratch_versions = 1` to keep only the most recent checkpoint, preventing disk accumulation (from 22 GB down to ~128 MB at any time).
