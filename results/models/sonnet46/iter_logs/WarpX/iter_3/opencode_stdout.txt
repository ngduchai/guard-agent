## Changes Made

### `Source/Evolve/WarpXEvolve.cpp`

1. **Fixed MPI deadlock in restart (primary bug)**: `VELOC_Restart_test` is a local (non-collective) function, so different ranks could return different checkpoint versions. If rank 0 finds version 46000 but rank 1 finds nothing (VELOC_FAILURE=-1), rank 0 would call the collective `VELOC_Restart_begin` alone, deadlocking. Fixed by:
   - Using `MPI_Allreduce(..., MPI_MIN)` to agree on the minimum version available across all ranks
   - Only entering the restart block if ALL ranks have that version
   - Always calling `VELOC_Restart_end` after `VELOC_Restart_begin` (even on failure), passing the collective success/failure flag

2. **Fixed checkpoint frequency** (root cause of 900s timeout): Changed from checkpointing every step to every 1000 steps (`VELOC_CKPT_INTERVAL = 1000`). With 80000 steps at ~1ms/step, checkpointing every step caused ~1 disk write per millisecond, making the full run take hundreds of seconds instead of ~80s.

3. **Made `veloc_ok` flag consistent across ranks**: Used `MPI_Allreduce(MPI_MIN)` after `VELOC_Init` to ensure all ranks agree on whether VeloC initialized successfully. This prevents some ranks entering checkpoint blocks while others skip them.

### `veloc.cfg`

Added `max_versions = 2` and `scratch_versions = 2` to cap disk usage — VeloC now retains only the 2 most recent checkpoint versions in both scratch and persistent directories, preventing disk accumulation across many checkpoints.
