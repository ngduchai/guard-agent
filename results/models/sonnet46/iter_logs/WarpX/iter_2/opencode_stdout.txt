## Root Cause

The previous attempt (iter_1) correctly implemented checkpoint/restart using VeloC, but had a critical bug in how checkpoint/restart sessions were terminated:

- `VELOC_Checkpoint_end(int success)` - the `success` parameter is `1` for success, `0` for failure
- `VELOC_Restart_end(int success)` - same convention

The code was incorrectly passing `VELOC_SUCCESS` (which equals `0`) as the success flag, telling VeloC that the checkpoint **failed**. This caused VeloC to not commit the checkpoint as valid, so when attempt 2 (restart) ran, it either:
1. Found an invalidated checkpoint and tried to restore from corrupt state → SIGABRT (exit 6)
2. Or VeloC's internal state was inconsistent due to the failed checkpoint marker

## Fix Applied

Changed two lines in `Source/Evolve/WarpXEvolve.cpp`:

1. `VELOC_Checkpoint_end(VELOC_SUCCESS)` → `VELOC_Checkpoint_end(1 /* success */)`
2. `VELOC_Restart_end(VELOC_SUCCESS)` → `VELOC_Restart_end(1 /* success */)`
