The previous attempt failed because `VELOC_Init(MPI_COMM_WORLD, "../../../veloc.cfg")` used a hardcoded relative path assuming the binary ran from `examples/laminar/bstep2d/`. In reality, the validator runs the binary from a different CWD (a validation output directory), so the path was wrong, VeloC init failed, `MPI_Abort` was called, and mpirun exited with code 255 — within 3 seconds, before any simulation or checkpointing happened.

## Fix Applied

1. `./veloc.cfg` — works for failure-free runs (validator copies cfg to the run CWD)
2. `../veloc.cfg` — works for kill attempts (validator copies cfg to parent of `attempt_N/` subdir)
3. `../../veloc.cfg` — extra fallback
4. `../../../veloc.cfg` — source tree root fallback

Also added `#include <unistd.h>` for the `access()` system call.
