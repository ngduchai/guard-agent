The `stage::Full` communicator is only called from `SuperLattice::communicate()` which is called from `setProcessingContext(Evaluation)`. This is for ensuring halos are up-to-date before output/evaluation.

The per-iteration communication in `collideAndStream()` happens through `executePostProcessors(PostCollide())` and `executePostProcessors(PostStream())` which run their own communicators.

So my concern about `_communicationNeeded = false` after `communicate()` is not a problem for the collide/stream communication - that's handled by separate communicators. The `Full` stage communication is only needed before evaluation (output).

The fix is sound. Let me also check whether there's any issue with `setProcessingContext(Simulation)` being called after `communicate()`:

After `sLattice.communicate()`:
- `_communicationNeeded = false`
- Halos are synchronized with current interior values

After `sLattice.setProcessingContext(ProcessingContext::Simulation)`:
- Blocks' processing context set to Simulation (no communication here)

When the first iteration starts with `sLattice.collideAndStream()`:
- Checks `_initialized` - yes, set by `setInitialValues()`

This is all correct.

One more thing to verify: the `sLattice.communicate()` after loading the checkpoint. Since `_communicationNeeded` was set to `true` by `initialize()` and hasn't been reset, this will work. But what if `_communicationNeeded` is somehow `false` when we get to the restart path? That shouldn't happen since `initialize()` always sets it to `true`.

The fix is complete. Let me provide a summary:

**Root cause**: When recovering from a checkpoint, the `LatticeStatistics::latticeTime` counter was not restored. It started at 0 and only accumulated the remaining iterations after restart. This caused `sig_buf[0] = stats.getTime()` in the validation output to be `totalSteps - iT_saved` instead of `totalSteps`, leading to a huge numerical difference (`max_abs_diff = 3.815e+05`).

**Fix**: Added `sLattice.getStatistics().resetTime(iT_saved)` immediately after loading the checkpoint, which sets the statistics time counter to `iT_saved`. After running the remaining `totalSteps - iT_saved` iterations, the counter becomes `iT_saved + (totalSteps - iT_saved) = totalSteps`, matching the baseline.

Also added `sLattice.communicate()` to ensure halo regions are properly synchronized after loading the checkpoint state.
