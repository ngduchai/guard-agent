**2. Modified `config.git.mk`** (the template that gets `cp`'d to `config.mk` by the build command):
- Added `-I/home/ndhai/usr/include` to `CXXFLAGS` so `veloc.h` is found

**3. Modified `bstep2d.cpp`** to add full checkpoint/restart support:
- `VELOC_Init(MPI_COMM_WORLD, "../../../veloc.cfg")` after `olb::initialize()` (binary runs from `examples/laminar/bstep2d/`, so `../../..` reaches the source root)
- Computes per-rank lattice serialization size with `sLattice.getSerializableSize()` and allocates a matching buffer
- Registers the buffer (id=1) and iteration counter `iT_saved` (id=2) with `VELOC_Mem_protect`
- On startup, checks for existing checkpoint with `VELOC_Restart_test("bstep2d", INT_MAX)` and restores state using `VELOC_Restart` + `sLattice.load(buffer)` if found
- Every 500 iterations: serializes lattice to buffer with `sLattice.save(buffer)`, updates `iT_saved`, calls `VELOC_Checkpoint`
- `VELOC_Finalize(0)` before `return 0`

### Key design decisions

- **Buffer-based serialization**: Uses OpenLB's built-in `Serializable::save(uint8_t*)` / `load(const uint8_t*)` API to serialize the lattice state to a flat buffer, which is then protected by VeloC
- **Per-rank data**: Each MPI rank has its own buffer sized to its local lattice blocks; VeloC collective mode handles per-rank checkpoint files transparently
- **Step tracking**: `iT_saved` records the next step to resume from; after restart, the loop starts at `iT_saved` instead of 0, so the final physics state matches the baseline run
