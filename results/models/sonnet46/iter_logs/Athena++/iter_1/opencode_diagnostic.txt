### 1. `veloc.cfg` (new file, created at project root)
```
scratch = /tmp/athena_veloc_scratch
persistent = /tmp/athena_veloc_persistent

```
**Why:** Required by the validator before `mpirun` is launched. Must be a static file with two distinct absolute `/tmp` paths. `mode = sync` ensures checkpoints complete synchronously before the next cycle.

### 2. `Makefile` (modified)
- Added `-I/home/ndhai/usr/include` to `CPPFLAGS`
- Added `-L/home/ndhai/usr/lib -Wl,-rpath,/home/ndhai/usr/lib` to `LDFLAGS`
- Added `-lveloc-client` to `LDLIBS`

**Why:** VeloC headers are at `/home/ndhai/usr/include/veloc.h` and the library is at `/home/ndhai/usr/lib/libveloc-client.so`. The `rpath` ensures the dynamic linker finds the library at runtime.

### 3. `src/main.cpp` (modified)

Four insertions:

1. **Include `<climits>` and `<veloc.h>`** at the top — needed for `INT_MAX` and the VeloC API.

2. **VeloC Init** right after MPI init (`VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")`) — must happen after MPI so VeloC can use the communicator for collective operations.

3. **Memory registration + restart check** after `pmesh->Initialize(0, pinput)` — registers a 3-element scalar buffer (time, dt, ncycle) as ID 0, and each MeshBlock's `phydro->u` array as ID `100 + lid`. Then calls `VELOC_Restart_test("athena_ckpt", INT_MAX)` to find the latest available checkpoint; if found, calls `VELOC_Restart` to restore state and writes it back into `pmesh->time`, `pmesh->dt`, and `pmesh->ncycle`.

4. **Checkpoint inside main loop** after `MakeOutputs` — populates the scalar buffer with current state then calls `VELOC_Checkpoint("athena_ckpt", pmesh->ncycle)` every cycle. The version number is monotonically increasing since `ncycle` strictly increases.

5. **VeloC Finalize** before `MPI_Finalize()` — `VELOC_Finalize(0)` with `cleanup=0` keeps checkpoint files on disk for subsequent restarts.
