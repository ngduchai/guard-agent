### 4. `src/main.cpp` (modified)
Added full VeloC checkpoint/restart support:

1. **Headers**: Added `#include "veloc.h"`, `<string>`, `<climits>`, `<unistd.h>`, `<libgen.h>`

2. **`VELOC_Init`**: Called after `HPCG_Init`, with auto-detection of the veloc.cfg path (checks `../veloc.cfg` relative to binary directory via `/proc/self/exe`, falls back to CWD)

3. **Memory protection**: Registered two regions before the main CG loop:
   - ID 1: `veloc_loop_counter` (int) — tracks how many CG sets completed
   - ID 2: `testnorms_data.values` array — accumulated residual norms

4. **Restart detection**: Uses `VELOC_Restart_test("hpcg_cg", INT_MAX)` to find the latest checkpoint, then calls `VELOC_Restart()` to restore state

5. **Checkpointing**: After each CG set completes, calls `VELOC_Checkpoint("hpcg_cg", veloc_loop_counter)` with the new count as the version

6. **Cleanup**: Calls `VELOC_Mem_unprotect` and `VELOC_Finalize(0)` before cleanup

**Recovery semantics**: On restart, `veloc_loop_counter` is restored to the last completed iteration count, and the CG loop starts from `i = veloc_loop_counter` — skipping already-completed sets but correctly populating `testnorms_data.values[0..counter-1]` from checkpoint. This produces the same final `validation_output.bin` as a clean run.
