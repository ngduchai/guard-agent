**2. Modified `src/EW.C`** — the `timesteploop()` function:

- **Include**: Added `#include <veloc.h>` at the top of the file.

- **Initialization** (after arrays allocated and zeroed, before time loop):
  - `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` — initializes VeloC using the collective MPI communicator
  - `VELOC_Mem_protect(0, veloc_scalar, 2, sizeof(double))` — registers a 2-element double array holding `{t, beginCycle}` (scalar state)
  - `VELOC_Mem_protect(1+g, U[g].c_ptr(), U[g].m_npts, sizeof(float_sw4))` for each grid g — registers U arrays
  - `VELOC_Mem_protect(1+mNumberOfGrids+g, Um[g].c_ptr(), Um[g].m_npts, sizeof(float_sw4))` for each grid g — registers Um arrays

- **Restart check** (before the time loop):
  - `VELOC_Restart_test("sw4ckpt", 0)` — tests for existing checkpoints
  - If found: `VELOC_Restart("sw4ckpt", version)` restores all registered memory, then copies `veloc_scalar` back into `t` and `beginCycle` so the loop resumes from where it left off

- **Checkpoint per step** (at the end of each time step, after `cycleSolutionArrays`):
  - Updates `veloc_scalar` with current `t` and `currentTimeStep+1` (the next step to execute on restart)
  - `VELOC_Checkpoint("sw4ckpt", currentTimeStep)` — writes all registered memory to persistent storage

- **Finalization** (after the time loop):
  - `VELOC_Finalize(0)` — cleans up VeloC (keeps checkpoints)

**3. Modified `Makefile`**: Added VeloC include path to `CXXFLAGS` and `-lveloc-client` with rpath to `linklibs`.

### Why this is correct

- The checkpoint captures the complete wave simulation state: `U[g]` (solution at t_n) and `Um[g]` (solution at t_{n-1}), plus the scalar time `t` and loop counter `beginCycle`. These are exactly the variables the time loop needs to resume correctly.
- `cycleSolutionArrays` rotates `Um←U, U←Up`, so immediately after it, `U` holds the newest solution. Checkpointing at this point captures a consistent state.
- On restart, `VELOC_Restart` overwrites the zeroed U/Um arrays with the saved data, and `beginCycle` is restored so `for (currentTimeStep = beginCycle; ...)` resumes at the right iteration.
- `veloc.cfg` is a static file in the source tree, parsed by the validator before mpirun, satisfying the infrastructure requirement.
