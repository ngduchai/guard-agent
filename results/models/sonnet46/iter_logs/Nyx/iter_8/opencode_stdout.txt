### `Source/Driver/nyx_main.cpp` (complete rewrite)

**Problem from iter_7**: Build error - `FillCoarsePatch` was incorrectly called as a static method with `amrex::AmrLevel::FillCoarsePatch(*amr_levels[lev], S_new, ...)`. It's an instance method.

**Key improvements**:

1. **Fixed `FillCoarsePatch` call**: Now called as `amr_levels[lev]->FillCoarsePatch(S_new, ...)` (instance method).

2. **Two-slot atomic checkpoint**: 
   - Data is written to slot 0 (`nyx_ckpt_v0/`) or slot 1 (`nyx_ckpt_v1/`) alternately
   - A tiny indicator file (`nyx_ckpt_cur`) is atomically updated via `rename()` to point to the current slot
   - `rename()` on a regular file is POSIX-guaranteed atomic
   - Recovery simply reads the indicator file and loads the corresponding slot

3. **Per-step checkpointing**: Checkpoints every simulation step instead of every 5 steps. This minimizes the amount of work needed after recovery while maintaining acceptable overhead (checkpoint I/O for small grids is fast).

4. **Correct recovery math**: With per-step checkpointing:
   - `ratio = 1 + T_ckpt/T_step` (independent of kill fraction)
   - For fast VisMF I/O (T_ckpt << T_step): ratio ≈ 1.1-1.2x → passes at 1.7x gate

5. **Simplified code**: Removed complex `findLatestCompleteCheckpointStep()` scan logic that was fragile. Recovery now just opens `nyx_ckpt_cur` and reads one byte.
