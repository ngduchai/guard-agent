## Summary of Changes

**Problem**: The previous attempt built successfully and wrote checkpoints, but the recovery (attempt 2) crashed with:
```
amrex::Error::get_comoving_a: invalid time !!!
```

**Root Cause**: Nyx is a cosmological code that tracks the cosmological scale factor `a` via four static class variables: `old_a`, `new_a`, `old_a_time`, `new_a_time`. The `get_comoving_a(time)` function requires that `time` lies within `[old_a_time, new_a_time]`. 

On restart:
1. `amrptr->init(0, stop_time)` initializes the simulation from scratch → sets `old_a_time = new_a_time = 0` (or small initial_dt after `computeInitialDt`)
2. `readVelocCheckpoint` restores `cumtime` to the checkpointed time (e.g., ~80s) and restores level time states
3. The main loop calls `coarseTimeStep(cumtime)` → internally calls `get_comoving_a(cumtime)`
4. `cumtime` (~80s) is not in the range `[0, small_initial_dt]` → CRASH

**Fix**: 
1. **In `writeVelocCheckpoint`**: Save `Nyx::old_a`, `Nyx::new_a`, `Nyx::old_a_time`, `Nyx::new_a_time` to the checkpoint header file
2. **In `readVelocCheckpoint`**: Restore these four values immediately after reading/broadcasting the header, BEFORE any AMR operations that might call `get_comoving_a`

This ensures that after restart, `get_comoving_a(cumtime)` correctly returns `new_a` (since `new_a_time = cumtime`).
The fix is complete. The key changes to `Source/Driver/nyx_main.cpp`:

1. **`writeVelocCheckpoint`** (line ~271-278): Added saving of `Nyx::old_a`, `Nyx::new_a`, `Nyx::old_a_time`, `Nyx::new_a_time` to the `Header.txt` file.

2. **`readVelocCheckpoint`** (lines ~356-460): 
   - Added variables to hold the saved comoving_a state
   - Read them from `Header.txt` (gracefully handling old-style checkpoints without these values)
   - Broadcast them to all MPI ranks
   - **Critically**, restore them BEFORE any AMR operation (regrid, setTimeLevel, etc.) that might call `get_comoving_a(time)` — placing the restoration at the very start of the AMR state restoration sequence
