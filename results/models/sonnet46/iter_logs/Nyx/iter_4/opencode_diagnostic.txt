## Summary

**Root cause of iter_3 failure**: The recovery run crashed with `get_comoving_a: invalid time` (exit code 6) after only 3.15 seconds.

**Analysis**: After `amrptr->init(0, stop_time)`, AMReX creates fine levels at t=0 with `curTime=0` and `prevTime=-dt_init`. When our restart code then sets comoving_a to checkpoint values (`old_a_time = T - dt_N`, `new_a_time = T`) and calls `amrptr->regrid(0, T)`, AMReX internally calls `Nyx::init(old_fine_level)` which reads `cur_time=0` and `prev_time=-dt_init` from the stale old fine levels. `Nyx::init(old)` then calls `vol_weight_sum("rho_E", prev_time=-dt_init, ...)` and `reset_internal_energy_interp(...)` which both invoke `get_comoving_a(-dt_init)`. This fails because `-dt_init` is far outside the checkpoint's valid comoving_a interval `[T - dt_N, T]`.

**Fix** (`Source/Driver/nyx_main.cpp`): Added **Step 2b** between the level-0 time fix and the regrid call. This step sets ALL existing fine levels' times to `ckpt_cumtime` using `setTimeLevel(ckpt_cumtime, lev0_dt, lev0_dt)` BEFORE calling regrid. With this fix, when `Nyx::init(old_fine_level)` runs during regrid, it reads `cur_time = ckpt_cumtime` and `prev_time = ckpt_cumtime - lev0_dt` — both within the valid comoving_a range — and `get_comoving_a` succeeds.
