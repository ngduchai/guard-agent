## Summary

**Root cause of iter_3 failure**: The recovery run crashed with `get_comoving_a: invalid time` (exit code 6) after only 3.15 seconds.

**Analysis**: After `amrptr->init(0, stop_time)`, AMReX creates fine levels at t=0 with `curTime=0` and `prevTime=-dt_init`. When our restart code then sets comoving_a to checkpoint values (`old_a_time = T - dt_N`, `new_a_time = T`) and calls `amrptr->regrid(0, T)`, AMReX internally calls `Nyx::init(old_fine_level)` which reads `cur_time=0` and `prev_time=-dt_init` from the stale old fine levels. `Nyx::init(old)` then calls `vol_weight_sum("rho_E", prev_time=-dt_init, ...)` and `reset_internal_energy_interp(...)` which both invoke `get_comoving_a(-dt_init)`.
