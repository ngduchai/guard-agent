The critical thing is that `VELOC_Restart_begin` and `VELOC_Restart_end` are collective. Between them, each rank independently reads its own checkpoint file. This is correct. ✓

However, there's one subtle timing issue: if rank 3's checkpoint file is larger than rank 0's (because rank 3 has more cells), rank 3 will take longer to read. Between `VELOC_Restart_end` (where ranks synchronize) and `MPI_Allreduce` (next collective), all ranks must have completed reading. Since `VELOC_Restart_end` is collective and happens AFTER each rank has read its file, this is fine. ✓
