## Summary

**Root Cause**: The previous implementation correctly checkpointed the spin state (`iarray[0]`) and simulation time (`time`), but **did not save the RNG state** (`ranapp->seed`). 

After a kill+restart, the `ranapp` random number generator was re-initialized with a fresh seed (from `ranmaster->uniform()` in `init()`). This caused the simulation to make completely different random decisions after recovery compared to what would have happened in an uninterrupted run, resulting in a diverging spin state and incorrect output.

**Fix**: Added the `ranapp->seed` to the VeloC checkpoint:

1. Added `VELOC_ID_RANAPP_SEED` (ID=5) for the RNG seed
2. Added a static buffer `veloc_ranapp_seed` to hold the seed value stably (VeloC needs a stable pointer)
3. In `veloc_protect()`: Register `veloc_ranapp_seed` with VeloC
4. In `veloc_checkpoint()`: Sync `ranapp->seed` → `veloc_ranapp_seed` before checkpointing
5. In `veloc_restart()`: After restore, copy `veloc_ranapp_seed` → `ranapp->seed`

The Park-Miller RNG used by SPPARKS has its entire state in a single `int seed`, so restoring `seed` is sufficient for full RNG state recovery. After recovery, the simulation continues generating exactly the same random sequence it would have produced without interruption, yielding bit-for-bit identical output.
