## Summary of Changes

The previous iteration failed because VeloC's default behavior is to **keep all checkpoint versions** (`max_versions=0` and `scratch_versions=0` default = keep everything). This caused:

1. **Kill attempt 1**: 10.84 GB of checkpoint files caused disk-guard to kill the first run before the failure injector could even fire (no checkpoint detected, exit=1).
2. **Kill attempt 2**: Despite detecting 36,488 checkpoint files, the **recovery run** was immediately killed by disk-guard (11.41 GB still on disk from the crash run) after only 11.91 seconds.

### Fixes Applied

1. **`veloc.cfg`** — Added `max_versions = 1` and `scratch_versions = 1`:
   - Limits VeloC to keeping only 1 checkpoint version on both persistent and scratch storage
   - With 4 MPI ranks and ~62,500 sites per rank (250 KB/rank), disk usage is bounded to ~2-4 MB instead of 11+ GB

2. **`src/app_lattice.cpp`** — Increased `VELOC_CKPT_INTERVAL` from 5 to 50 sweeps:
   - Reduces I/O frequency while still providing good recovery granularity
   - With 18,800 total sweeps and kill at ~64%, ~237 checkpoints will exist before kill; only the latest is retained
