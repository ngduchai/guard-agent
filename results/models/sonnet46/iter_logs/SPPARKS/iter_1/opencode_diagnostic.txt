### Solution: VeloC Integration

**1. `veloc.cfg` (new file, project root)**
- Required by the validator to know checkpoint directories before launching the binary
- Uses distinct absolute `/tmp` paths for `scratch` and `persistent`
- `mode = sync` ensures checkpoints are synchronously written

**2. `src/MAKE/Makefile.mpi` + `src/Obj_mpi/Makefile`**
- Added `-I/home/ndhai/usr/include` to `CCFLAGS` for VeloC headers
- Added `-L/home/ndhai/usr/lib -lveloc-client -Wl,-rpath,/home/ndhai/usr/lib` to `LIB`

**3. `src/main.cpp`**
- Added `#include "veloc.h"` and global `int veloc_enabled`
- Calls `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` right after `MPI_Init`
- Calls `VELOC_Finalize(0)` before `MPI_Finalize`

**4. `src/app_lattice.h`**
- Declared three new protected methods: `veloc_protect()`, `veloc_checkpoint()`, `veloc_restart()`

**5. `src/app_lattice.cpp`**
- **`veloc_protect()`**: Registers all simulation state with VeloC via `VELOC_Mem_protect` — covers `time`, `naccept`, `nattempt`, `nsweeps`, `iarray[i]` (spin states), `darray[i]` (double states). Called once per process.
- **`veloc_checkpoint()`**: Wraps `VELOC_Checkpoint()` with an incrementing version counter.
- **`veloc_restart()`**: Uses `VELOC_Restart_test("spparks_ckpt", INT_MAX)` to find the latest checkpoint, then `VELOC_Restart()` to restore. Returns 1 if restored.
- **`iterate()`**: Calls `veloc_protect()`, then `veloc_restart()`. If restarted with a KMC solver, recomputes propensities from the restored site state (since `setup()` ran before restoration).
- **Periodic checkpoints**: Every `VELOC_CKPT_INTERVAL=5` sweeps/events in all iterate paths (`iterate_kmc_global`, `iterate_kmc_sector`, `iterate_rejection`, `iterate_app_update_only`).

### Restart Correctness
- `stoptime` is set by `App::run()` before `iterate()` based on the original `time=0`, so it remains correct (e.g., 10.0)
- After VeloC restores `time=5.0` from checkpoint, the iterate loops run only the remaining 5.0 simulation time units
- For KMC, propensities are recomputed from restored site state before the main loop
