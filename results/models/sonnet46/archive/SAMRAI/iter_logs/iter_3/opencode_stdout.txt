Here's a summary of the changes made to fix the checkpoint/restart correctness issue:

## Changes Made

**File: `source/test/applications/LinAdv/main.cpp`**

### Problem Analysis
The iter_2 validation failed with `output_correct=False` after recovery. The key failure was `max_abs_diff=1.648e+08` (4.6% relative error) in the final simulation output. The recovery exited cleanly (exit=0) but produced wrong results.

### Root Cause Candidates
1. **File format mismatch**: The old checkpoint format had no magic header, so corrupt/truncated files could cause silent misinterpretation of the binary format
2. **Wrong patch matching**: The restore function matched patches by `local_id` rather than spatial box coordinates - if AMR load balancing redistributed patches between ranks, local_ids wouldn't match
3. **Single-rank file reading**: Each rank only read its own file - if patches migrated between ranks during the fast-forward, those patches would never be found

### Fixes Applied

1. **New file format with magic header** (`0x55564C32` = "UVL2"):
   - Added a magic number to the uval checkpoint files
   - Changed file format: removed `local_id` (unreliable), kept `ndim`, `lo[]`, `hi[]`, `num_cells`, `data[]`
   - Reader checks magic header first; mismatched or truncated files are cleanly rejected

2. **Box-coordinate matching** (`applyUvalFile`):
   - New `applyUvalFile()` function matches patches by `(level, lo[], hi[])` spatial coordinates instead of `local_id`
   - This is correct even if AMR load balancing assigned the same patch to a different MPI rank

3. **All-rank file reading** (`readUvalFromAllRankFiles`):
   - New `readUvalFromAllRankFiles()` reads checkpoint files from ALL MPI ranks (not just the local rank)
   - Each rank applies only patches that match its own current hierarchy
   - This correctly handles any patch redistribution between checkpoint and restart
