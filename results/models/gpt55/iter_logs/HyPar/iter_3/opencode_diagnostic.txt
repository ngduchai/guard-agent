Root cause from prior validation:

- Failure-free runs exited immediately with:
  - `VeloC restart failed ... version -2`
- The code treated any value other than `VELOC_FAILURE` (`-1`) as a valid checkpoint version.
- In this environment, clean startup can return another negative status (`-2`), so the application attempted an invalid restart before writing any checkpoints.
