**Restart Behavior**
- On process start, after the input-created app has allocated/setup state, the code calls `VELOC_Restart_test()`.
- If a checkpoint exists, it restores registered memory through `VELOC_Recover_mem()`.
- It preserves the newly parsed target `stoptime` so resumed execution continues toward the requested endpoint.
- It re-runs `setup()` after recovery so transient communication/solver structures are rebuilt from restored persistent state.
- It only attempts restart once per process startup, preventing normal multi-`run` input scripts from self-restarting from checkpoints they just wrote.
