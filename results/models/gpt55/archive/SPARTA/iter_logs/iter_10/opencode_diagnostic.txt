  - Removed the early checkpoint restore from `create_particles`.
  - Restart now happens only from the `run` command path, after the input script has completed setup for timestep, stats, computes, and output. This avoids restoring checkpointed particles into a partially initialized command flow.

- `src/update.cpp`
  - Changed repeated `veloc_restart_if_available()` calls after a successful restore to return `0`, so “already restored” is not treated as a fresh restore event.
  - Fixed fallback checkpoint search to match VeloC’s API contract: `VELOC_Restart_test(name, version)` searches for checkpoints strictly less than `version`, so after a failed candidate the next bound should be `version`, not `version - 1`.
