Implemented a focused fix for the rejected restart path.

What changed:

- Updated `src/update.cpp` restart logic so all MPI ranks agree on one VeloC checkpoint version before recovery:
  - Each rank calls `VELOC_Restart_test`.
  - Ranks collectively select the minimum available version with `MPI_Allreduce`.
  - This avoids the previous failure where ranks could restore different checkpoint versions and then enter mismatched MPI collectives.

- Made restart success collective:
  - A rank-local recovery failure now causes all ranks to reject the restart consistently.
  - Sorting and `sparta_veloc_restarted` are only applied after all ranks successfully recover.

- Tightened checkpoint payload validation:
  - Metadata timestep must match the agreed checkpoint version.
  - Particle payload size must exactly match `meta.nlocal * sizeof(Particle::OnePart)`.
  - This prevents accepting stale or mismatched rank-local checkpoint data.

- Confirmed `veloc.cfg` already exists at the source tree root with valid absolute `/tmp` paths:
  - `scratch = /tmp/sparta_veloc_scratch`
  - `persistent = /tmp/sparta_veloc_persistent`
  - `mode = sync`

Root cause addressed:

- Prior validation failed because the failure-prone recovery exited with `MPI_ERR_TRUNCATE` in `MPI_Allreduce`.
- The likely cause was inconsistent restored timestep/state across ranks after a kill near an in-flight checkpoint.
- The new collective checkpoint-version negotiation prevents ranks from restarting from different VeloC versions.
