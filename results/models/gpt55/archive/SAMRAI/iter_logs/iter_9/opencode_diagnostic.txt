  - Any VeloC-reported version is now rounded down to the nearest completed `LINADV_CHECKPOINT_INTERVAL`, including versions below/at the interval boundary.
  - This avoids trying to restart from non-application checkpoint versions.

- Persisted real per-level time integrator state:
  - Added `TimeRefinementIntegrator::getCheckpointState(...)`.
  - Added overloaded `TimeRefinementIntegrator::restoreCheckpointState(...)` that restores:
    - per-level step counts
    - max step counts
    - simulation times
    - old/old-old times
    - max and actual dt values
  - This replaces the previous heuristic restart that set finer levels to step `0` and all level times/dts to the coarse value.

- Extended LinAdv checkpoint format from `LINADV_VELOC_1` to `LINADV_VELOC_2`:
  - `LinAdv::writeCheckpointState(...)` now writes the integrator arrays before hierarchy/patch data.
  - `LinAdv::rebuildCheckpointHierarchy(...)` skips that metadata while rebuilding patch levels.
  - `LinAdv::readCheckpointState(...)` reads metadata and returns it to `main`.

- Added a narrow restart synchronization hook:
  - `HyperbolicLevelIntegrator::synchronizeTimeDependentDataForRestart(...)`
  - This mirrors restored `CURRENT` state into `OLD` when three time levels are active, keeping SAMRAI time-dependent contexts coherent after checkpoint load.

Why this addresses the validator failure:

- The prior validator output showed failure-free correctness passed, checkpoint files existed, and recovery exited cleanly, but failure-prone output diverged numerically after restart.
- That points to incomplete restart state, not missing VeloC calls.
- The old restart path restored the solution field and coarse step/time/dt, but reconstructed finer-level integrator state approximately. For AMR/refined timestepping, those per-level counters and times affect regridding, synchronization, and subsequent advances.
- The checkpoint now captures and restores that actual evolved integrator state.

I could not run a build or validator in this environment because no shell/build execution tool is available here, so this is source-checked only.
