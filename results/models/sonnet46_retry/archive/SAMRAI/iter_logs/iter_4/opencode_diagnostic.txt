## Summary of Changes

### Root Cause of the Timeout (iter_3 failure)
The recovery run (attempt_2) took 107.32s and timed out. This was caused by `VELOC_Restart` (the memory-region-based restore) silently failing, leaving `veloc_scalars` = {0.0, 0.0, 0.0}. When `dt_now = veloc_scalars[1] = 0.0` was used, `advanceHierarchy(0.0, ...)` entered an infinite loop (dt=0 means no time advancement). The simulation was killed by the 106.2s timeout.

### Changes Made

**1. `source/test/applications/LinAdv/main.cpp`**

Completely replaced the VeloC memory-region based approach with a `VELOC_Route_file` based approach:

- **Checkpoint**: Uses `VELOC_Checkpoint_begin` → `VELOC_Route_file("state.bin", ...)` → manual `fwrite` (header + patch data) → `VELOC_Checkpoint_end`. MPI_Allreduce ensures all ranks agree on checkpoint success before calling `VELOC_Checkpoint_end`.

- **Restore**: Uses `VELOC_Restart_test` → `VELOC_Restart_begin` → `VELOC_Route_file("state.bin", ...)` → manual `fread` with full validation (positive dt, non-negative loop_time, iter >= 1) → MPI_Allreduce to ensure collective success/failure → `VELOC_Restart_end`. If restore fails for any rank, all ranks fall back to initial state.

- All error paths are properly handled - no silent failures that could cause infinite loops.

**2. `veloc.cfg`**
Added `max_versions = 2` to keep the 2 most recent complete checkpoints, providing a fallback if the latest was killed mid-commit.

**3. `source/SAMRAI/algs/TimeRefinementIntegrator.h` and `.cpp`**
(Already modified in iter_3): Extended `restoreCheckpointState` to accept and apply `dt_level0` parameter, ensuring the correct dt is used after restart.
