## Changes Made

**File**: `source/test/applications/LinAdv/main.cpp`

### Problem
The recovery attempt was crashing with a segfault (exit code 139) during restart. The crash occurred in the complex AMR hierarchy reconstruction code for finer levels (levels 1+), specifically in the `makeNewPatchLevel` / `initializeLevelData` / `resetHierarchyConfiguration` sequence.

### Root Cause
The old code tried to reconstruct finer AMR levels by:
1. Removing existing levels from the hierarchy
2. Manually building a `BoxLevel` from checkpoint box data
3. Calling `makeNewPatchLevel` with the manually-built `BoxLevel`
4. Calling `initializeLevelData` and `resetHierarchyConfiguration`

This sequence requires proper overlap connectors between levels (which are normally set up by the `GriddingAlgorithm`). Without them, SAMRAI crashes.

### Fix
**Simplified checkpoint format**: Save only level-0 (coarse grid) data + 3 scalar values (loop_time, dt_now, iteration_num).

**Simplified restart logic**:
1. After `initializeHierarchy()`, overwrite level-0 patch data with checkpoint data (same patch structure)
2. **Remove all finer levels** (just call `removePatchLevel` from finest to 1)
3. Call `restoreCheckpointState` to set the time integrator's internal state
4. Let the main time-stepping loop naturally rebuild finer levels via `regridAllFinerLevels` on the first `advanceHierarchy` call

The finer levels are rebuilt correctly because:
- The `GriddingAlgorithm` uses gradient detection on the restored level-0 data
- This is the same SAMRAI code path used for normal regridding (crash-proof)
- The initial finer-level data is interpolated from the correctly-restored level-0 data
