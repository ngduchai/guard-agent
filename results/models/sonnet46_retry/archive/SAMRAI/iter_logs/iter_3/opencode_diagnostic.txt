## Summary of Changes

### Root Cause
The recovery run (attempt 2) crashed with SIGSEGV at `main+0x1f4c`. The crash was in `gridding_algorithm->regridAllFinerLevels(...)` called directly from the restart block. This call propagates to `tagCellsForRefinement` → `tagGradientDetectorCells` → `getDataContext()` on the `LinAdv` object. The data context (`d_data_context` in `HyperbolicPatchStrategy`) was null at this point — it's only set by `HyperbolicLevelIntegrator` during active integration, not from external user code.

### Files Modified

**`source/test/applications/LinAdv/main.cpp`**

### How Recovery Now Works

2. VeloC restores level-0 patch data (checkpoint state) and loop scalars
3. Stale finer levels are removed via `removePatchLevel`
4. The first `advanceHierarchy()` call advances level 0 using the restored checkpoint data, then internally calls `regridAllFinerLevels` (with proper data context set by the integrator) to recreate finer levels 1 and 2 from the correct level-0 data
