## Implementation Summary

### What was done

Added VeloC-based checkpoint/restart support to the SAMRAI LinAdv AMR application.

### Architecture

**Checkpoint strategy**: At the end of each coarse timestep, checkpoint:
1. A 3-element scalar array: `{loop_time, dt_now, iteration_num}`
2. The level-0 CURRENT `uval` data for all local patches (the authoritative coarse-grid state)

**Restart strategy**: After `initializeHierarchy()`:
1. Check for an existing checkpoint via `VELOC_Restart_test`
2. If found: restore level-0 CURRENT data from the checkpoint
3. Remove all finer levels (levels 1, 2) that `initializeHierarchy()` created with step-0 data
4. Re-create finer levels via `regridAllFinerLevels()` from the restored level-0 data (gradient detection tags the same cells → same mesh → data filled by spatial refinement from level-0)
5. Restore the integrator internal state (step counter, simulation time) via the new `restoreCheckpointState()` method

**Why this works**: Level-0 CURRENT data after each coarse step already includes all contributions from finer levels (via `standardLevelSynchronization`). With `regrid_interval=1` (default) and gradient detection (not Richardson extrapolation), the finer levels are fully determined by the level-0 state. After removing old finer levels and calling `regridAllFinerLevels()`, the new finer levels are correctly initialized from level-0 data only (no contamination from step-0 initialization data).
