## Root Cause Analysis

The previous iterations failed because `src/main.cpp` called `lammps->input->one("write_restart ...")` and `lammps->input->one("read_restart ...")`, but **`write_restart` and `read_restart` are NOT in this LAMMPS source tree** — neither the `.h` nor `.cpp` files exist. LAMMPS's `execute_command()` function doesn't recognize these commands and calls `error->all(FLERR, "Unknown command: write_restart ...")` which executes `MPI_Abort(world, 1)`. This caused the crash at ~11 seconds (the time of the first checkpoint attempt after 3300 MD steps).

## Fix Applied

Complete rewrite of `src/main.cpp` with a new checkpointing strategy:

### Checkpoint (write)
1. `VELOC_Checkpoint_begin(name, version)` — begin checkpoint phase
2. `VELOC_Route_file(orig_name, routed)` — get VeloC-managed path for per-rank binary file
3. Write binary per-rank atom file (binary format: int64 header + AtomRec structs with exact double positions/velocities)
4. `VELOC_Checkpoint_mem()` — write memory regions (timestep, version, total_steps)
5. `VELOC_Checkpoint_end(1)` — commit (sync mode)

### Restart (read)
1. `VELOC_Restart_begin(name, version)` + `VELOC_Recover_mem()` — restore metadata
2. `VELOC_Route_file(orig_name, routed)` — get routed path for checkpoint files
3. Read binary per-rank atom data (exact binary, no text round-trip)
4. `VELOC_Restart_end(1)` — end restart
5. Create fresh LAMMPS instance, set `units lj`, `atom_style atomic`, `create_box` (from checkpoint box dims using `%.17g` precision)
6. `lammps_create_atoms(...)` — inject all atoms from checkpoint (gathered via `MPI_Allgatherv`)
7. Reapply force field, `reset_timestep N`, `run R pre yes` to complete

### Key Design Choices
- **Binary format**: Preserves exact double-precision values (no text round-trip error)
- **Per-rank files via `VELOC_Route_file`**: VeloC manages file versioning and persistence
- **`lammps_create_atoms`**: Uses LAMMPS's library API with `domain->ownatom` to correctly distribute atoms
- **`pre yes` on restart**: Rebuilds neighbor lists and forces from restored positions
- **`pre no` for subsequent chunks**: Maintains trajectory continuity after checkpoints (VeloC file writes are read-only on atom arrays)
