## Summary of Changes

**File modified**: `src/main.cpp`

### Root Cause Analysis
All previous iterations (3-5) failed because the binary crashed with exit code 1 within 4-11 seconds. The failures were caused by:

1. **`VELOC_Init` failing or crashing during checkpoint**: The previous code used temp files for setup commands, which introduced complex dependencies. The crash at ~11s matched the time of the first checkpoint attempt (~10s for 3000 steps).

2. **Failure-free output wrong**: Tracked to using wrong setup approach that didn't properly match the baseline's simulation behavior.

### New Implementation Strategy

1. **Absolute veloc.cfg path**: `resolve_veloc_cfg()` searches CWD and parent directories, then uses `realpath()` to get an absolute path. This ensures `VELOC_Init` succeeds regardless of which CWD the validator uses.

2. **Pre-create VeloC directories**: Both scratch and persistent directories are created before `VELOC_Init` is called, handling the case where the validator deletes (not just empties) these directories.

3. **No temp files**: Instead of writing setup commands to a temp file and passing it via `-in`, commands are fed line-by-line via `lammps->input->one()`. The LAMMPS instance is created with `-log none` (no input file).

4. **Restart binary in persistent dir**: Written directly to `/tmp/lammps_veloc_persistent/lammps_ckpt_vN.bin`. The persistent directory is never cleared between the kill run and the recovery run.

5. **VeloC memory regions only**: VeloC tracks only small metadata (timestep, version). The large restart binary is managed separately.

6. **Correct simulation chunking**: First chunk uses `pre yes` (matching baseline's initialization), subsequent chunks use `pre no` (maintaining floating-point continuity). This ensures the failure-free output matches the baseline.
