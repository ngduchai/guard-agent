The path returned is typically: `<scratch>/<name>/<filename>` or similar. In our case, calling `VELOC_Route_file("lammps_restart.bin", routed)` might return something like `/tmp/lammps_veloc_scratch/lammps_restart.bin` (same for ALL ranks!).

If all ranks get the SAME path from `VELOC_Route_file`, then:
- Ranks 1-3 create empty files at that path
- Rank 0 broadcasts its path (which is the same as everyone else's)
- `write_restart` writes the actual restart binary to that path
- BUT: ranks 1-3 already wrote empty files there!

Wait, would ranks 1-3 overwrite rank 0's file? No, because ranks write sequentially with MPI. Actually the order is:
1. All ranks call `VELOC_Route_file` → get a path
2. Non-zero ranks create empty files at `local_routed`
3. Rank 0 broadcasts its path
4. All call `write_restart shared_path`

If all ranks get the SAME path from `VELOC_Route_file`, then:
- Ranks 1-3 create empty files at the shared path
- Then ALL ranks call `write_restart` which writes the actual data from rank 0
- `write_restart` overwrites the empty files with real data ✓
