## Summary

The build failure in iteration 1 was caused by **undefined references to VeloC functions** (`VELOC_Route_file`, `VELOC_Restart_end`, `VELOC_Finalize`, etc.) because the `lmp` executable target was not linked against the VeloC client library.

**Root cause**: `cmake/CMakeLists.txt` had `target_link_libraries(lmp PRIVATE lammps)` but no reference to `libveloc-client.so`.

**Fix applied**: Added VeloC library discovery and linking to `cmake/CMakeLists.txt` (lines 221-235 after the `install(TARGETS lmp ...)` line):

```cmake
find_library(VELOC_CLIENT_LIB
  NAMES veloc-client
  HINTS /home/ndhai/usr/lib /home/ndhai/usr/lib64
  NO_DEFAULT_PATH)
if(NOT VELOC_CLIENT_LIB)
  find_library(VELOC_CLIENT_LIB NAMES veloc-client)
endif()
if(VELOC_CLIENT_LIB)
  target_include_directories(lmp PRIVATE /home/ndhai/usr/include)
  target_link_libraries(lmp PRIVATE ${VELOC_CLIENT_LIB})
else()
  message(FATAL_ERROR "VeloC client library (libveloc-client) not found")
endif()
```

The library `libveloc-client.so` is at `/home/ndhai/usr/lib/` and the header `veloc.h` is at `/home/ndhai/usr/include/`. The checkpoint/restart logic in `main.cpp` and the `veloc.cfg` file were already correct from iteration 1.
