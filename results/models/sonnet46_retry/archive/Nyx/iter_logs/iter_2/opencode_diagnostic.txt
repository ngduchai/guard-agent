Here's a summary of what was fixed and why:

## Root Cause

The previous implementation had an MPI deadlock risk: `VELOC_Restart_test` is called independently on each rank and could return different values (different version numbers, or VELOC_FAILURE on some ranks). If any rank saw no checkpoint but others did, the `has_checkpoint` flag would differ across ranks, causing:
- Some ranks to call `VELOC_Restart` (collective)
- Other ranks to skip it and proceed to the main loop's `amrptr->coarseTimeStep()` (also collective)

Both sides block forever waiting for the other to join the collective. This is the classic MPI deadlock described in the diagnosis.

## Changes Made

**`Source/Driver/nyx_main.cpp`**:
1. **`MPI_Allreduce` after `VELOC_Init`**: Ensures all ranks consistently see success or failure, using `MPI_MIN` so any single rank failure disables checkpointing globally.
2. **`MPI_Barrier` + `MPI_Allreduce` for `VELOC_Restart_test`**: All ranks query the checkpoint state, then `MPI_Allreduce` with `MPI_MIN` ensures a globally agreed-upon version. If any rank can't find the checkpoint, no rank restarts.
3. **`MPI_Barrier` + `MPI_Allreduce` in `velocRestart`**: After `VELOC_Restart`, reduces the result so all ranks agree on success or failure. Prevents divergent code paths after failure.
4. **Checkpoint every 10 steps** (`VELOC_CKPT_INTERVAL = 10`): Reduces I/O load from 3000+ checkpoint operations to ~300.

**`veloc.cfg`**: Added `max_versions = 2` to limit checkpoint accumulation on disk.
