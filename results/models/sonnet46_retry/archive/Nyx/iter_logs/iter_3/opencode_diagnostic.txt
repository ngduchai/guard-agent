Also checking: when VeloC mem restore:
```cpp
int v_ver = VELOC_Restart_test(VELOC_MEM_NAME, INT_MAX);
int agreed_ver = -1;
MPI_Allreduce(&v_ver, &agreed_ver, 1, MPI_INT, MPI_MIN, MPI_COMM_WORLD);
if (agreed_ver >= 0) {
    VELOC_Restart(VELOC_MEM_NAME, agreed_ver);
}
```

`VELOC_Restart_test` returns VELOC_FAILURE (-1) or the version number (≥0). Using `MPI_MIN`: if all ranks return -1, `agreed_ver = -1` → no restore. If any rank returns ≥0, `agreed_ver` might be -1 (if another rank returns -1). In sync mode, all ranks should return the same value.

But in case of inconsistency: Using `MPI_MIN` is conservative. If ANY rank has no checkpoint, we don't attempt restore. This is the safe approach to avoid deadlocks. ✓

The VeloC mem restore is not critical for correctness (we restore AMReX from its own checkpoint). It just updates `veloc_ckpt_step`. Even if it fails, the simulation works correctly. ✓
