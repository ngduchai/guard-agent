## Root Cause Analysis

The previous iteration (iter_4) failed because on restart attempt (attempt_2), the application crashed with:
```
amrex::Error::get_comoving_a: invalid time !!!
SIGABRT
```

The crash occurred because `Nyx::old_a_time`, `Nyx::new_a_time`, `Nyx::old_a`, and `Nyx::new_a` are C++ static class variables initialized to -1.0. They're only set in the `Nyx` constructor when `time == 0.0` (fresh start). On restart, the constructor is called with the checkpoint time (non-zero), so these statics remain at -1.0. When `computeNewDt()` calls `integrate_comoving_a(cur_time, dt)`, it finds neither `old_a_time` nor `new_a_time` match `cur_time`, causing an `amrex::Error()` abort.

## Fix Applied

**File modified**: `Source/Driver/nyx_main.cpp`

1. **Added `#include <fstream>`** for file I/O

2. **Added `NYX_COMOVING_FNAME` constant** (`"nyx_comoving_state"`)

3. **Added `writeComovingState()`**: Writes the 4 comoving scalars (`old_a_time`, `new_a_time`, `old_a`, `new_a`) to `<chkdir>/nyx_comoving_state` after each AMReX checkpoint

4. **Added `readComovingState()`**: Reads the 4 comoving scalars back on restart; broadcasts values from rank 0
