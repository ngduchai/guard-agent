## Summary of Changes

### Root Cause Analysis
The binary crashed with `MPI_ABORT on rank 2` within 1 second. The crash was caused by **hash table corruption**: when remote events (received from other MPI ranks) were drained from the priority queue during the checkpoint hook, they were freed to the `free_q` WITHOUT being removed from the hash table (`pe->hash_t`). 

The hash table is used to match cancel (anti-)messages to events. With stale entries pointing to freed/re-used event memory, when the next checkpoint's rollback processed cancel messages, it retrieved freed event memory from the hash table. The freed event had `state.owner = TW_pe_free_q`, triggering `tw_error("Cancelled event, no owner!")` → `MPI_Abort`.

### Files Modified

**`models/phold/phold.main.c`**:

1. **Added `hash-quadratic.h` include** to access `tw_hash_insert` and `tw_hash_remove`

2. **Added `is_remote` field to `ckpt_event_t`** to track whether each checkpointed event was originally a remote (network-received) event

3. **Added `core_rng` fields to `ckpt_rng_t`** to correctly save/restore the tiebreaker RNG state for bit-identical recovery

4. **Fixed drain loop in `veloc_gvt_hook`**: Before freeing a remote event, removes it from the hash table (`tw_hash_remove`) to eliminate the stale entry

5. **Fixed `inject_event_into_pq`**: After re-enqueueing a remote event to the PQ, re-inserts it into the hash table (`tw_hash_insert`) so anti-messages can correctly find and cancel it

6. **Updated RNG save/restore** to include `core_rng` (tiebreaker RNG) ensuring deterministic recovery
