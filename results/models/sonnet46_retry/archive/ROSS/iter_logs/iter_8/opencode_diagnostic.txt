Here's a summary of the changes made to fix the disk exhaustion issue:

## Root Cause
The checkpoint footprint was 11+ GB (52,852 files) because:
1. Checkpointing happened every 5 GVT rounds → ~13,000 checkpoint versions per rank in 88 seconds
2. `max_versions` was not set → VeloC kept ALL versions (default: keep all)

## Fixes Applied

### 1. `veloc.cfg` — Added version retention limits
```
max_versions = 1
scratch_versions = 1
```
This tells VeloC to keep only the most recent checkpoint version, automatically deleting older ones.

### 2. `models/phold/phold.main.c` — Wall-clock throttling
- Added `#include <time.h>` and a `wall_now()` helper using `clock_gettime(CLOCK_MONOTONIC)`
- Changed `CKPT_GVT_INTERVAL` to `CKPT_WALL_INTERVAL = 15.0` seconds
- The GVT hook now checks if 15 wall-clock seconds have elapsed since the last checkpoint and returns immediately if not
- Added `g_last_ckpt_wall` global initialized at simulation start
- Updated `tw_trigger_gvt_hook_every(10)` (down from 5) — the hook fires every 10 GVT rounds but only actually checkpoints every 15 seconds

### Result
With these changes, the simulation (~88 seconds) will produce at most 5-6 checkpoint versions, but with `max_versions=1`, only the latest is retained. Total disk usage: ~1-2 MB instead of 11+ GB.
