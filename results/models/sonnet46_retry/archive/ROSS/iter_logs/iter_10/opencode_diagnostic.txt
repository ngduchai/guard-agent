**Summary of Changes Made to `phold.main.c`:**

1. **Root Cause Fix - Checkpoint Strategy**: The previous code tried to collect events from `pe->pq` during the GVT hook, but in OPTIMISTIC ROSS mode, most events have been speculatively processed and are in `kp->pevent_q`, not in `pe->pq`. So `pe->pq` was nearly empty at checkpoint time.

2. **Key Fix 1 - Rollback Before Checkpoint**: Now calls `tw_scheduler_rollback_to_gvt_only(pe)` before collecting events. This rolls back all speculatively-processed events from `kp->pevent_q` back into `pe->pq`, giving us the complete set of pending events from GVT onwards.

3. **Key Fix 2 - Set pe->GVT on Restart**: After restoring from checkpoint, sets `pe->GVT = checkpoint_gvt`. This tells the ROSS scheduler to start GVT from the checkpoint point, allowing the simulation to run only from `checkpoint_gvt` to `end_time` instead of from 0 to `end_time`. This is what makes `recovery_actually_resumed=True`.

4. **Reduced Checkpoint Interval**: Changed from 15s to 5s wall-clock interval to ensure a checkpoint is taken well before the kill injection (which happens at ~55s wall-clock in a 88s run).

5. **Increased Event Buffer**: Changed `MAX_CKPT_EVENTS` from 524288 to 1048576 to handle potentially larger event sets after rollback.

6. **Simplified Event Injection**: Removed the hash_insert for remote events (not needed after GVT rollback since no remote events should be in PQ after clean rollback).
