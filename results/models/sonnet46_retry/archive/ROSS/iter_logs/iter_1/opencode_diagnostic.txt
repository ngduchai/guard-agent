Added full VeloC checkpoint/restart support:

- **`VELOC_Init`** called after `tw_init` with `MPI_COMM_WORLD`
- **`VELOC_Mem_protect`** registers per-LP RNG state buffer (`ckpt_rng_t` array)
- **GVT hook** (`veloc_gvt_hook`) triggered periodically:
  - Sequential mode: every `CKPT_TIME_INTERVAL` virtual time units (rescheduled from hook)
  - Optimistic/Conservative: every `CKPT_GVT_INTERVAL` GVT rounds
  - For optimistic: calls `tw_scheduler_rollback_and_cancel_events_pe` to get committed LP state
  - Captures LP RNG streams (`Ig/Lg/Cg/count` + normal RNG state)
  - Drains PQ, saves all pending events (`recv_ts`, `dest_lpid`, `sig`, message), re-injects them
  - Writes checkpoint via `VELOC_Checkpoint_begin/mem/end` + `VELOC_Route_file` for event queue
- **PE post_lp_init hook** (`veloc_post_lp_init`) on restart:
  - Detects checkpoint via `VELOC_Restart_test(CKPT_NAME, INT_MAX)`
  - Restores LP RNG states via `VELOC_Recover_mem`
  - Reads saved event queue from VeloC-routed file
  - Clears init events from PQ, injects checkpoint events
  - `VELOC_Finalize(0)` called at end (keeps checkpoints)
