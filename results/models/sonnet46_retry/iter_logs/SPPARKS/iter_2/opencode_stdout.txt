```

This limits VeloC to keeping only the 2 most recent checkpoint versions on both persistent and scratch storage, preventing unbounded disk growth.

### 2. `src/app_lattice.cpp` - Reduced checkpoint frequency
Changed `veloc_checkpoint()` to only write checkpoints every 50 sweeps:

static const int VELOC_CKPT_INTERVAL = 50;

```

## Root Cause Analysis

The previous attempt was rejected because:
1. **Checkpoint every sweep** → ~11,684 checkpoint versions per rank in 22 seconds
2. **No version retention limit** → all versions kept on disk  
3. **Total disk usage**: 11.5 GB → exceeded the 10.74 GB disk limit in just 22 seconds
4. **Disk guard terminated** the process before any useful checkpoints were available for recovery

## Expected Behavior After Fix

- **Checkpoint interval**: Every 50 sweeps → ~376 checkpoints for a full 18,800-time-unit run
- **Versions retained**: 2 → maximum 2 versions on disk at any time
- **Disk usage**: 4 ranks × 2 versions × ~250 KB = ~2 MB total (well within limits)
- **Recovery**: When process is killed at ~60% runtime, the most recent checkpoint (~50 sweeps before the kill) is available for recovery
- **Recovery completion**: Restores spin state, time, and RNG seed, then continues simulation to `stoptime=18800` and writes `validation_output.bin`
