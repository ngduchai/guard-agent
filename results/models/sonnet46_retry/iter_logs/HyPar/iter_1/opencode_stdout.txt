## What was done

### 1. `veloc.cfg` (created)
**Why:** The validator parses this file before launching `mpirun` to locate checkpoint directories. Without it, validation immediately fails. The two paths must be distinct absolute `/tmp` paths.

### 2. `CMakeLists.txt` (modified)
**Why:** VeloC headers and libraries need to be found at compile/link time. Added:
- `include_directories(/home/ndhai/usr/include)` — VeloC header location
- `link_directories(/home/ndhai/usr/lib)` — VeloC `.so` location
- `add_compile_definitions(with_veloc)` — enables `#ifdef with_veloc` guards in source
- `link_libraries(veloc-client)` — the actual VeloC client library name (`libveloc-client.so`)
- `CMAKE_BUILD_RPATH` / `CMAKE_INSTALL_RPATH` — ensures the shared library is found at runtime without needing `LD_LIBRARY_PATH`

### 3. `src/Simulation/Solve.cpp` (modified)
**Why:** This is where the time integration loop lives. The checkpoint/restart logic was inserted here because:
- **Before the loop:** `VELOC_Init` starts the library; `VELOC_Mem_protect` registers `sim[ns].solver.u` (the solution array) for each domain plus an integer `veloc_saved_iter` tracking the next iteration; `VELOC_Restart_test` detects an existing checkpoint and, if found, restores all registered memory regions and sets `sim[ns].solver.start_iter` so `TimeInitialize` computes `waqt` correctly.
- **Loop start changed** from `TS.iter = 0` to `TS.iter = TS.start_iter` — ensures we skip already-completed iterations on restart.
- **Inside the loop:** every 10 iterations, `veloc_saved_iter = TS.iter + 1` is set (the resume point) and `VELOC_Checkpoint` is called, which atomically saves all registered regions.
- **After `TimeCleanup`:** `VELOC_Finalize(0)` shuts down VeloC cleanly.
