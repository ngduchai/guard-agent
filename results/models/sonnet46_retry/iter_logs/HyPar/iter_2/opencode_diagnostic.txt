All fixes are in place. Here's a summary of what was changed and why:

## Root Cause (iter_1 failure)

The previous attempt checkpointed every 10 iterations with no version limit. With 4.7M total iterations over 52s of runtime before kill, this created ~287K checkpoint versions and **554K+ files** in the scratch/persistent directories. When the restart attempt began, `VELOC_Restart_test("hypar_ckpt", INT_MAX)` had to scan through all 554K files to find the latest version — this file system scan alone took longer than the 120.2s recovery timeout, causing the entire validation to time out.

## Fixes

### 1. `veloc.cfg` — Added version limits
```
max_versions = 1
scratch_versions = 1
```
VeloC now keeps only the **1 most recent checkpoint**, replacing the previous one each time. The restart test now finds the checkpoint instantly (just 1 version to check).

### 2. `src/Simulation/Solve.cpp` — Time-based checkpointing + safe collective logic

- **Checkpoint interval**: Changed from every 10 iterations to every **5 seconds wall-clock time**, polled every 10,000 iterations to avoid per-iteration overhead
- **Collective safety**: `VELOC_Checkpoint` is collective in sync mode; added `MPI_Allreduce(MPI_MAX)` so all ranks agree on when to checkpoint — prevents deadlock from clock skew between ranks
- **Restart consensus**: Added `MPI_Allreduce(MPI_MIN)` on `VELOC_Restart_test` result so all ranks take the same code path (restart or fresh start) — prevents deadlock if ranks disagree on checkpoint availability
- **Version tracking**: Checkpoint versions start at 1 (or `previous_version + 1` on restart) and increment monotonically, satisfying VeloC's requirement
