**1. Fixed `veloc.cfg` path detection** (`find_veloc_cfg()`):
- The validator copies `veloc.cfg` to different directories depending on run type:
  - Failure-free runs: copies to CWD → try `"veloc.cfg"` first
  - Kill-attempt runs: copies to parent of CWD → try `"../veloc.cfg"` second
- Previously hard-coded `"../veloc.cfg"` caused exit code 255 on failure-free runs

**2. Read `CKPT_EVERY` from environment**:
- The validator sets `CKPT_EVERY=10000` to control checkpoint frequency
- With 32954 iterations at ~7.4ms/iter, first checkpoint at iter 10000 = ~74s
- Kill happens at ~56% = ~137s — checkpoint is well before the kill
- Previously hardcoded `CKPT_FREQ=500` caused excessive checkpoints → 11GB disk usage → disk-guard termination

**3. Removed `in` array from checkpoint**:
- The `in` array is reconstructible: `IN(i,j) = COEFX*i + COEFY*j + iter_done` after `iter_done` iterations
- Ghost cells are refreshed by halo exchange at start of next iteration
- Reduces checkpoint size by ~50% (from ~64MB/rank to ~32MB/rank)
- Now only `out` (accumulated stencil values) and `iter_done` are checkpointed

**4. Used `MPI_Abort` instead of custom error exit on VELOC_Init failure**:
- Cleaner collective abort rather than rank 0 exiting independently

### `veloc.cfg`

Added:
```
max_versions = 1
scratch_versions = 1
```
Limits VeloC to keeping only 1 checkpoint version in both scratch and persistent directories, preventing accumulated disk usage across multiple checkpoint events.
