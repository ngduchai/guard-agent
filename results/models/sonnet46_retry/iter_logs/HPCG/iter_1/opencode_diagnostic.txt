## Summary of Changes

### 1. `veloc.cfg` (new file, created in source tree root)

**What:** Static config file required by VeloC before the binary runs.
**Why:** The validator parses this file before invoking mpirun. Without it, the validator immediately FATALs. The two paths (`scratch` and `persistent`) must be different absolute `/tmp` paths.

### 2. `setup/Make.Linux_MPI` (modified)

- Added `-I/home/ndhai/usr/include` to `HPCG_INCLUDES` so `#include <veloc.h>` resolves.
- Added `-L/home/ndhai/usr/lib -lveloc-client -Wl,-rpath,/home/ndhai/usr/lib` to `HPCG_LIBS`:
  - The VeloC library file is named `libveloc-client.so` (not `libveloc.so`)
  - The rpath embeds the library search path into the binary so it works at runtime

### 3. `src/main.cpp` (modified)

- Added `#include <climits>` and `#include <veloc.h>`
- Added `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` immediately after `MPI_Init`
- In the main CG timing loop section:
  - Registered 4 memory regions with `VELOC_Mem_protect()`:
    - `completed_sets` (int): tracks how many CG sets are done — the restart cursor
    - `testnorms_data.values[]`: the accumulated residual norms
    - `times[]`: accumulated timing data
    - `last_cg_scalars[3]`: niters, normr, normr0 from last CG call
  - On startup, calls `VELOC_Restart_test("hpcg_cg", INT_MAX)` to detect existing checkpoints
  - If a checkpoint exists, calls `VELOC_Restart()` to restore protected memory and resumes the loop from `completed_sets` (not from 0)
  - The loop now starts at `i = completed_sets` instead of 0
  - After each CG set, calls `VELOC_Checkpoint("hpcg_cg", completed_sets)` with a monotonically-increasing version number
- Added `VELOC_Finalize(0)` before `MPI_Finalize` (with `cleanup=0` to preserve checkpoint files)
