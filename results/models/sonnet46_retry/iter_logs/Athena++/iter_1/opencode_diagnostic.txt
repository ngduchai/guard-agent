### What was done

**1. Created `veloc.cfg`** (static config file at source tree root)
- Required by the validator before `mpirun` launches the binary
- Specifies absolute `/tmp` paths that differ from each other for scratch and persistent storage
- Uses `mode = sync` for synchronous (reliable) checkpointing

**2. Modified `Makefile`**
- Added `-I/home/ndhai/usr/include` to `CPPFLAGS` so `veloc.h` and `veloc/defs.h` can be found
- Added `-L/home/ndhai/usr/lib` to `LDFLAGS` and `-lveloc-client -Wl,-rpath,...` to `LDLIBS`
- The actual library is named `libveloc-client.so` (not `libveloc.so`)

**3. Modified `src/main.cpp`**
- Included `<veloc.h>` and `<climits>` (for `INT_MAX`)
- Called `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` immediately after MPI init, before anything else
- After `pmesh->Initialize()`, registered memory regions with VeloC:
  - ID 0: a `double[3]` array for `{time, dt, ncycle}`
  - IDs 1..N: the `phydro->u` conserved variable arrays for each local MeshBlock
- Called `VELOC_Restart_test(veloc_name, INT_MAX)` to detect an existing checkpoint
- If checkpoint exists: called `VELOC_Restart()` to restore hydro data and mesh scalars (`time`, `dt`, `ncycle`)
- In the main integration loop, after each cycle completes: called `VELOC_Checkpoint(veloc_name, pmesh->ncycle)` to save state
- Called `VELOC_Finalize(0)` before `MPI_Finalize()` at all exit paths

### Key design decisions

- **Checkpoint timing**: After `ncycle++` and `time += dt` and `NewTimeStep()`, so the checkpoint captures the fully-advanced state with the next timestep already computed
- **Version = ncycle**: Monotonically increasing, so VeloC can track ordering correctly
- **Restart detection uses `INT_MAX`**: `VELOC_Restart_test(name, INT_MAX)` finds the latest checkpoint with any version number
- **Fresh init then overwrite**: Athena++ always runs its problem generator first (so all data structures are valid), then the checkpoint data overwrites the hydro arrays and scalars - a clean "soft restart" pattern
- **`mode = sync`**: Ensures checkpoints are fully written before returning, so any checkpoint found on restart is guaranteed to be complete
