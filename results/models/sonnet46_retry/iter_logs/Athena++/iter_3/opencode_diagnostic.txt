## Root Cause

The build has been failing with "undefined reference to `VELOC_Init'" across iter_1 and iter_2. The code in `src/main.cpp` already had VeloC calls added, and `Makefile` already had the correct `-lveloc-client` linker flags. However:

- The validator runs `configure.py` before `make` to regenerate the `Makefile` from `Makefile.in`
- The old `Makefile.in` template only had `@LINKER_FLAGS@` and `@LIBRARY_FLAGS@` placeholders
- When configure.py runs without `--lib_path=/home/ndhai/usr/lib --lib=veloc-client` arguments, those placeholders become empty/default values
- So the regenerated `Makefile` didn't link against `libveloc-client`
- Result: linker error

## Fix Applied

Modified `Makefile.in` to hardcode VeloC flags outside the `@...@` placeholders so they survive configure.py regeneration:

```makefile
CPPFLAGS := @PREPROCESSOR_FLAGS@ -I/home/ndhai/usr/include
LDFLAGS := @LINKER_FLAGS@ -L/home/ndhai/usr/lib
LDLIBS := @LIBRARY_FLAGS@ -lveloc-client -Wl,-rpath,/home/ndhai/usr/lib
```

## Existing Infrastructure (Already Correct)

- **`veloc.cfg`**: Present with `scratch=/tmp/athena_veloc_scratch`, `persistent=/tmp/athena_veloc_persistent`, `mode=sync` ✓
- **`src/main.cpp`**: Has complete VeloC checkpoint/restart implementation:
  - `VELOC_Init` after MPI_Init
  - `VELOC_Mem_protect` for scalars (time/dt/ncycle) and hydro arrays
  - `VELOC_Restart_test` + `VELOC_Restart` on startup to resume from checkpoint
  - `VELOC_Checkpoint` each cycle to save state
  - `VELOC_Finalize` before MPI_Finalize
