### Key Design Decisions

1. **File-based checkpointing**: Uses `VELOC_Route_file()` to write custom binary files per rank, containing ntimestep, laststep, and all local particles. This handles variable-size particle arrays.

2. **Multi-run script correctness**: Stores `laststep` in each checkpoint. On restart, the `laststep` is used to cap the nsteps of subsequent `run()` calls so the simulation ends at the original planned endpoint.

3. **Single restore attempt**: The restore is only attempted on the first `run()` call, preventing double-restores in multi-run scripts.

4. **Checkpoint frequency**: ~10 checkpoints spread across each run, plus one at the final step.
