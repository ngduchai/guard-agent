Here's a summary of what was done and why:

## Root Cause Analysis

The recovery attempt (attempt 2) crashed with a segfault (exit 139) in `MallocPlus::memory_realloc_all()`, called from the restart logic in `main()`.

The crash happened because `mesh->resize(saved_ncells)` calls `mesh_memory.memory_realloc_all(saved_ncells)`, which iterates over **all** entries in `mesh_memory` (dozens of arrays including face arrays, neighbor arrays, etc.) and calls `realloc()` on each. This triggered heap corruption, likely due to interaction with VeloC's internal memory management or signal handlers during `realloc()` execution.

## Fix Applied

**File changed**: `clamr_mpionly.cpp`

1. **Added `#include <string.h>`** for `memcpy`

2. **Replaced the crash-prone restart logic**: Instead of calling `mesh->resize(saved_ncells)` (which calls `memory_realloc_all` on all mesh arrays), the new code:
   - Reads checkpoint data into plain `malloc()` buffers first (no MallocPlus interaction during file I/O)
   - Validates `saved_ncells` before any allocation
   - Uses **targeted** `mesh_memory.memory_realloc(saved_ncells, ptr)` calls for only the 3 specific arrays that need resizing: `i`, `j`, `level`
   - Similarly uses targeted `state_memory.memory_realloc()` for `H`, `U`, `V`
   - Copies data from temp buffers to the properly-sized arrays via `memcpy`
   - Frees temp buffers

3. **Added MPI collective check**: After per-rank file reads, uses `MPI_Allreduce` to ensure all ranks agree on restart success, preventing deadlocks if any rank's file is missing.

4. **Rebuilt derived state after restart**: Calls `MPI_Allgather` to rebuild `nsizes`/`ndispl`/`ncells_global`, then calls `calc_spatial_coordinates`, `calc_celltype`, and `calc_neighbors_local` to reconstruct all derived mesh data.

**`veloc.cfg`** was already correct with distinct absolute paths for scratch and persistent directories.
