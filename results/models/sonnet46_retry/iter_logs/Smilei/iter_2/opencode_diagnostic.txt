## Root Cause

The previous attempt's code checked `VELOC_Restart_test()` return value against `VELOC_FAILURE` (defined as `-1`), but VeloC was returning **`-2`** when no checkpoint existed (e.g., after the validator cleared the scratch/persistent directories). 

Since `-2 != -1`, the condition `veloc_restart_version != VELOC_FAILURE` evaluated to `true`, causing the app to attempt a restart from a non-existent checkpoint. This immediately failed with:

```
[VELOC] Restart_begin failed at version -2
[VELOC] Restart failed — aborting
MPI_ABORT was invoked...
```

This happened in **every** run (all 3 kill attempts and the failure-free run), causing exit=1 at ~1s before any checkpoints could be written.

## Fix Applied

**`src/Smilei.cpp:604`** — Changed the restart detection check from:
```cpp
bool veloc_do_restart = ( veloc_restart_version != VELOC_FAILURE );
```
to:
```cpp
bool veloc_do_restart = ( veloc_restart_version >= 0 );
```

This correctly treats any negative return value (whether `-1`, `-2`, or other codes) as "no checkpoint available", and only attempts restart when VeloC returns a valid non-negative version number.
