### 3. `src/Smilei.cpp` (modified)
**Why**: This is the main PIC loop where all simulation state lives. The checkpoint/restart must be woven into the simulation lifecycle.

Key additions:
- `VELOC_Init(smpi.world(), "veloc.cfg")` — called after MPI init, using the MPI communicator for collective mode
- `VELOC_Restart_test(name, INT_MAX)` — detects any existing checkpoint before initialization
- After full initialization: `smileiVelocRestart()` loads checkpoint data (fields + particles) over the freshly-initialized state, and sets `itime` to resume from the saved step
- In the main loop at `itime % 50 == 0`: `smileiVelocCheckpoint()` writes binary files containing EM fields (Ex/Ey/Ez/Bx/By/Bz/Bx_m/By_m/Bz_m) and particle arrays (Position, Momentum, Weight, Charge) per patch per species
- `VELOC_Finalize(0)` — called before exit to keep checkpoint files

The file-based approach via `VELOC_Route_file` was chosen because particle arrays have variable sizes that change during the simulation, making memory-region registration impractical.
