  - `smileiCheckpointWrite()` — serializes per-rank state (itime, time_prim, time_dual, then for each patch: EM fields data_, then for each species: all `double_prop_`/`short_prop_`/`uint64_prop_` arrays + `first_index`/`last_index`/`cell_keys`) into a binary blob written through `VELOC_Route_file()`, framed by magic-number headers and trailer.
  - `smileiCheckpointRead()` — symmetric reader that resizes structures as needed and restores all state.
- In `main()`:
  - `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` right after `SmileiMPI` constructs MPI.
  - Right before the time loop, calls `VELOC_Restart_test()`; if a checkpoint exists, calls `smileiCheckpointRead()` and resumes `itime` from the restored value + 1.
  - Inside the time loop, every 50 iterations (and at the final iteration), calls `smileiCheckpointWrite()`.
  - `VELOC_Finalize(0)` before MPI shutdown (both in the normal path and the test-mode early-exit).

### Recovery semantics
On a mid-execution crash, the next launch:
1. Initializes the simulation from scratch (same Params, same patch grid, same species creation).
2. Detects a checkpoint via `VELOC_Restart_test()` and loads field values + particle arrays per patch/species, plus simulation time scalars.
3. Continues the time loop from the iteration following the checkpoint.
4. Reaches the same final state as the un-killed run, so `dumpValidationSignatureSmilei` emits identical numbers.
