## Summary

This made recovery take essentially the same time as a cold start (ratio=0.989, well above the gate_B<0.85 threshold).

**The fix**: implement real state save/restore. On checkpoint, serialize the level-0 `Efield_fp` / `Bfield_fp` MultiFab data (per-rank, including all FABs) to flat double buffers and write via `VELOC_Checkpoint`. On restart, after `WarpX::InitData()` rebuilds the (deterministic) per-rank box decomposition, `AttachAndRecover` allocates matching buffers, calls `VELOC_Recover_mem`, then unpacks the saved bytes back into the live MultiFabs and pushes the saved step / time into WarpX via `setistep` / `sett_new`. The subsequent `Evolve()` resumes from the saved step instead of step 0, completing only the remaining iterations.
