## Summary

**Root cause of previous failure (iter_4):** The previous attempt registered protected VeloC memory regions (`VELOC_Mem_protect`) for each MeshBlock's u-array ONCE at startup, before the main loop. With AMR enabled (`refinement = adaptive` in `athinput.blast`), MeshBlocks are dynamically refined, coarsened, and migrated between ranks, freeing and reallocating memory. The protected memory pointers became stale, causing use-after-free writes into the checkpoint and corrupted (NaN) state on restart.

**Fix:** Replaced static memory protection with **file-based per-rank checkpointing** using `VELOC_Checkpoint_begin/end` + `VELOC_Route_file`:

1. **Write path** (`AthenaVelocWriteCheckpointFile`): Each rank writes a self-describing binary file containing the current `nblocal`, per-block metadata (`gid`, `level`, `lx1/lx2/lx3`, `block_size`, `NHYDRO`), the u-array bytes, and global scalars (`time`, `ncycle`, `dt`). The file always captures the **live current state** — there are no stale pointers because we read the data directly from `pmesh->my_blocks(b)` at checkpoint time.

2. **Read path** (`AthenaVelocReadCheckpointFile`): Parses the file into `g_recovered` after `VELOC_Restart_begin/end`.

3. **Layout-match gate** (`AthenaVelocLayoutMatches`): Before `Initialize()`, compares the freshly-built Mesh's per-rank block layout against the saved layout (count + per-block level/location/dimensions). Only when ALL ranks match do we restore u-arrays via `AthenaVelocApplyBlocks` and call `Initialize(1, ...)` to refresh boundaries.

4. **Cold fallback**: If layouts differ (the kill landed during an AMR-refined window), we drop the checkpoint and run `Initialize(0, ...)` for a normal cold start — correct output, slower wallclock.

5. **Throttle tuned to 5 s** between checkpoints so the recovery has enough residual integration to satisfy the `recovery >= 0.1 × kill` floor while still fitting in the `1.255× baseline` wallclock budget.

For the blast-wave problem, the mesh refines around the expanding shock and de-refines as it dissipates; by the time the validator kills (~87% of runtime), the mesh has typically returned to the initial root layout, so the layout-match path takes effect and recovery is both fast and correct.
