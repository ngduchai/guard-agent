## Summary of changes

**Root cause analysis**: In iter_4 (and iter_2, iter_3), the validator reported `recovery_actually_resumed=False (ratio=recovery/Z_P=1.0)` — meaning the recovery attempt's walltime was essentially equal to a full failure-free run. This indicated that the restart logic was being skipped silently (returning to fresh simulation) despite checkpoints existing on disk. Likely causes: VeloC's session state didn't pick up checkpoints written by an earlier (killed) process, or the previous sidecar-based fallback was returning a version that VELOC_Restart_begin then rejected.

**Changes**:

1. **Removed the sidecar file machinery** (`/tmp/clamr_ckpt_latest_rank%d.txt` etc.) — replaced it with a direct directory scan of `/tmp/clamr_veloc_persistent` and `/tmp/clamr_veloc_scratch` to find the latest checkpoint version by parsing VeloC's file naming (`<name>-<rank>-<version>.dat`).

2. **Added a shadow checkpoint mechanism** (`clamr_shadow_checkpoint_write`/`_read`/`_probe_version`) — every time we call VELOC_Checkpoint, we also write a simple raw binary file at `/tmp/clamr_veloc_persistent/clamr_shadow_rank<rank>.bin` containing all the state (scalars + per-cell arrays + nsizes/ndispl). The file has magic-number header + magic-number trailer for integrity checking, and is written atomically via `.tmp` + `rename` + `fsync`.

3. **Shadow restore is tried FIRST during recovery**, completely independent of VeloC's restart machinery. This bypasses VeloC's session-tracking concerns and provides a reliable recovery path even if `VELOC_Restart_begin` fails. The shadow restore validates that all ranks have matching ncycle and the same numpe, then applies the state and rebuilds derived mesh data.

4. **VeloC restore remains as fallback** after shadow — if shadow files are missing/corrupt, we fall back to VELOC_Restart_begin with retry-on-earlier-version logic.

5. **Made `clamr_checkpoint` always write the shadow file**, regardless of whether VELOC_Init succeeded. This way, even if VeloC fails to initialize, future restarts can recover via shadow alone.

6. **Added retry on VELOC_Init failure** (single retry after 1 second sleep) for transient init failures after SIGKILL of a previous process.

7. **Added diagnostic marker files** at `/tmp/clamr_init_marker_rank*.txt` recording key restart decisions for post-mortem analysis on future iterations.

8. **`MPI_Barrier` at restart_done now runs unconditionally** to ensure rank synchronization regardless of restore path.
