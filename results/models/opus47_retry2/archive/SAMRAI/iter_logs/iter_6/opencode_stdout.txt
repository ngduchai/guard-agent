## Summary of fix

**Root cause identified from iter_4/iter_5 validate_stdout.txt:**

The validator reported `recovery_actually_resumed=False (ratio=recovery/Z_P=1.013)` — the recovery took 83.5s, essentially identical to a fresh full run (82.4s). The restart was *not* loading the saved checkpoint and was running the full simulation from `t=0` again. The checkpoint files were being written correctly (225 files visible to the validator), but recovery ignored them.

**Why:** The code called `VELOC_Restart_test(ckpt_name, 0)`. Per the VeloC API contract (see `/home/ndhai/usr/include/veloc.h:121-125`):

> `IN needed_version - maximum version to look for`  
> `returns - VELOC_FAILURE if no checkpoint < needed_version found, latest version < needed_version otherwise`

With `needed_version = 0`, no version is strictly less than 0, so the call always returns `VELOC_FAILURE`. The `latest_version > 0` check then falls through into the "no prior checkpoint found" branch and the run starts from scratch.

**Fix in `source/test/applications/LinAdv/main.cpp`:**

1. Pass `INT_MAX` instead of `0` to `VELOC_Restart_test` so it returns the latest version actually present on disk. Added `#include <climits>`.
2. Added a collective `MPI_Allreduce(MIN)` over each rank's `local_latest_version` so all ranks restart from the same version (avoids inconsistent per-rank physics state if one rank's persistent write was still in flight when the crash hit).

With the fix, restart will actually load the latest checkpoint, override `TimeRefinementIntegrator::start_time` to the checkpointed `loop_time`, overwrite the u-field on every patch, and resume the time loop from there — skipping the ~76% of work already completed before the kill. That should bring recovery well under the 0.85×Z_P gate-B threshold and the total kill+recovery well under the 1.488×Z_P per-app cap.
