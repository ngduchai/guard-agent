Let me summarize what I changed:

The previous attempt (iter_7) failed correctness because the restored state was potentially incomplete on each rank's hierarchy. The original logic had each rank read ONLY its own per-rank state file, but SAMRAI's load balancer can distribute level-0 patches to different ranks between the original run and the recovery run. When a patch's data was written to rank 2's state file but in recovery it belongs to rank 0, rank 0 wouldn't find it in its own file and the patch would remain at initial-condition values—corrupting the restored state.

**Fix made**: In `main.cpp`, the restart-state-reading loop now reads ALL N per-rank state files into a single global map keyed by (level, lo, hi). When `applyStateToHierarchy` iterates over rank-local patches and looks up by box geometry, it finds the right record regardless of which rank originally wrote it.

The other restart logic (start_time override, strip finer levels, sidecar bookkeeping fallback, coordinator-based version discovery) is preserved from iter_7—those parts were already correct and the wall-time gate had passed at 1.01x.
