Summary of changes:

**Root cause analysis from iter_6:**
- Validator showed `recovery_actually_resumed=False` with `ratio=recovery/Z_P=0.998` and walltime ratio `1.58x > 1.488x cap`.
- Recovery (80s) was essentially equal to a full fresh run (80.4s), meaning the restart code path was not actually skipping any work.
- The previous attempt's restart logic relied on `VELOC_Restart_test()` + cross-rank MPI_Allreduce to agree on a version. If any single rank's `VELOC_Restart_test` returned `FAILURE` (e.g., because a checkpoint write was in flight on that rank at crash time), the global agreement step fell back to "no restart" → fresh start → 80s recovery.

**Changes to `source/test/applications/LinAdv/main.cpp`:**

1. **Coordinator file (`/tmp/samrai_veloc_persistent/linadv-latest-version.txt`)** — rank 0 publishes the latest version atomically only AFTER all ranks have successfully written both their per-rank physics file and their VeloC checkpoint. On restart every rank reads (rank 0) + broadcasts this single integer, so all ranks agree on the version without depending on `VELOC_Restart_test` semantics.

2. **Plain-text bookkeeping sidecar (`linadv-bk-v{V}-r{R}.txt`)** — backup for the four scalars (iteration_num, loop_time, dt_now, num_failures) that does NOT depend on VeloC's internal metadata being healthy. Used to overwrite VeloC-loaded values if the latter look stale/zero (which would silently skip the `start_time` override).

3. **Deterministic absolute paths for the per-rank physics state file** (under `/tmp/samrai_veloc_persistent`) instead of `VELOC_Route_file` — VeloC's routing semantics for non-protected-memory files were not reliable for restart-time lookup.

4. **Collective `MPI_Allreduce(MIN)` on success flag** before publishing the coordinator, so a partial-failure checkpoint never advances the coordinator (and never causes a hung barrier).

5. **Guard against `ckpt_loop_time >= end_time`** — prevents the override from creating an empty time loop that would emit an initial-condition signature (which is what iter_3 evidently did — recovery exited in 0.17s with wrong output).

6. **Stderr `dbg(...)` instrumentation** at every decision point in the restart path so future failures are diagnosable from the captured logs.

The checkpoint write order is now: write per-rank physics file → write per-rank sidecar → `VELOC_Checkpoint` → collective `MPI_Allreduce` on success → rank 0 writes the coordinator. The coordinator therefore only ever points to a fully-completed, cross-rank-consistent checkpoint, and restart logic consults it as the single source of truth.
