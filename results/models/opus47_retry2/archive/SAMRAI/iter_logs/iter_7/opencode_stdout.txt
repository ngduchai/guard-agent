**Root cause analysis from iter_6:**
- Validator showed `recovery_actually_resumed=False` with `ratio=recovery/Z_P=0.998` and walltime ratio `1.58x > 1.488x cap`.

**Changes to `source/test/applications/LinAdv/main.cpp`:**

1. **Coordinator file (`/tmp/samrai_veloc_persistent/linadv-latest-version.txt`)** ŌĆö rank 0 publishes the latest version atomically only AFTER all ranks have successfully written both their per-rank physics file and their VeloC checkpoint. On restart every rank reads (rank 0) + broadcasts this single integer, so all ranks agree on the version without depending on `VELOC_Restart_test` semantics.

2. **Plain-text bookkeeping sidecar (`linadv-bk-v{V}-r{R}.txt`)** ŌĆö backup for the four scalars (iteration_num, loop_time, dt_now, num_failures) that does NOT depend on VeloC's internal metadata being healthy. 3. **Deterministic absolute paths for the per-rank physics state file** (under `/tmp/samrai_veloc_persistent`) instead of `VELOC_Route_file` ŌĆö VeloC's routing semantics for non-protected-memory files were not reliable for restart-time lookup.

4. **Collective `MPI_Allreduce(MIN)` on success flag** before publishing the coordinator, so a partial-failure checkpoint never advances the coordinator (and never causes a hung barrier).

5. **Guard against `ckpt_loop_time >= end_time`** ŌĆö prevents the override from creating an empty time loop that would emit an initial-condition signature (which is what iter_3 evidently did ŌĆö recovery exited in 0.17s with wrong output).

6. **Stderr `dbg(...)` instrumentation** at every decision point in the restart path so future failures are diagnosable from the captured logs.

The checkpoint write order is now: write per-rank physics file ŌåÆ write per-rank sidecar ŌåÆ `VELOC_Checkpoint` ŌåÆ collective `MPI_Allreduce` on success ŌåÆ rank 0 writes the coordinator. The coordinator therefore only ever points to a fully-completed, cross-rank-consistent checkpoint, and restart logic consults it as the single source of truth.
