Summary of the fix for this iteration:

The previous iteration (iter_5) timed out at 900s. The validate_stdout showed `recovery TIMEOUT after 109.6s` — the recovery binary ran for the full 110s (= 1.0x failure-free baseline) and was killed. This indicates the restart logic was failing silently, causing the binary to run from scratch instead of restoring from a checkpoint.

**Root cause**: `VELOC_Restart_test(CKPT_NAME, 0)` was being called with `needed_version=0`. Per the VeloC header documentation, this function returns "VELOC_FAILURE if no checkpoint < needed_version found, latest version < needed_version otherwise". Since checkpoint versions are positive (1, 2, 3, ...), no version is `< 0`, so the call always returns FAILURE. Every restart attempt was abandoned, and the binary ran from scratch — taking the full 110s = exactly the recovery timeout.

**Fix**: Changed `VELOC_Restart_test(CKPT_NAME, 0)` to `VELOC_Restart_test(CKPT_NAME, INT_MAX)` so any positive version we write is discoverable.

- `CHUNK_STEPS = 5000` (already set in iter_5; 7 chunks total over 33000 steps)
- All collective MPI calls are reached on all ranks regardless of code path
