1. **NUM_CHUNKS: 4 → 3** — fewer checkpoints means less overhead.

2. **Chunked run pattern: `pre yes post yes` (×4) → `pre yes post no` / `pre no post no` / `pre no post yes`** — This is the most impactful change. The original pattern did `lmp->init()` AND `Finish::end(1)` for EVERY chunk (4 full inits + 4 full finishes for the same total work). The new pattern does exactly ONE `lmp->init()` at the start and ONE `Finish::end(1)` at the end — functionally identical to the baseline's single `run TOTAL` command. This should reduce failure-free time from "much slower than baseline" to "essentially baseline + 2 checkpoint write calls".

3. **Added `MPI_Barrier(comm)` after each checkpoint** — defensive synchronization to prevent any rank-divergence during/after VELOC_Checkpoint.

4. **Removed `VELOC_Finalize` from exception paths** — VeloC may have collective internal state, so calling Finalize on only some ranks (when an exception only fires on a subset) could deadlock. `MPI_Abort` is sufficient to terminate all ranks.

5. **Changed `LAMMPSException` handler from `MPI_Barrier + MPI_Finalize + exit(1)` to `MPI_Abort(..., 1)`** — avoids potential deadlock if only some ranks throw the exception.
