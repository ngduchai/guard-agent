## Summary

**Root cause identified**: The previous attempt called `VELOC_Restart_test(CKPT_NAME, 0)`. Per the VeloC header documentation at `/home/ndhai/usr/include/veloc.h:124`:

> "returns VELOC_FAILURE if no checkpoint < needed_version found, latest version < needed_version otherwise"

Passing `0` asks for checkpoints with version `< 0`, which always returns failure. This caused restart to silently fail. The recovery then fell through to a cold replay of the full simulation, which timed out against the validator's recovery cap. The "MPI deadlock" diagnosis was a red herring — the issue was that recovery was doing 99s of cold-start work and getting killed.

**Fix** (`src/main.cpp:389`): Pass `INT_MAX` instead of `0` so VeloC returns the latest available checkpoint version.

With this fix:
- Recovery actually loads the checkpoint via `VELOC_Restart_begin` / `VELOC_Recover_mem`
- Per-rank atom state is restored from the saved buffer
- `reset_timestep` advances LAMMPS' clock to the checkpoint step
- Only the remaining ~5500 steps run (one chunk), well within the 98s recovery timeout
- Ratio should drop from ~1.74x to ~1.1-1.2x, passing both the 1.7x and 1.9x wall-time gates
