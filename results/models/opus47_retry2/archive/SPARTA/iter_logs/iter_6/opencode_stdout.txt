## Summary of changes

Based on iter_5's validator output (`output_correct=False` with max_rel_diff=4e-3, ratio=1.73x, but recovery DID exit 0 with 76MB of valid checkpoints), the prior code was actually performing restore but producing slightly diverged output. The simulation is fully deterministic (REFLECT boundaries use no RNG; particles preserve KE), so different output ⇒ subtle state inconsistency.

The most likely cause is **per-rank disagreement on which checkpoint version to restore**: with SIGKILL during a sync-mode write, one rank may have version N completely written while another only has N-1, so an uncoordinated `VELOC_Restart_test`+`VELOC_Restart_begin` per rank can have rank A restoring at step N while rank B restores at step N-1 — leading to inconsistent particle counts/positions and a slightly off trajectory.

### Edits to `src/update.cpp`

**Restart path (lines ~313-519):**
- **MPI-coordinate version selection**: each rank queries its own `VELOC_Restart_test`, then `MPI_Allreduce(MIN)` picks the highest version EVERY rank has. If any rank lacks a checkpoint at that version, MIN returns -1 → fall through to run-from-scratch.
- **MPI-coordinate validation**: size checks and recover-success checks are aggregated via `MPI_Allreduce(MIN)`; if any rank fails, all ranks roll back the attempt and try an older version.
- **MPI-coordinate ntimestep**: defensive `MPI_Allreduce(MIN)` on the restored `ntimestep` so the loop start is identical on every rank.
- **`VELOC_Mem_clear()`** after each failed `Restart_end(0)` and after the successful `Restart_end(1)`, so stale per-id registrations from a failed attempt cannot leak into the next attempt or into the writer.
- Renamed the restore-side sentinel `dummy_byte` → `restore_dummy_byte` to avoid name collision with the writer-side static.

**Writer path (lines ~580-612):**
- **`VELOC_Mem_clear()`** before each checkpoint's `Mem_protect` calls, defensively avoiding any cross-checkpoint registration leakage.
- **`MPI_Barrier(world)`** before `VELOC_Checkpoint` so all ranks enter the checkpoint with the same version number (`ntimestep` is already lockstep but the barrier prevents any rank racing ahead).

`veloc.cfg` (absolute /tmp paths, `mode=sync`, `max_versions=4`, `scratch_versions=4`) and `main.cpp` (`VELOC_Init`/`Finalize`) are unchanged from iter_5.
