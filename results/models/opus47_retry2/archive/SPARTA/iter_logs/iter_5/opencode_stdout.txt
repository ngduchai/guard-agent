## Summary of changes

The prior iter_4 attempt made **no source edits** (per inspection.md "NO SOURCE EDITS"); the on-disk code reflects iter_3, which failed with `recovery_actually_resumed=False (ratio=1.013, recovery≈110s≈Z_P)` — the restart was not actually skipping work.

Diagnosis: The validator confirmed `output_correct=True` even though `recovery_elapsed` ≈ failure-free runtime. Since this SPARTA test uses only specular `rr` boundaries (no RNG in the timestep loop), the simulation is fully deterministic given particle state — so correct output without time savings means **the restore branch failed silently** and recovery re-ran from step 0.

The most likely culprit was the **two-pass recover**: the iter_3 code called `VELOC_Recover_selective` for scalars first, then re-protected and called it again for particles. That left the second recover returning junk; particles came up as all-zero and the prune logic wiped `nlocal` to 0 (matching the 2 KB failure-prone checkpoint snapshot vs. 76 MB failure-free).

### Edits to `src/update.cpp` (restart path)

- **Single-pass recover**: query `VELOC_Recover_size` for all 4 ids first (metadata only), derive `nlocal` from the particle byte size, grow the buffer, `VELOC_Mem_protect` all 4 ids, then one `VELOC_Recover_selective` covering all of them.
- **Removed aggressive prune**: trust the saved state. The prior `parts[p].icell >= grid->nlocal` check could mistakenly wipe legit particles if grid->nlocal differed at restore time.
- **Up to 16 version probes** instead of 8, walking backwards via `probe_max = ckpt_v` so torn most-recent checkpoints fall through to older valid ones.
- **`probe_max = INT_MAX`** so restart_test boundary handling can't accidentally miss the latest version.
- **`SPARTA_CKPT_INTERVAL` 500 → 200** so the latest good checkpoint is closer to the kill point.
- Added explicit stderr diagnostics on every restart_test, restart_begin, size query, recover, and final restored state line for the next iteration's logs.

The checkpoint writer (loop body around line 564) was already correct and is unchanged: it captures `ntimestep`, `laststep`, `nlocal`, and the per-rank `particle->particles` array via `VELOC_Mem_protect` + `VELOC_Checkpoint`.

`veloc.cfg` and `main.cpp` remain as iter_3 left them (absolute /tmp paths, `mode=sync`, `max_versions=4`, `scratch_versions=4`, VELOC_Init result logged).
