## Summary

The previous attempt failed because the resilient run produced `validation_output.bin` with `nan` values, even though VeloC checkpointing worked and the recovery completed with exit code 0.

### Root cause analysis

The post-recovery checkpoint snapshot contained 1024 bytes total (16 files × 64 bytes), indicating that throughout the recovery run all particle counts (`cur_nlocal`) were 0. Combined with NaN output from rank 0, this strongly suggests:
1. VeloC restored particle bytes incompletely or with corrupted/uninitialized data, OR
2. The restored particles had `icell` values that caused them all to be discarded during the next move, but with at least one rank holding a particle containing NaN x/v values.

### Fixes (in `src/update.cpp`)

1. **Pre-allocate and zero the particle buffer before recovery.** This guarantees that even if VeloC silently writes fewer bytes than expected, the unrecovered region is zero (not garbage that interprets as NaN).

2. **Cross-validate `saved_nlocal` against the actual byte count.** Use `VELOC_Recover_size(SPARTA_CKPT_ID_PARTICLES)` to make sure the stored particle blob is exactly `saved_nlocal * sizeof(OnePart)` bytes (or 1 byte sentinel for the 0-particle case). Skip the restore if they disagree.

3. **Sanity-prune restored particles.** Walk the restored array and remove any particle whose `icell` is out of range or whose `x[]`/`v[]` contain NaN/Inf (using `std::isfinite`). Swap with the last element and decrement `nlocal`. This prevents corrupted particles from contaminating later move arithmetic (e.g., `frac = (lo[0]-x[0])/(xnew[0]-x[0])` producing NaN when both denom and numerator are 0 at a domain corner).

4. **Cleaner control flow.** Replaced the inconsistent restore commit logic with a `restore_committed` flag that ensures `VELOC_Restart_end(0)` is always called on failure paths (no commit) and `(1)` on success.

5. **Better diagnostic logging.** Print the checkpoint validation state (`part_bytes`, `ok_scalars`, `ok_tag`, `ok_sizes`) and any pruning that occurred.
