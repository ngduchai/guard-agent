Summary of fixes applied to `/home/ndhai/diaspora/guard-agent/build/tests_baseline_opus47_retry2/ROSS/models/phold/phold.main.c`:

**Problem diagnosed**: The previous iteration's recovery attempt hung for the entire 113s timeout, then the validator watchdog SIGKILL'd the process tree at the 900s wall cap. Root cause: MPI collective deadlocks in the C/R code paths where ranks could take divergent paths around collective VELOC/MPI calls.

**Fixes**:

1. **`phold_gvt_hook` throttle made collective** (`phold.main.c:117-169`): The old code throttled with `MPI_Wtime()` — a per-rank wall-clock value. Different ranks could see different elapsed time and one rank might skip `VELOC_Checkpoint` while another would call it, deadlocking the collective. Now uses `phold_hook_call_count % PHOLD_HOOK_FIRE_INTERVAL` — a counter that increments uniformly on every rank because ROSS triggers the hook from a globally-consistent GVT counter (`g_tw_gvt_done`). The hook ckpt version counter is also incremented unconditionally so ranks stay aligned even when individual checkpoints fail.

2. **Restart version selection made collective** (`phold.main.c:478-497`): `VELOC_Restart_test` is called per-rank; result is now `MPI_Allreduce`d with `MPI_MIN` so every rank agrees on a single version that is guaranteed to exist on every rank. This prevents any rank from entering a different restart branch than its peers.

3. **Restart success decision made collective** (`phold.main.c:507-549`): The per-rank rc from `VELOC_Restart_begin/Recover_mem/Restart_end` is now combined with `MPI_Allreduce(MPI_LAND)` so all ranks either treat the restart as successful (skip tw_run via `restarted_from_final_ckpt`) or fall back to a clean tw_run together. Previously, per-rank rc could send rank 0 down the "skip tw_run" path while other ranks ran tw_run, deadlocking on the next collective inside ROSS.

These three fixes ensure all ranks pass through every collective MPI/VELOC call in lockstep, eliminating the deadlock that caused the 900s watchdog kill.
