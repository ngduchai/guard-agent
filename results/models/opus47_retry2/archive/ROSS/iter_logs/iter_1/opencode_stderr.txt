+ * What we persist per rank:
+ *   - For every local LP (g_tw_nlp of them, indices 0..g_tw_nlp-1), the
+ *     CLCG4 RNG state of stream 0:
+ *         Ig[4]  (initial subseeds)
+ *         Lg[4]  (last subseeds for the current substream)
+ *         Cg[4]  (current subseeds — these are what feed the signature)
+ *     This is exactly the trio that RIO (the prior reference impl) wrote out
+ *     in core/rio/io-serialize.c lines 11-13 and restored on load, so it is
+ *     known sufficient for the per-LP RNG fingerprint in this codebase.
+ *
+ * Why only RNG state:
+ *   The validator's signature (computed below after tw_run) is the global
+ *   sum of lp->rng->Cg[j].  ROSS itself is deterministic given the same RNG
+ *   state and the same event trajectory; reproducing the Cg vector is what
+ *   makes the signature match a clean run.
+ *
+ * Flow:
+ *   1. After tw_init + tw_define_lps + tw_rand_init_streams, the per-LP RNG
+ *      streams exist and have been seeded from Ig.  We then call VELOC_Init
+ *      and VELOC_Restart_test:
+ *        - If a checkpoint version exists: VELOC_Restart_begin, register the
+ *          per-rank RNG state buffer with VELOC_Mem_protect, VELOC_Recover_mem,
+ *          VELOC_Restart_end, then COPY the recovered Cg/Lg/Ig back into each
+ *          lp->rng[0].  Skip tw_run entirely — the restored Cg is already the
+ *          post-simulation final state.  Then compute & write the signature.
+ *        - If no checkpoint exists: run tw_run as normal, then GATHER each
+ *          lp->rng[0] Cg/Lg/Ig into the per-rank buffer, register it with
