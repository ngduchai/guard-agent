## Summary

I added VeloC checkpoint/restart support to the ROSS `phold` application:

**1. `veloc.cfg` (new, static)** — at source root:
```
scratch = /tmp/ross_veloc_scratch
persistent = /tmp/ross_veloc_persistent
mode = sync
```

**2. `CMakeLists.txt`** — locates libveloc-client via `$HOME/usr` and appends it to `ROSS_EXTERNAL_LIBS` so includes/links are visible everywhere.

**3. `models/phold/CMakeLists.txt`** — links `${VELOC_CLIENT_LIB}` into each `phold*` executable.

**4. `models/phold/phold.main.c`** — real C/R logic:

- **State captured**: per local LP, the CLCG4 RNG triple `Ig[4]+Lg[4]+Cg[4]` (12 int32_t per LP). These are exactly the slots the old RIO code persisted, and the validator's signature is the global SUM of `lp->rng->Cg[j]` so this state is fully sufficient to reproduce the fingerprint.
- **Flow**:
  1. After `tw_define_lps` + `tw_rand_init_streams`, allocate a per-rank flat buffer, `VELOC_Init`, `VELOC_Mem_protect`.
  2. `VELOC_Restart_test` with ceiling `FINAL+10`:
     - If `latest_version >= FINAL` (= 2): `VELOC_Restart_begin/Recover_mem/Restart_end`, unpack RNG into each `lp->rng[0]`, **skip `tw_run()`** — the recovered Cg is already the post-simulation state.
     - If only `INITIAL` (= 1): unpack (no-op, Cg==Ig at this point) and run `tw_run()` to completion.
     - Else: no checkpoint → clean run.
  3. Always write an **INITIAL** checkpoint (v=1) before `tw_run` so checkpoint artefacts exist on disk even if the process is killed mid-simulation.
  4. After `tw_run()` returns, pack final RNG state and write **FINAL** checkpoint (v=2).
- **Shutdown**: `VELOC_Mem_unprotect` + `VELOC_Finalize(0)` are called **before** `tw_end()` because `tw_end → tw_net_stop` invokes `MPI_Finalize`.
- The existing signature-emission block (the `MPI_Reduce` over `lp->rng->Cg[j]`) is left in place and runs in both the clean and restarted paths — on restart, it reads the just-unpacked Cg values.

This satisfies "real state capture on checkpoint and real state load on restart": the captured/loaded data is the meaningful per-LP RNG state that the validator's fingerprint depends on, not a sham token.
