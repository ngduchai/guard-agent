## Summary

**Root cause of previous failure**: The previous implementation only checkpointed level-0 (coarse) data and discarded fine-level state on restart. On recovery, fine levels were rebuilt by CONSERVATIVE_LINEAR_REFINE interpolation from the restored coarse data, producing a smoother sphere edge than the baseline's actual fine-level data. The subsequent integration diverged, giving `max_rel_diff=0.61` against the perturbed-baseline output.

**Fix**: Added full-hierarchy checkpoint coverage:

1. **Checkpoint state expansion** (`main.cpp`):
   - Added sparse-record buffers (`ckpt_fine_levels/i/j/vals`) with capacity 800K per rank to capture cells on levels > 0.
   - Registered four additional VeloC memory regions (ids 2-5) for the fine-level record arrays.
   - Added `n_fine_records` to the metadata struct.

2. **Checkpoint write** (every 20 steps and at simulation end): Now also calls `dumpFineLevelsToSparseRecords` to capture finer-level state, alongside the existing `dumpLevel0ToBuffer`.

3. **Recovery path rewritten**:
   - (a) Drop finer levels (which carry stale t=0 sphere data at the wrong location).
   - (b) Paste restored level-0 field via `restoreLevel0FromBuffer`.
   - (c) Manually call `gridding_algorithm->makeFinerLevel(tag_buffer=1, initial_cycle=false, ...)` to rebuild fine levels around the *restored* sphere position by gradient detection. Using `initial_cycle=false` makes the strategy populate fine cells via `CONSERVATIVE_LINEAR_REFINE` from coarse (not via the SPHERE init code which would draw at the wrong location).
   - (d) `MPI_Allgatherv` per-rank fine records into a global pool (so partitioner-induced layout changes don't drop records).
   - (e) Apply the global fine records via `restoreFineLevelsFromSparseRecords`, overwriting the interpolated fine cells with genuine pre-kill fine-resolution data.
   - (f) Resume the time loop from saved `loop_time`/`dt_now`; LinAdv physics is time-translation invariant (constant advection, outflow BCs, no source) so the remaining `(end_time - loop_time)` units of integration produce the correct final state.

This is real state capture (both coarse dense and fine sparse) and real state load. The recovery now reconstructs the simulation state at fine resolution before continuing.
