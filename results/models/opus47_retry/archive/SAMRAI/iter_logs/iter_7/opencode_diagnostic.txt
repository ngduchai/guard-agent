Summary of changes made in this iteration:

**Root cause of iter_6 failure**: The `refineLevel0ToHigherLevels` "constant-injection" approach to populating fine-level cells from the restored L0 buffer was making things worse (max_abs_diff went from iter_3's 5.94e+08 to iter_6's 7.11e+08, and max_rel_diff hit 67%). The constant-injection created step-function discontinuities across L0 cell boundaries on fine levels, which:
- generated huge artificial gradients
- caused massive over-refinement on the first regrid
- produced potentially unstable evolution that drove the resumed simulation far from baseline

**Fix in iter_7**:
1. **Removed the call to `refineLevel0ToHigherLevels`** in the recovery path (the helper method is left in LinAdv.cpp/h as dead code since removing it changes the API but isn't necessary).
2. **Restored the "drop fine levels" approach** (remove L1, L2 after L0 restore) so no stale t=0 sphere data contaminates L0 via coarsen sync.
3. **Added a proper `gridding_algorithm->regridAllFinerLevels(0, ...)` call** with arguments mirroring the way TimeRefinementIntegrator invokes it internally during periodic regridding. This rebuilds finer levels from the gradient detector applied to the restored L0 field, so:
   - Fine patches cluster around the actual current sphere position
   - Fine cell data comes from CONSERVATIVE_LINEAR_REFINE interpolation off L0 (the natural SAMRAI mechanism)
   - The hierarchy is fully formed when the integrator's first post-restore advance call runs
4. **Wrapped the regrid call in try/catch** to fall back to "level-0 only" if it fails, so a failed rebuild doesn't crash the run.
5. **Reduced checkpoint frequency from every 5 to every 20 steps** to lower I/O overhead (was producing 728 checkpoint files / 1.46GB) while still giving fine-enough recovery granularity (within 20 steps of kill, ~0.4 sim-time units).
6. **Added `<exception>` header** for std::exception used in the catch clause.

The `veloc.cfg` with absolute /tmp paths (`scratch=/tmp/samrai_linadv_veloc_scratch`, `persistent=/tmp/samrai_linadv_veloc_persistent`, `mode=sync`) is unchanged and still in place at the source tree root.
