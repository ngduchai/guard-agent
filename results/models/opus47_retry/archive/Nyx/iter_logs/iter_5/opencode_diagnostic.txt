2. **nyx_main.cpp**:
   - Added `NYX_VELOC_TRAILER` magic (0x454E444E = 'NDNE' in mem) and `NYX_VELOC_RECOVERY_FALLBACK_WINDOW` (32).
   - Checkpoint blob layout is now `[Header][state][trailer_magic]` so we can detect truncated/partial writes.
   - Restart logic now walks backward through versions if the latest is corrupt:
     - All ranks agree on `latest` via `ReduceIntMax` 
     - For each version V from latest down to (latest-32):
       - Call `VELOC_Restart_begin(name, V)` on all ranks
       - Consensus reduce on Restart_begin success
       - If all succeeded, call `Recover_size`, `Mem_protect`, `Recover_mem`
       - Validate header magic, trailer magic, and total size consistency
       - Consensus reduce on validation success
       - If all ranks agree: use this version
       - Otherwise: `Restart_end(0)` and try previous version
   - Checkpoint cadence increased from 25 to 50 (fewer mid-write kill windows, less I/O).
   - Retention changed from "keep only newest" to "keep newest 4 versions" so the fallback has older complete checkpoints available.
   - Removed unused `prev_version` variable.

The key fix for iter_4's failure: when VeloC recovered a corrupted blob (likely killed mid-write), we now detect the corruption (magic + trailer + size cross-check) and walk backward to find a valid earlier checkpoint, rather than falling through to a fresh-from-zero init that times out.
