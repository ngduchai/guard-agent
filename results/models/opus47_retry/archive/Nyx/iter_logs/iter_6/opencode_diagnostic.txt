Summary of changes to `Source/Driver/nyx_main.cpp`:

1. **Added `<fcntl.h>` include** for POSIX `open()`.

2. **Added sidecar checkpoint subsystem** (helper functions `nyx_sidecar_path/write/read`):
   - Per-rank file at `<dir>/nyx_sidecar_r<NNNN>.ckpt` in both VeloC scratch and persistent directories.
   - Atomic write: write to `.tmp`, `fsync`, then `rename` to final name. POSIX guarantees the final name is either the previous complete content or the new complete content, never a half-written file. This eliminates the mid-write corruption that caused all earlier iterations to silently fall back to fresh init.
   - File header with magic + payload size lets us detect torn writes if rename itself races.

3. **Recovery path: try sidecar FIRST, VeloC walk-back second.**
   - Read sidecar from persistent (preferred) and scratch dirs.
   - Validate the embedded NyxVelocHeader magic, trailer magic, and size consistency.
   - Reach cross-rank consensus via `ReduceIntMin` — only resume if EVERY rank has a valid sidecar.
   - On consensus success, commit it and skip the VeloC walk-back entirely.
   - `VELOC_Restart_test` is still called unconditionally to keep collective call balance even when sidecar already succeeded.

4. **Checkpoint path: write the sidecar after each successful `VELOC_Checkpoint`** with the same blob bytes.

5. **Reduced checkpoint cadence from 50 → 20** coarse steps. With atomic-rename writes there's no mid-write corruption risk, so more frequent checkpoints just mean less work to redo on recovery (better `recovery/Z_P` ratio).

The existing AMReX-bookkeeping patch (`setCumTime`, `setLevelSteps`, `setLevelCount`, `setDtLevel`, `setTimeLevel`) is unchanged and runs after `Amr::init()` to ensure the `coarseTimeStep` loop resumes from the restored physical time rather than t=0.
