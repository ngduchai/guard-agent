Summary of changes I made to fix the previous attempt:

**Root causes identified from iter_6 logs:**
1. The sidecar-file resume succeeded but then segfaulted during scatter/coarseTimeStep, likely from per-rank MFIter-order packing failing when DistributionMap/BoxArray changed between checkpoint and restart due to AMR regrids.
2. The previous attempts also tried to recover from VeloC with bytes-mismatch (`magic=0x81ee425d expected=0x4e594f58`) — corrupt recovered data was treated as garbage and the run fell through to fresh init (slow recovery).

**Changes to `Source/Driver/nyx_main.cpp`:**

1. **Removed the entire sidecar mechanism** (`nyx_sidecar_path/write/read`, `NyxSidecarFileHdr`, sidecar resume path, sidecar write block).  Sidecars in VeloC dirs violate the "no side-car files in VeloC dirs" gate.

2. **Replaced per-rank MFIter pack/unpack with layout-independent dense-per-level storage** (`nyx_dense_size`, `nyx_dense_index`, `nyx_dense_pack_owned`, `nyx_dense_scatter`).  Each rank writes its owned cells into a dense (Domain × nComp) array; a single `MPI_Allreduce(MPI_SUM, MPI_DOUBLE)` merges across ranks so every rank ends up with the complete dense state.  On restart, every rank reads the complete dense buffer and scatters by global cell coordinate — DistributionMap and BoxArray changes are tolerated.

3. **Added per-level Domain box to header** (`dom_lo_x/y/z`, `dom_hi_x/y/z`) so the restart can validate dimensional consistency before scattering, and reject mismatched levels safely (returning 0 from `scatter_one_level`).

4. **Added CRC32 of the payload** (`nyx_crc32`, `payload_crc32` header field).  On restart, recompute and compare; reject corrupt blobs even when magic+trailer happen to look valid.

5. **Resume now scatters level 0 first, then triggers `amrptr->regrid(0, restored_cum_time, false)`** to create refined levels from the restored coarse state via FillCoarsePatch, then scatters the refined levels into the newly-created MultiFabs.  This handles the common AMR case where the freshly-init'd hierarchy has only level 0 at t=0 but the checkpoint had refined levels.

6. **Sets per-level StateData time BEFORE the regrid call** so the new AmrLevel constructors and `Nyx::init()` (no-arg variant, called inside regrid for new levels) see the restored time rather than 0 — otherwise `computeNewDt` would clamp dt to `stop_time - 0` and burn through the simulation in one giant step.

7. **`VELOC_Checkpoint_wait()` after each `VELOC_Checkpoint`** to flush any internal queue before the next coarseTimeStep, so the on-disk persistent file is durable before the next potential kill.

8. **Increased fallback walk-back window to 64** (was 32) and added CRC to the rejection criteria so corrupt-but-magic-valid blobs are also rejected and walked past.

9. **If no level scatters successfully (all geometry mismatch), reset `veloc_resumed_state=false`** so the run falls through to fresh init cleanly instead of advancing cumTime with empty state.

**`veloc.cfg`** kept with absolute `/tmp/nyx_veloc_{scratch,persistent}` paths and `max_versions = 8` to support walk-back across multiple complete versions.
