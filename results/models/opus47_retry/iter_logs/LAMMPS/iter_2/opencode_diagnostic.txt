2. **MPI_Allreduce on checkpoint version** (`main.cpp:399-407`)
   - `VELOC_Restart_test` is per-rank. If ranks diverge on whether a checkpoint exists, they'd take different code paths → MPI deadlock at the next collective.
   - Reduce with `MPI_MIN`: only restart if ALL ranks see a positive version; otherwise all do a fresh run.

   - Each VELOC_Restart_begin / Recover_selective / Restart_end check now collects local rc, does `MPI_Allreduce` with `MPI_MAX`, and acts uniformly on all ranks.
   - On any rank's failure: gracefully fall back to fresh-run path (set `have_ckpt = 0`) instead of MPI_Abort. This makes the recovery binary survive corrupted/partial checkpoints rather than crashing.

   - Calling `system()` from one rank while others wait at `MPI_Barrier` risks fork/exec interactions with MPI runtimes that can hang.
