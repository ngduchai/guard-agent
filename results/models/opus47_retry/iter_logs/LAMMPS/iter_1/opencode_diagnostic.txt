3. **Real state capture**: At every CKPT_CHUNK_STEPS (1000) simulation steps, snapshots the LAMMPS atom data (per-rank tag/type/mask/image/x/v arrays plus metadata) into VeloC-protected buffers and calls VELOC_Checkpoint. This is genuine state — losing it means losing simulation progress.

5. **Cleanup on success**: Deletes checkpoint files when the simulation completes normally so that a subsequent fresh run isn't misled by stale checkpoints.
