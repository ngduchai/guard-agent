## Summary of changes

I diagnosed the iter_3 failure from `validate_stdout.txt`:
- **Output correct**, **checkpoints written** (24 files, 25 MB), but `recovery_actually_resumed=False (ratio=1.001)` — recovery took 109.9 s vs Z_P 109.8 s, meaning restart didn't actually short-circuit any work.

**Root cause** (in `src/main.cpp`): only rank 0 ever wrote a file at its `VELOC_Route_file` path. Ranks 1–3 called `VELOC_Route_file` (so the data file was "registered" with VeloC) but never created the file there. When `VELOC_Checkpoint_end(1)` ran, those ranks' file copies were missing, the version wasn't registered globally for them, and on restart `VELOC_Restart_test` returned 0 for some rank — `MPI_MIN` reduced to 0, `have_ckpt=0`, and the recovery silently fell back to a full fresh run.

**Fix** (single file: `src/main.cpp`):
- After `write_data` (single shared file produced only on rank 0), broadcast the file size and bytes to every rank and have every rank write a COMPLETE copy at its own per-rank `VELOC_Route_file` path. This guarantees `VELOC_Checkpoint_end(1)` succeeds on every rank, so the version is registered globally and is visible to `VELOC_Restart_test` after a crash.
- On restart, every rank's routed path holds the complete data file. Rank 0 copies it back to `LMP_DATA_LOCAL` for `read_data` (which only opens on rank 0 and broadcasts internally).
- Reduced `CKPT_CHUNK_STEPS` from 5000 → 3000 so the per-checkpoint gap is ~10 s, giving the killer a tighter window and a smaller "wasted" tail.
- Every per-rank rc is `MPI_Allreduce`d before any divergent control flow; every collective call (`VELOC_Restart_begin/end`, `VELOC_Checkpoint_begin/end`, `lammps->input->one`, `MPI_Bcast`, `MPI_Allreduce`) is reached by every rank on every code path.
- Added stderr diagnostics ("[veloc] Restart_test:…", "[veloc] Restart phase:…", "[veloc] Checkpoint v=… …") so a future failed iteration can be diagnosed in seconds.

The static `veloc.cfg` (absolute `/tmp/lammps_veloc_scratch`, `/tmp/lammps_veloc_persistent`, `mode = sync`) and the CMake VeloC link were already in place and were not modified.
