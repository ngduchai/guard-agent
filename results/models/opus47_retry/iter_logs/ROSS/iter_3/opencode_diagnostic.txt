**Key changes to `phold.main.c`**:

1. **Reduced checkpoint frequency** — `tw_trigger_gvt_hook_every(2000)` (was 50). This drops failure-free baseline overhead significantly (~24 ckpts vs ~12000).

2. **Per-LP RNG snapshot at every GVT hook** — saves `Cg/Lg/Ig` for both `lp->rng` (model) and `lp->core_rng` (tiebreaker) for all LPs. These are the deterministic state that drives PHOLD's final Cg-sum signature.

3. **Per-rank event-queue snapshot at every GVT hook** — drains `pe->pq` to record each pending event's `(recv_ts, dest_gid, src_gid)`, then re-enqueues to leave the queue unchanged.

4. **Two-tier version scheme** — intermediate ckpts use versions `1..999999`, final-state ckpt uses `>= 1000000`. On restart `VELOC_Restart_test` picks the latest; we can tell which kind it is from the version number.

5. **Restart paths**:
   - **Final ckpt loaded** → skip `tw_run()` entirely (the loaded per-rank Cg sum is already the final answer).
   - **Intermediate ckpt loaded** → set `g_veloc_have_intermediate=1`. In `phold_init` we short-circuit (no events, no rng draws). In the LP `pre_run` callback (which fires AFTER `tw_init_lps` but BEFORE the scheduler loop, inside `tw_sched_init`), we drain whatever ended up in `pe->pq`, copy each LP's saved RNG state back onto the live LP, and re-inject the saved events with their original timestamps and destinations. From there `tw_run` resumes simulation from where the previous run died.
   - **No ckpt** → fresh full run, write FINAL ckpt at the end.

6. **veloc.cfg** unchanged from iter_2 — absolute `/tmp` paths, sync mode.
