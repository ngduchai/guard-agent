Summary of fixes applied to `src/app_lattice.cpp`:

1. **`veloc_try_recover()`** — Fixed MPI deadlock. Previously each rank independently chose a checkpoint version (or returned early on per-rank failure), and inconsistent decisions across ranks caused the collective `VELOC_Restart` to hang. Now:
   - All ranks compute their local available version, then `MPI_Allreduce(MPI_MIN)` agrees on a single version that exists on every rank.
   - If any rank lacks a checkpoint (agreed_v ≤ 0), all ranks skip recovery uniformly.
   - After `VELOC_Restart`, the success/failure is reduced with `MPI_LAND` so every rank treats it as success or failure consistently.

2. **`iterate()` post-recovery early return** — The `if (time >= stoptime) return;` check was per-rank. If one rank's recovered time crossed stoptime and others' didn't, the returning rank would skip all downstream collectives (`timer->barrier_start`, `comm->sector`, etc.), deadlocking the rest. Replaced with an `MPI_LAND` collective so all ranks agree before returning.

3. **`veloc_do_checkpoint()` throttle** — The 5-second throttle used per-rank `MPI_Wtime()`, so different ranks could make different skip/checkpoint decisions, leading to a deadlock on the collective `VELOC_Checkpoint`. Replaced with an `MPI_LAND` over the local skip decision: only skip when every rank wants to skip, otherwise all ranks checkpoint.

These changes preserve the real state capture/restore (no shortcuts) while keeping every MPI-collective code path in lockstep across ranks, eliminating the deadlock that timed out iter_2 during recovery.
