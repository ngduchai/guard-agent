Summary of changes:

**`veloc.cfg`**: Added `max_versions = 1` and `scratch_versions = 1` to instruct the VeloC backend to keep only the latest checkpoint version on persistent and scratch storage. The prior cfg had no retention limit, so every checkpoint version accumulated until the validator's 10.7 GB disk-guard killed the run.

**`examples/laminar/bstep2d/bstep2d.cpp`**:
- Bumped `CKPT_INTERVAL` from 200 → 2000 iterations. The previous cadence produced hundreds of checkpoint writes per run; at ~17 MB each that blew through the disk budget. The new cadence still lands many checkpoints across the failure-free runtime (the validator first polls for files at ~50% of runtime, after which dozens of checkpoints will already exist) while keeping per-run write volume small.
- Added `parse_veloc_cfg()` + `unlink_prior_versions()` that, after each successful `VELOC_Checkpoint`, walk the scratch and persistent directories and remove any files matching `<ckpt-name>-<rank>-<v>*` for `v < saveIt`. This is belt-and-braces on top of the cfg retention knobs in case the backend's purge lags.
- Added the needed `<cstring>`, `<string>`, `<dirent.h>`, `<sys/types.h>`, `<unistd.h>` includes.
