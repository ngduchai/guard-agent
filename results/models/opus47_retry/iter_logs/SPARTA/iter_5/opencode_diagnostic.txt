## Summary

**Root cause of iter_4 failure**: Recovery produced NaN output and tiny (704B) checkpoints. The two-phase `VELOC_Mem_protect` + `VELOC_Recover_size` approach had a chicken-and-egg problem: the particle count (id=2) needed to be known to size the buffer correctly before recovery. Even with a "phase-1 recovery then phase-2 protect" workaround, the restored particle data ended up empty/corrupt, so post-recovery the simulation ran with nlocal=0 and produced empty validation output.

**Fix**: Replaced the mem-protect approach with **direct file I/O via `VELOC_Route_file`** — a simpler pattern that's part of the standard VeloC API:

### Checkpoint write (per rank, every 2000 steps):
1. `VELOC_Checkpoint_begin(name, version)` — opens a checkpoint version
2. `VELOC_Route_file("sparta_snap_rankN.bin", routed)` — returns the in-scratch path
3. `fopen + fwrite header + fwrite particles + fflush + fsync + fclose` — writes a self-describing snapshot
4. `MPI_Allreduce` of success flag → `VELOC_Checkpoint_end(write_ok_all ? 1 : 0)` — atomically commits or discards the version (in sync mode also copies to persistent)

### Restore (once, on first entry to `Update::run`):
1. `VELOC_Restart_test` → `MPI_Allreduce(MIN)` to agree on a version across ranks
2. `VELOC_Restart_begin(name, v)` + `MPI_Allreduce(MIN)` of success
3. `VELOC_Route_file(...)` to get the snapshot path
4. `fopen + fread header (check magic/version) + grow particle array + fread particles`
5. Cross-rank success reduction; if all OK commit `ntimestep`, `particle->nlocal`, recompute `nglobal`, invalidate `sorted`; if any failure, roll back to fresh state
6. `VELOC_Restart_end(success)` 

### Snapshot layout:
A fixed-size 32-byte header (magic + format version + ntimestep + nlocal + pad) followed by `nlocal` raw `Particle::OnePart` structs. The format-version field future-proofs against layout changes; the magic prevents loading garbage.

The simulation loop already correctly handles `cumulative_target`-based step accounting so the post-restore loop runs only the remaining steps.
