## Summary of Changes from iter_2 to iter_3

The previous attempt's recovery process crashed with a SEGFAULT in `MallocPlus::memory_realloc_all` called from `mesh->resize()` and `state->resize()`. The crash signature showed `realloc()` being called with a wild pointer, indicating heap corruption.

**Root cause hypothesis**: `mesh->resize(saved_ncells)` and `state->resize(saved_ncells)` both call `memory_realloc_all(saved_ncells)` which iterates over **all** ~40+ entries in MallocPlus and reallocates each to `saved_ncells`. This blew up some entries that should have different sizes (like ncells_ghost-sized face arrays), corrupting heap state. The realloc loop then encountered a corrupted entry with a wild `mem_ptr` and crashed.

**Fix**:
1. **Replaced `mesh->resize`/`state->resize` with per-name `memory_realloc(saved_ncells, "name")`** — only resize the 6 specific arrays we restore (`i`, `j`, `level`, `H`, `U`, `V`). Other MallocPlus entries are left untouched and will resize themselves naturally in `calc_neighbors_local` and `do_calc` as needed.

2. **Added file-size cross-check** — verify the file size on disk matches what we expect from header+arrays+footer to catch corruption that magic+footer didn't detect.

3. **Bumped format version from 2 to 3** — invalidate any pre-existing partial-format checkpoints.

4. **Tightened ncells sanity check** — reject implausibly large values (>50M cells).

5. **Added per-rank diagnostic prints** with rank ID for easier debugging.

6. **NULL-pointer check after realloc** — bail out if memory_realloc somehow returned NULL pointers, before doing any fread that would corrupt heap.
