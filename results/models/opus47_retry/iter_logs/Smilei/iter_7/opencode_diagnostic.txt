**Root cause of the iter-6 segfault (exit code 139)**:

The previous attempt added `canonical_em_fields()` which included `EM->rhoold_` as the 14th field. However, looking at `ElectroMagn::initElectroMagnQuantities()` in `ElectroMagn.cpp:96-118`, **`rhoold_` is NOT initialized to NULL** (only set in 2D/3D subclass constructors under specific conditions). In the 1D test namelist, `rhoold_` contains uninitialized garbage memory after construction.

When the checkpoint code then called `fld->data()` on this garbage pointer, the process segfaulted.

**Fix** in `src/Smilei.cpp`:

1. **Removed `EM->rhoold_` from `canonical_em_fields()`** (line 527-552) — leaving only the 13 fields that `initElectroMagnQuantities()` explicitly NULL-initializes: `Ex_, Ey_, Ez_, Bx_, By_, Bz_, Bx_m, By_m, Bz_m, Jx_, Jy_, Jz_, rho_`.

2. **Bumped the magic** from `SMILEIC6` to `SMILEIC7` (line 135) so any stale 14-field checkpoint files from iter-6's broken format are rejected on load.

3. `veloc.cfg` is already in place with absolute paths to distinct `/tmp` directories (`/tmp/smilei_veloc_scratch` and `/tmp/smilei_veloc_persistent`) and `mode = sync`.
