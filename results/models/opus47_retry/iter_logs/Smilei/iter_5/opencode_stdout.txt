Summary of changes for iter-5:

1. **Atomic checkpoint writes**: Save now writes to `<routed>.tmp`, fsyncs, then `rename(2)` atomically to the final `routed` path. Any kill mid-write leaves only a `.tmp` file (which will be ignored by VeloC and Restart_test), preventing corrupted checkpoints from being selected on recovery.

2. **Trailing magic for integrity**: Each checkpoint file now ends with the same `SMILEI_CKPT_MAGIC` as the header. The load function reads and verifies this trailer; mismatch or short-read marks the file as corrupted.

3. **Fallback on load failure**: If `smileiCheckpointLoad` fails for the latest version (e.g., due to filesystem inconsistency), the recovery loop steps down by `CKPT_EVERY` and tries the older checkpoint, up to 32 attempts. This guarantees that any committed (renamed) checkpoint can be loaded.

4. **MPI collective agreement on load success**: The load function does `MPI_Allreduce(MIN)` on per-rank load success so all ranks either succeed together or fail together — preventing inconsistent partial restarts.

5. **MPI collective agreement on VELOC_Restart_begin**: Similar `MPI_Allreduce(MIN)` on `begin_ok` so a single-rank failure causes all ranks to bail out coherently.

6. **Reduced checkpoint cadence**: Every 200 iterations instead of 100, cutting write overhead in half while still keeping recovery work bounded (~200 iters re-execution worst case).

7. **Better error diagnostics**: Write/open failures now log `errno` and `strerror(errno)`, making future debugging easier.

8. **Magic bump to "SMILEIC5"**: Rejects any stale checkpoints from iter-4 (which used the old format without trailer).
