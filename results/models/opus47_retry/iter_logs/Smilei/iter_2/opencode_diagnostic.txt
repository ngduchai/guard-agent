Summary of changes for this iteration:

**Root cause analysis from `validate_stdout.txt`:**
- Failure-free run: exit 139 (SIGSEGV) with stack trace ending in `_IO_fwrite` 
- Both the failure-prone and failure-free legs produced checkpoint files (proving VeloC was initialised and writing), but the binary then crashed
- The crash was downstream of a problematic chain of small `fwrite()` calls in the previous checkpoint code (hundreds of per-element writes through the `ckpt_w` wrapper while VeloC's FP state was in flux)

**Changes to `src/Smilei.cpp`:**
1. **Completely rewrote** the checkpoint save/load pipeline to use a **single bulk `fwrite()`** of a pre-assembled byte buffer per rank, eliminating the per-element fwrite pattern that was the segfault vector
2. Replaced ad-hoc serialisation over `double_prop_`/`short_prop_`/`uint64_prop_` with a deterministic, format-explicit `pack_particles`/`unpack_particles` pair that handles `Position`, `Position_old`, `Momentum`, `Weight`, `Charge`, `Chi`, `Tau`, `Id`, `cell_keys`, `first_index`, `last_index` with conditional emission keyed on `has_*` flags
3. On restart, the routed file is read into a `std::vector<uint8_t>` in one `fread()` and parsed in-memory via `RBuf` (no FP juggling during deserialisation)
4. Added `MPI_Barrier(smpi.world())` immediately before `VELOC_Checkpoint_begin` and `VELOC_Restart_begin` to guarantee collective entry under VeloC's `mode=sync`
5. Added `SMILEI_VELOC_FIRST_CKPT=100` lower bound and dropped the final-iteration checkpoint to avoid mixing VeloC's file I/O with the validation-output dump that follows the loop
6. Removed unused `<sys/stat.h>` include and the dead `host_nparts_` write

**Unchanged (already correct from previous iteration):**
- `veloc.cfg` (static, root of tree, two distinct absolute /tmp paths)
- `makefile` VeloC paths (`-I`, `-L`, `-lveloc-client`, `-Wl,-rpath`)
- `VELOC_Init` after MPI init, `VELOC_Restart_test`/`Load` before time loop, `VELOC_Finalize(0)` before `return 0`
