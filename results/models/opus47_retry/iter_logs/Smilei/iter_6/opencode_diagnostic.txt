## Summary of changes for iter 6:

**Root cause hypothesis:** The EFAULT errors during checkpoint write came from iterating `EM->allFields`, which contains a mixed bag of fields:
- Canonical "owner" fields (Ex_, Ey_, Ez_, Bx_, By_, Bz_, etc.) - always have valid `data_` pointers
- Per-species view fields (`Jx_s[ispec]`, `Jy_s[ispec]`, `Jz_s[ispec]`, `rho_s[ispec]`) - can be reallocated/aliased via `Field::deallocateDataAndSetTo()` leaving dangling pointers
- Complex fields (cFields with `data_=NULL`)
- Optional/lazy-allocated fields (envelope, BTIS3, AM-mode, PXR, etc.)

When `write(fd, fld->data(), n*sizeof(double))` was called with a dangling pointer (from `deallocateDataAndSetTo` aliasing), the kernel detected the unmapped pages and returned EFAULT.

**Fix in `/home/ndhai/diaspora/guard-agent/build/tests_baseline_opus47_retry/Smilei/src/Smilei.cpp`:**

1. **New helper `canonical_em_fields(ElectroMagn*)`** (lines 521-547): returns a fixed-length ordered list of just the 14 canonical owner field pointers (Ex_, Ey_, Ez_, Bx_, By_, Bz_, Bx_m, By_m, Bz_m, Jx_, Jy_, Jz_, rho_, rhoold_). These are owned exclusively by their EMfields object, allocated by `allocateDims()`, never aliased.

2. **Save side** uses `canonical_em_fields(EM)` instead of `EM->allFields` (lines 617-621), so dangling pointers can no longer leak into `write()`.

3. **Load side** mirrors the same list (line 880) so the on-disk layout aligns.

4. **Defense-in-depth staging buffers** in `stream_vec`, `unstream_vec`, the field write/read, and the particle `write_padded_double/short/u64` / `read_into_double/short/u64` helpers: bulk payloads are now memcpy'd through a freshly-allocated `std::vector<T> staging(...)` before/after the syscall. This means if any source pointer is dangling, the userspace memcpy crashes loudly (SIGSEGV) rather than the kernel returning EFAULT silently — and in the common (valid) case, the syscall always sees memory we just allocated.

5. **Magic bumped** from `SMILEIC5` to `SMILEIC6` (line 135) so any stale checkpoint files from prior iterations with the old (allFields-iterating) format are rejected on load.

The skipped per-species `Jx_s/Jy_s/Jz_s/rho_s` fields are recomputed every time-step in `VectorPatch::sumDensities()` before they're used, so excluding them from the checkpoint doesn't affect physical correctness.
