## Changes Made

### Root Cause Analysis
In iter_3, the app ran to completion with exit=0 during all 3 kill attempts, with 0 checkpoint files observed. The root causes were:
1. `CKPT_INTERVAL = 60000` was correct, but there may have been issues with VeloC not finding scratch/persistent directories if they were deleted (not just emptied) by the validator's cleanup
2. The `max_ckpts = 1` field in veloc.cfg is NOT a valid VeloC config key — the correct keys are `max_versions` and `scratch_versions`
3. The `purgeOldCheckpoints()` function (though the logic was correct) was unnecessary complexity that could race with VeloC's own version management

### Files Modified

**`veloc.cfg`** (both root and `examples/laminar/bstep2d/`):
- Replaced invalid `max_ckpts = 1` with correct VeloC fields: `max_versions = 1` and `scratch_versions = 1`

**`examples/laminar/bstep2d/bstep2d.cpp`**:
1. **Directory creation**: Added `mkdir(VELOC_SCRATCH)` and `mkdir(VELOC_PERSISTENT)` at startup (rank 0 + MPI_Barrier) before VELOC_Init, ensuring directories exist even if the validator removed them
2. **CKPT_INTERVAL reduced**: `60000 → 50000` (first checkpoint at ~8s, well before any kill at 50%+ of 122s = 61s+)
3. **veloc.cfg path**: CWD-relative `"veloc.cfg"` is tried first (validator copies it there), with absolute fallbacks
4. **Removed `purgeOldCheckpoints()`**: Eliminated entirely; rely on VeloC's `max_versions=1`/`scratch_versions=1` for disk management
5. **VELOC_Init return value**: Now logged to stderr for diagnostics
6. **Added diagnostic logging**: Checkpoint results logged per event
