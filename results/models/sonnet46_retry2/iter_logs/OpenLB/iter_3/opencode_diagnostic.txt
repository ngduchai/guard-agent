Actually, in VeloC with mode=sync, VELOC_Checkpoint is a collective call that blocks until all ranks complete. All ranks would then call purgeOldCheckpoints at the same time. Since each rank writes its own file (e.g., `bstep2d_ckpt-60000.dat.0`, `bstep2d_ckpt-60000.dat.1`, etc.), all ranks trying to unlink all old files simultaneously is fine - they'll each try to unlink the same files, and some might fail with ENOENT, but that's harmless.

However, there's a subtle race: what if rank 0 deletes the files before rank 1 finishes reading them? Since VELOC_Checkpoint is synchronous (mode=sync) and returns only after completion, all checkpoint files are written by the time we call purge. So purge happens after all data is committed. This is safe.
