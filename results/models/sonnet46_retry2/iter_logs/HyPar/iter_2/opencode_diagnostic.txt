Here's a summary of what I changed and why:

## Summary of Changes

### Root Cause of Failure
The previous iteration checkpointed **every single iteration** of 4,700,000 total iterations. This caused:
1. ~884,435 checkpoint files written during the initial run → massive I/O overhead, slowing the run from 86.8s to 102.6s
2. During recovery, the process started from essentially the beginning (the checkpoint at nearly the kill point) and needed to handle enormous file overhead
3. Recovery timed out at 99.6s → ratio of 2.49x (vs 1.2x cap)

### Fix Applied (`src/Simulation/Solve.cpp`)

**1. Reduced checkpoint frequency** from every iteration to every `n_iter/20` iterations:
```cpp
int veloc_ckpt_interval = TS.n_iter / 20;  // ~235,000 iterations per checkpoint
```
With 4,700,000 total iterations and ~54,000 iterations/second, this means:
- Only ~20 checkpoints written per full run (~4.35 seconds between checkpoints)
- Kill at 60.4s → last checkpoint at ~60.4s - ~2.2s = ~58.2s
- Recovery needs to redo only ~2.2s of work plus startup
- Expected ratio: ~(60.4 + 3) / 86.8 ≈ **0.73x** — well under the 1.2x cap

**2. Added MPI_Allreduce for checkpoint version consensus** to prevent potential rank disagreements on whether a checkpoint exists:
```cpp
MPI_Allreduce(&ckpt_version, &global_ckpt_version, 1, MPI_INT, MPI_MIN, MPI_COMM_WORLD);
```
This ensures all ranks agree on the version before calling `VELOC_Restart`, preventing deadlocks where some ranks enter restart and others don't.
