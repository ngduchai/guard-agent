### Problem
The SPARTA DSMC application had no checkpoint/restart support. The validator kills the process mid-run to simulate a fault, then restarts and expects the simulation to complete correctly using the VeloC library.

### Solution

**1. `veloc.cfg` (source tree root + `examples/free/`)**
- Required by the validator (root copy) and by the binary at runtime (`examples/free/` copy, since the binary runs in that directory).
- Specifies `scratch = /tmp/sparta_veloc_scratch`, `persistent = /tmp/sparta_veloc_persistent`, `mode = sync`.

**2. `src/MAKE/Makefile.mpi`**
- Added `-I/home/ndhai/usr/include` so the compiler finds `veloc.h`.
- Added `-L/home/ndhai/usr/lib -Wl,-rpath,/home/ndhai/usr/lib` and `-lveloc-client` so the linker links against libveloc-client.

**3. `src/main.cpp`**
- `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` after `MPI_Init` — initializes VeloC in collective (MPI) mode.
- `VELOC_Finalize(0)` before `MPI_Finalize` — cleans up VeloC.

**4. `src/update.cpp`**
- `veloc_do_checkpoint()`: Writes per-rank checkpoint files (routed by VeloC) containing `ntimestep`, `nlocal`, and the full `particles[]` array.

  - During the **loop**: checkpoints every 5000 steps using the current `ntimestep` as the VeloC version (monotonically increasing).
