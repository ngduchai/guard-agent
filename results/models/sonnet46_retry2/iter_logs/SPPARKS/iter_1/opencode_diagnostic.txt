## Summary

I implemented checkpoint/restart support for the SPPARKS MPI application using the VeloC library. Here's what was done and why:

### 1. `veloc.cfg` (new file, source tree root)
**Why:** The validator requires this static file before launching the binary. It defines the scratch and persistent storage directories (both must be absolute `/tmp` paths, distinct from each other) and the checkpoint mode.

### 2. `src/MAKE/Makefile.mpi` + `src/Obj_mpi/Makefile`
**Why:** Added VeloC include path (`-I/home/ndhai/usr/include`), library path + RPATH (`-L/home/ndhai/usr/lib -Wl,-rpath,...`), and the VeloC client library (`-lveloc-client`) so the application links against VeloC.

### 3. `src/main.cpp`
**Why:** VeloC must be initialized before any checkpoint calls and finalized after the simulation. Added `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` right after `MPI_Init` and `VELOC_Finalize(0)` before `MPI_Finalize`.

### 4. `src/app_lattice.h`
**Why:** Added `veloc_version` member (monotonically increasing checkpoint version counter) and declarations for three helper methods.

### 5. `src/app_lattice.cpp`
**Core logic — why each part:**

- **`veloc_register_state()`**: Registers all simulation state with VeloC — per-site integer arrays (`iarray`), per-site double arrays (`darray`), simulation time, sweep counter, accepted event counter, RNG seed, and per-site RNG seeds (for COLOR_STRICT mode). This is the complete state needed for restart.

- **`veloc_do_checkpoint(version)`**: Wrapper around VeloC's begin/mem/end sequence.

- **`veloc_restore_latest()`**: Checks for an existing checkpoint using `VELOC_Restart_test(name, INT_MAX)` (INT_MAX finds any available version) and restores state if found.

- **`iterate()`**: The entry point for each simulation run. Registers state, attempts restart, and if successful: updates ghost sites via `comm->all()` and recomputes KMC propensities for the restored site states. This is critical — without propensity recomputation, the KMC solver would use stale probabilities from the original (pre-crash) state.

- **Periodic checkpointing in each iterate loop**: Every 10 sweeps (or 1000 events for global KMC) during the main loop, plus a final checkpoint at the end of each run. The version counter increments monotonically to satisfy VeloC's strict versioning requirement.
