### 4. `Source/Evolve/WarpXEvolve.cpp`
Added a VeloC checkpoint/restart framework in an anonymous namespace:

**At the start of `WarpX::Evolve()`**: calls `velocTryRestart()` which:
1. Sizes flat buffers to match local MultiFab data (E/B field fp components)
2. Registers all buffers with `VELOC_Mem_protect`
3. Calls `VELOC_Restart_test(name, INT_MAX)` to detect any existing checkpoint
4. If found, calls `VELOC_Restart()` to restore all registered memory regions
5. Unpacks restored buffer data back into the E/B field MultiFabs
6. Updates `istep[lev]`, `t_new[lev]`, `t_old[lev]` to the checkpointed values

**In the time-step loop**: every `VELOC_CKPT_INTERVAL = 2000` steps, calls `velocDoCheckpoint()` which:
1. Packs all local MultiFab FAB data into flat double buffers
2. Stores step number and current time in metadata buffer
3. Calls `VELOC_Checkpoint(name, step)` to persist all protected regions

**Key design decisions:**
- Checkpoints every 2000 steps (40 checkpoints total for 80000-step run)
- Uses `mode = sync` in VeloC config - checkpoint is synchronous and complete before continuing
- Checkpoints the full FAB data including ghost cells for direct restoration without FillBoundary
- The restart correctly sets `istep` so the loop resumes from the checkpointed step
- All VeloC API calls are made collectively (all MPI ranks participate simultaneously)
