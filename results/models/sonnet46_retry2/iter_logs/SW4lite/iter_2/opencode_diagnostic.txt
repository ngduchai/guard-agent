Here is a summary of the changes made to fix the disk exhaustion problem:

**Root cause**: The previous implementation checkpointed at every single timestep (~461 steps total), writing ~400MB per checkpoint with no retention limit. This generated 20GB of checkpoint data in just 12 seconds, causing the disk guard to kill the app before any real work could be validated.

**Changes made:**

1. **`veloc.cfg`** — Added `max_ckpts = 2`:
   ```
   max_ckpts = 2
   ```
   This tells VeloC to keep at most 2 checkpoint versions, automatically purging older ones. Disk usage is now bounded at ~2x the size of a single checkpoint.

2. **`src/EW.C`** (lines 2774-2791) — Added checkpoint interval guard:
   ```cpp
   int ckpt_interval = mNumberOfTimeSteps / 10;
   if( ckpt_interval < 5 ) ckpt_interval = 5;
   if( currentTimeStep % ckpt_interval == 0 || currentTimeStep == mNumberOfTimeSteps )
   {
       // ... checkpoint code ...
   }
   ```
   This limits checkpoints to every ~10% of total timesteps (≈46 steps for the validation test), reducing checkpoint count from ~460 to ~10. Combined with `max_ckpts=2`, total disk usage stays well under the 10.7GB guard limit.
