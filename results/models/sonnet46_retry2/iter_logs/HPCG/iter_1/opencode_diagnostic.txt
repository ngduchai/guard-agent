**Logic flow for fault tolerance:**
- Fresh run: `VELOC_Restart_test` returns VELOC_FAILURE (negative), so `start_iter = 0`, full run
- After crash mid-run: `VELOC_Restart_test` returns latest checkpoint version (e.g., 50 if crashed after iteration 50), `VELOC_Restart` loads `veloc_iter=50`, `testnorms_data.values[0..49]`, and `times` array, `start_iter=50`, resumes from iteration 50
- Final result: Since `testnorms_data.values[0..numberOfCgSets-1]` is fully restored + completed, `dumpValidationSignatureBin` produces identical output to a non-interrupted run

## Changes Made

### 1. `veloc.cfg` (new static file in source tree root)
**Why:** Required by the validator before `mpirun` launches. Contains absolute /tmp paths for scratch/persistent checkpoint storage.

### 2. `setup/Make.Linux_MPI`
**Why:** The binary needs to compile with VeloC headers and link against `libveloc-client`. Added `-I/home/ndhai/usr/include` to includes and `-L/home/ndhai/usr/lib -lveloc-client -Wl,-rpath,/home/ndhai/usr/lib` to libs.

### 3. `src/main.cpp`
**Why:** Core checkpoint/restart logic. Key changes:
- `VELOC_Init(MPI_COMM_WORLD, "veloc.cfg")` — right after MPI_Init
- Three protected memory regions: iteration counter (`veloc_iter`), norms array (`testnorms_data.values`), timing array (`times`)
- Before the CG timing loop: `VELOC_Restart_test` → if checkpoint exists, `VELOC_Restart` to reload state and set `start_iter`
- The main `for` loop starts from `start_iter` instead of 0
- After each CG set: `VELOC_Checkpoint("hpcg_cg", i+1)` saves iteration index + accumulated results
- `VELOC_Finalize(0)` before MPI_Finalize

### 4. `bin/xhpcg_run`
**Why:** The validator runs the binary from a per-run CWD that differs from the source tree. The runner already symlinks `hpcg.dat`; I added a parallel symlink for `veloc.cfg` so `VELOC_Init("veloc.cfg")` finds the config regardless of CWD.
