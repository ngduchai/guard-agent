   - Main loop: Checkpoint every 10 cycles using VeloC + manual file I/O
   - Cleanup: `VELOC_Finalize(0)` before `MPI_Finalize`

The checkpoint/restart strategy:
- **Checkpoint**: VeloC manages scalar metadata (time, ncycle, dt) via `VELOC_Mem_protect`/`VELOC_Checkpoint_mem`. Additional per-block binary files are written directly to VeloC's persistent directory for the hydro conserved variables.
- **Restart**: VeloC detects checkpoint via `VELOC_Restart_test`, restores scalar metadata via `VELOC_Recover_mem`, and reads per-block hydro data from the persistent directory. Restoration happens BEFORE `Initialize` so the boundary exchange uses the correct state.
