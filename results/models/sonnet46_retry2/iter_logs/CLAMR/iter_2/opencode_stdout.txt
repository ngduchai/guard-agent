Here's a summary of what was diagnosed and fixed:

## Root Cause

The recovery run (attempt 2) segfaulted in `MallocPlus::memory_realloc_all` called from `main`. The previous restart code called `mesh->resize(ckpt_ncells)` and `state->resize(ckpt_ncells)`, which both invoke `memory_realloc_all()`. This function iterates over **all** entries in the MallocPlus pointer dictionary and calls `realloc` on each. After the normal initialization (which runs `calc_neighbors_local`, `rezone_all`, etc.), the mesh's MallocPlus dictionary contains arrays in complex states — some with ghost cell sizes, some with different sizes after rezone operations. Calling `memory_realloc_all` on this state led to a realloc on a corrupted/stale pointer.

## Fix

In the restart code, replaced:
```cpp
mesh->resize(ckpt_ncells);      // calls memory_realloc_all - crashes
state->resize(ckpt_ncells);     // calls memory_realloc_all - crashes
mesh->memory_reset_ptrs();
state->memory_reset_ptrs();
```

With targeted per-array `memory_realloc` calls that only resize the specific arrays needed for checkpoint restoration:
```cpp
mesh->i     = (int *)    mesh->mesh_memory.memory_realloc(ckpt_ncells, mesh->i);
mesh->j     = (int *)    mesh->mesh_memory.memory_realloc(ckpt_ncells, mesh->j);
mesh->level = (uchar_t *)mesh->mesh_memory.memory_realloc(ckpt_ncells, mesh->level);
state->H    = (state_t *)state->state_memory.memory_realloc(ckpt_ncells, state->H);
state->U    = (state_t *)state->state_memory.memory_realloc(ckpt_ncells, state->U);
state->V    = (state_t *)state->state_memory.memory_realloc(ckpt_ncells, state->V);
```

This matches the same safe pattern already used throughout the codebase in `apply_boundary_conditions`, `remove_boundary_cells`, etc.
