### 3. `clamr_mpionly.cpp` (modified)

**What**: Full checkpoint/restart implementation using VeloC file-based I/O.

**Why**: CLAMR is an AMR code with dynamically changing mesh sizes, so file-based I/O (via `VELOC_Route_file`) is more appropriate than fixed `VELOC_Mem_protect` regions.

**Key decisions**:

- **VELOC_Init** after `L7_Init` (which establishes MPI)
- **Checkpoint trigger**: After every `outputInterval` cycles (when `ncycle % outputInterval == 0 && ncycle > 0`). Each checkpoint writes the local `ncells`, scalars (`ncycle`, `simTime`, `deltaT`, `H_sum_initial`), mesh topology (`i`, `j`, `level`), and state arrays (`H`, `U`, `V`) to a VeloC-routed binary file.
- **Restart detection**: After normal initialization (`mesh->init`, `state->init`, `state->fill_circle`), check for existing checkpoint with `VELOC_Restart_test`. If found: read the checkpoint file, resize buffers to match saved ncells, overwrite topology/state arrays, rebuild `nsizes`/`ndispl`/`noffset`, and recompute derived mesh data (`calc_celltype`, `set_bounds`, `calc_neighbors_local`).
- **H_sum_initial guard**: The initial mass is only overwritten from the current mass on a fresh start; on restart, the saved value is used for conservation checks.
- **VELOC_Finalize** before `L7_Terminate` at program end.
