## Summary of Changes

**File modified**: `source/test/applications/LinAdv/main.cpp`

**Root cause of the crash**: In the previous attempt, `restoreCheckpoint()` called `gridding_algorithm->regridAllFinerLevels()` to recreate fine AMR levels after loading checkpoint data. This crashed with SIGSEGV in `RefineSchedule::recursiveFill` because:
- `regridAllFinerLevels()` → `initializeLevelData(initial_time=false)` needs to time-interpolate between CURRENT and NEW contexts on the coarser level
- After `resetTimeDependentData()`, NEW context is deallocated
- The crash was due to accessing null NEW context data

**Fix**: Redesigned the checkpoint/restart strategy:

1. **`writeCheckpoint`**: Now saves data for ALL levels (not just level 0), using the new `writeLevelData` helper function. This provides a complete snapshot at each checkpoint.

2. **`restoreCheckpoint`**: Completely redesigned:
   - Reads checkpoint data and restores level-0 CURRENT-context data (level-0 mesh is always identical between runs)
   - **Removes fine levels** after restoring data (avoids any need to call `regridAllFinerLevels`)
   - Updates integrator scalars (`d_integrator_time`, `d_step_level[0]`, etc.) to match checkpoint
   - **Does NOT call `regridAllFinerLevels`**

3. **After restart**: The first `advanceHierarchy()` call works with level-0 only:
   - Advances level-0 correctly from checkpoint data
   - At the natural regrid point (every 2 steps), SAMRAI's internal `regridAllFinerLevels` is called — at this point BOTH CURRENT (checkpoint) and NEW (advanced) contexts exist on level-0, so the time interpolation works safely
   - Fine levels are recreated correctly from the restored level-0 data

This approach avoids the crash entirely while producing correct physics output.
