The previous implementation (iter_5) failed because:
1. After recovery, it only restored **level 0** data from checkpoint
2. It removed ALL fine levels (1, 2) and let the time integrator rebuild them
3. Fine levels were rebuilt via **spatial interpolation** from the restored coarse level
4. For the SPHERE problem with sharp discontinuities (uval_inside=80, uval_outside=5), interpolated fine-level data is fundamentally different from the original evolved data
5. The `dumpValidationSignature` sums over ALL cells in ALL levels, so wrong fine-level data causes the large `max_abs_diff=6.25e8`

## Fix

The new implementation correctly checkpoints and restores **ALL AMR levels**:

### Checkpoint Changes
- Added `finest_level_number` to header
- For each level, writes the `ratio_to_level_zero` (refinement ratio)
- For each patch, writes `local_id` and `owner_rank` (needed to match boxes on restore)
- Changed magic number from `0x4C56434C` to `0x4C56434D` to force fresh checkpoints

### Restore Changes
1. Remove ALL fine levels from the initial hierarchy
2. Restore level 0 data (fixed structure, match by `(local_id, owner_rank)`)
3. For each fine level in checkpoint:
   - Construct a `BoxLevel` with the checkpoint's exact box layout (using `addBoxWithoutUpdate` to preserve original `local_id`)
   - Call `patch_hierarchy->makeNewPatchLevel(ln, box_level)` to recreate the level
   - Allocate CURRENT patch data on the new level
   - Fill each patch with checkpoint data (matching by `(local_id, owner_rank)`)
4. Set integrator state (time, step count, dt, level times)
5. After restore: call `hyp_level_integrator->resetHierarchyConfiguration(...)` to rebuild the cached ghost-fill communication schedules for all levels

This approach correctly restores the **exact** evolved state at all AMR levels, including the precise patch layout and cell data that the original simulation had at checkpoint time.
