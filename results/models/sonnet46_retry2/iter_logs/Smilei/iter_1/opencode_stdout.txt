## Implementation Summary

### Design Rationale

**State to checkpoint**: The simulation state in Smilei consists of:
1. The current timestep counter (`itime`, `time_prim`, `time_dual`)
2. EM fields per patch (Ex, Ey, Ez, Bx, By, Bz, Bx_m, By_m, Bz_m, Jx, Jy, Jz, rho) - fixed-size arrays
3. Particle data per species per patch (position, momentum, weight, charge, optional Chi/Tau/Id) - variable-size

**File-based approach**: Used `VELOC_Route_file` for all data to handle variable-size particle arrays. This is more flexible than `VELOC_Mem_protect` which requires pre-registered fixed-size regions.

**Checkpoint frequency**: Every timestep, ensuring minimal re-computation on restart (at most 1 step is lost).

**Restart logic**: Placed AFTER `PatchesFactory::createVector` so the simulation data structures are properly initialized before we overwrite them with checkpoint data.

**Collective MPI**: All VeloC calls happen outside OMP parallel regions, ensuring all MPI processes participate in collective operations.

### Files Modified

1. **`veloc.cfg`** (created): Required static configuration for VeloC runtime
2. **`makefile`**: Added VeloC include/library paths
3. **`src/Smilei.cpp`**: Added complete checkpoint/restart logic with VeloC integration
