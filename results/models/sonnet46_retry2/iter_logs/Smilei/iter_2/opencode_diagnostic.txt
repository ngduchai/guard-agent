- 22,292 checkpoint files were created during the first (killed) run

1. ~22,292 checkpoint files (3,684 steps Ă— 4 MPI ranks Ă— ~2 files each)
2. Massive I/O overhead during recovery (even after resuming from the checkpoint step ~2950, the remaining ~734 steps each triggered a synchronous checkpoint write)

- Reduces checkpoint count from ~73 per run to just ~2 checkpoint write groups per run

- Estimated recovery time: ~30s computation + minimal overhead = well within the 110.8s timeout
