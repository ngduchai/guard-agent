The fixes look solid. Let me summarize what was done:
The changes made to fix the disk exhaustion problem:

**`veloc.cfg`** - Added version retention limits:
- `max_versions = 1`: Keep only 1 previous checkpoint on persistent storage (instead of unlimited)
- `scratch_versions = 1`: Keep only 1 previous checkpoint on scratch storage (instead of unlimited)

This alone reduces from 30,099 files (~12GB) to at most 8 files (~16MB) at any time.

**`phold.main.c`** - Reduced checkpoint frequency:
1. Changed `SEQ_CKPT_INTERVAL` from `500.0` to `4000.0` (simulation time between sequential checkpoints, reducing from ~38 to ~5 checkpoints)
2. Added `CKPT_WALL_INTERVAL_S = 20.0` wall-clock gate: a checkpoint only happens if 20+ real seconds have elapsed since the last one
3. Changed `tw_trigger_gvt_hook_every(5)` to `tw_trigger_gvt_hook_every(128)`: GVT hook fires 25× less often in parallel mode

These changes together ensure:
- At most ~4 checkpoints per 90-second run (one every 20+ wall seconds)
- At most 2 checkpoint versions retained at any time (VeloC auto-purges older ones)
- Maximum disk usage ~16MB instead of 12GB
