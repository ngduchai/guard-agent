#### 3. `models/phold/phold.main.c` (modified)
The core checkpoint/restart logic:

**Checkpoint (via GVT hook `phold_gvt_hook`):**
1. **For optimistic mode**: Call `tw_kp_rollback_to(kp, GVT)` for each KP to revert speculative processing and get committed LP state + RNG
2. **Drain the priority queue**: Collect all pending events (recv_ts, dest_gid), then re-enqueue them
3. **Write binary checkpoint file** via `VELOC_Route_file` → per-rank file containing: LP states (phold_state), RNG streams (Ig/Lg/Cg/count), pending event records
4. **Commit** via `VELOC_Checkpoint_end(1)`

For sequential mode (no `tw_trigger_gvt_hook_every` support), use timestamp-based triggers re-armed from within the hook.

**Restart detection (in `main`):**
- `VELOC_Restart_test(VELOC_CKPT_NAME, INT_MAX)` to find the latest checkpoint version
- If found: call `phold_load_checkpoint(version)` → reads per-rank binary file into restore buffers, sets `g_phold_restart = 1`

**State restoration (via `phold_pre_run`):**
- `phold_init` skips all event generation when `g_phold_restart == 1`
- `phold_pre_run` (now registered as `pre_run_f`) restores each LP's `phold_state` and RNG stream (Ig/Lg/Cg) from checkpoint data
- On LP 0's `pre_run` call: injects all saved pending events using `tw_event_new(dest_gid, saved_recv_ts, lp)` — since `kp->last_time = 0` at pre_run time, `recv_ts = 0 + saved_recv_ts = saved_recv_ts` (exact restoration of absolute event timestamps)
