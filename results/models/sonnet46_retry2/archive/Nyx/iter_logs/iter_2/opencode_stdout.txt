## Summary of Changes

**Root Cause**: The previous implementation wrote VisMF checkpoint files to per-rank paths (e.g., `scratch/rank_0/nyx_lev0_stype0`, `scratch/rank_1/nyx_lev0_stype0`). Since `VisMF::Write` and `VisMF::Read` are MPI-collective operations that ALL ranks must call with the SAME path name, this caused:
1. **Recovery crash** (exit code 6): On restart, each rank tried to read from its own rank-specific path. The VisMF internal MPI communication expected all ranks to coordinate on the same file, causing data corruption or abort.
2. **Failure-free run hang**: In the clean run, each rank called `VisMF::Write` with different paths. VisMF's internal `NFilesIter` communication protocol broke down because ranks were writing to different files while expecting to coordinate, causing a deadlock.

**Fix Applied to `nyx_main.cpp`**:
1. **Removed rank from path**: `veloc_mfab_path` now returns a shared path like `/tmp/nyx_veloc_persistent/nyx_mfab/lev0_stype0` (no rank in path).
2. **Correct directory creation**: Only rank 0 creates the directory, followed by a barrier.
3. **Proper VisMF::Exist**: The collective file existence check is now used correctly.
4. **BoxArray match broadcast**: Added explicit Bcast to ensure all ranks agree on whether BoxArrays match before deciding to read or skip.
5. **Removed unused scratch dir**: Simplified the path management.
6. **Improved time sync**: Used `setTimeLevel` instead of just `setNewTimeLevel` to properly synchronize both old and new time slots on restart.
