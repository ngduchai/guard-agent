## Root Cause Analysis

The recovery was timing out at 1.86× baseline because the restored simulation ran much slower than expected. The sentinel file IS found and MultiFab data IS restored (level 0), but two bugs caused the simulation to take very small timesteps after restart:

1. **Bug 1 (Critical): `dt_level` not restored.** After `amrptr->init(0, stop_time)`, `dt_level[i]` is set by `computeInitialDt` using the **initial conditions** (with `init_shrink=0.1`, giving a very small initial dt ≈ 0.0008). `computeNewDt` limits the new dt to `change_max × dt_level[i] = 1.05 × 0.0008 = 0.00084`. Instead of resuming with the checkpoint dt (~0.4+ comoving time units), the simulation had to slowly grow dt from 0.0008 at 5% per step — requiring thousands of steps to reach the correct dt, far exceeding the 1.28× timeout.

2. **Bug 2 (Secondary): Finer level time inconsistency.** After `init(0, stop_time)`, levels 1 and 2 have their state times set to the initial conditions time (~0). When `computeNewDt` calls `get_level(i).est_time_step()`, which calls `get_comoving_a(curTime_of_level_i ≈ 0)`, this time is outside the valid range `[old_a_time, new_a_time]` = `[restored_time - dt_last, restored_time]`. This causes either a crash (iter_4) or incorrect behavior.

3. **Bug 3 (Secondary): FillPatch time mismatch during regrid.** `Nyx::init(AmrLevel& old)` reads `old_level->state[...].curTime()` to determine the FillPatch time. Without setting all levels to `restored_time` before regrid, FillPatch would request data from level 0 at the wrong time.

## Fix Applied

The restore block in `nyx_main.cpp` now:

1. **Step 3**: Restores `dt_level[i] = dt_last / prod(n_cycle[0..i])` for all levels — ensures `computeNewDt` starts with the correct checkpoint dt and doesn't throttle to a tiny value.

2. **Step 4**: Sets time metadata for ALL levels (including 1, 2) to `restored_time` BEFORE calling regrid — ensures `Nyx::init(old)` uses the correct FillPatch time.

3. **Step 5**: Calls `amrptr->regrid(0, restored_time)` to rebuild finer levels from the restored level-0 checkpoint data — ensures levels 1, 2 have physically consistent state (not IC data from t=0).

4. **Step 6**: After regrid, sets time metadata and dt_level for ALL levels (including newly created ones) — ensures `get_comoving_a(curTime())` always succeeds for all levels.
