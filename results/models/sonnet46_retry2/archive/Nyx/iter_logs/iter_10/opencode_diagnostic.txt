### Root Cause Analysis
Previous attempts failed because:
1. The sentinel file was written AFTER VeloC checkpoint files appeared in the VeloC dirs, so the validator would kill the process before the sentinel was written
2. The complex regrid-based restart logic was fragile and produced incorrect results
3. The failure-free run would find a stale sentinel from attempt_1, restart with buggy logic, and produce wrong output

### Solution

**Strategy**: Implement the stripped-out `Amr::checkPoint()` and `Amr::restart()` functions directly in the AMReX source, then use AMReX's ParmParse-based restart mechanism (`amr.restart`) for clean checkpoint/restart.

#### Files Modified:

1. **`subprojects/amrex/Src/Amr/AMReX_Amr.cpp`**
   - Added `#include <AMReX_VisMF.H>`
   - Implemented `Amr::restart(filename)`: reads checkpoint Header, restores level hierarchy with correct BoxArrays, loads MultiFab data, sets all AMR metadata
   - Implemented `Amr::checkPoint()`: writes AMReX-format checkpoint with Header + per-level MultiFab data

2. **`subprojects/amrex/Src/Amr/AMReX_AmrLevel.cpp`**  
   - Implemented `AmrLevel::checkPoint(dir, os, how, dump_old)`: writes per-level metadata to header stream, writes MultiFab data via VisMF
   - Implemented `AmrLevel::restart(papa, is, bReadSpecial)`: loads MultiFab data from checkpoint files using absolute paths derived from `papa.theRestartFile()`

3. **`Source/Driver/nyx_main.cpp`** (complete rewrite)
   - Uses `find_latest_checkpoint()` to find the most recent checkpoint in `/tmp/nyx_veloc_ckpt/`
   - If found: adds `amr.restart = <path>` to ParmParse BEFORE creating `Amr*`
   - `amrptr->init()` automatically calls `Amr::restart()` when `amr.restart` is set
   - After init: calls `read_nyx_comoving()` to restore Nyx-specific comoving parameters
   - Time loop: every CKPT_INTERVAL=50 steps, calls `amrptr->checkPoint()`, writes Nyx comoving params, moves checkpoint to `/tmp/nyx_veloc_ckpt/`, THEN calls VeloC checkpoint (creating files in VeloC dirs for validator detection)
   - The key ordering: AMReX checkpoint is complete BEFORE VeloC files appear → validator detects VeloC files → waits 5s → kills → recovery finds valid AMReX checkpoint

4. **`veloc.cfg`** (unchanged, already correct)
